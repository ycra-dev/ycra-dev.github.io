## 배운 것

### 기술적 학습

**가상화 기본 개념**
- **Virtualization**: 물리적 하드웨어를 논리적으로 분할하거나 통합하여 추상화하는 기술
- **Hypervisor Types**: Type 1 (Bare-metal, 하드웨어 직접 실행) vs Type 2 (Hosted, OS 위에서 실행)
- **Hardware Virtualization**: CPU의 가상화 확장 기능 (Intel VT-x, AMD-V)을 활용한 성능 향상

**가상화 기법**
- **Trap and Emulate**: 게스트 OS의 특권 명령어를 트랩하여 VMM이 에뮬레이션하는 방식
- **Paravirtualization**: 게스트 OS를 수정하여 하이퍼콜(hypercall)로 직접 VMM 호출, 오버헤드 감소
- **Binary Translation**: 특권 명령어를 동적으로 변환하여 실행 (전가상화 방식)

**가상화 응용**
- **Container**: OS 레벨 가상화로 프로세스 격리, VM보다 가볍고 빠름
- **JVM**: 바이트코드 수준의 플랫폼 독립적 가상화, JIT 컴파일러 활용
- **Live Migration**: 실행 중인 VM을 다운타임 없이 다른 물리 서버로 이동

**VMM 리소스 관리**
- **VMM CPU Scheduling**: 가상 CPU를 물리 CPU에 스케줄링, 공정성과 성능 균형
- **VMM Memory Management**: Shadow Page Table, Extended Page Table (EPT/NPT) 등
- **Distributed System**: 여러 독립적인 컴퓨터가 협력하여 하나의 시스템처럼 동작
