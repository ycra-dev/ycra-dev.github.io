## 오늘 배운 것

Operating System Concepts 1장을 읽으면서 운영체제의 기본 개념 세 가지를 정리했다.

- **운영체제**: 하드웨어와 사용자 사이의 중재자. 자원 할당(Resource Allocator)과 프로그램 제어(Control Program) 역할
- **커널**: OS에서 항상 실행되는 핵심 프로그램. 시스템 프로그램, 미들웨어와 구분됨
- **인터럽트**: 하드웨어가 CPU에게 이벤트 발생을 알리는 메커니즘

## 이해한 내용

### 운영체제

- 컴퓨터 시스템 = 하드웨어 + 운영체제 + 응용 프로그램 + 사용자
- OS는 정부(government)와 비슷함. 그 자체로는 유용하지 않지만, 다른 프로그램이 일할 수 있는 환경을 만들어줌
- 두 가지 핵심 역할:
  - Resource Allocator: CPU, 메모리, I/O 등 자원을 효율적/공정하게 분배
  - Control Program: 프로그램 실행 관리, 오류 및 부적절한 사용 방지

### 커널

- "the one program running at all times on the computer"
- OS 구성: 커널 + 시스템 프로그램 + (모바일의 경우) 미들웨어
- 부팅 시 부트스트랩 프로그램이 커널을 메모리에 로드

### 인터럽트

- 왜 필요? → I/O 작업 완료 등 이벤트를 CPU에게 알려야 함
- 동작 흐름:
  1. 하드웨어가 시스템 버스로 CPU에 신호
  2. CPU가 현재 작업 중단, 인터럽트 서비스 루틴(ISR)으로 점프
  3. ISR 실행 후 원래 작업 재개
- CPU는 매 명령어 실행 후 interrupt-request line을 확인
- 종류:
  - Nonmaskable: 끌 수 없음 (심각한 오류용)
  - Maskable: 끌 수 있음 (일반 장치 서비스 요청)

## 헷갈린 점 / 막힌 점

- 인터럽트 벡터가 정확히 어떻게 구성되는지, 운영체제마다 다른지?
- 미들웨어가 모바일 OS에서만 언급되는데, 데스크톱 OS에는 해당 개념이 없는 건지?
