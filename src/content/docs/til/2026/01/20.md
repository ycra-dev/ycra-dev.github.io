## 오늘 배운 것

CLAUDE.md 파일을 활용해 Claude Code가 서로 다른 역할을 수행하도록 설정하고, 멀티 에이전트 협업(Multi-Agent Collaboration) 테스트를 진행했다.

## 이해한 내용

### CLAUDE.md란?

Claude Code가 레포지토리에서 작업할 때 참조하는 가이드 파일이다. 이 파일에 역할, 워크플로우, 규칙 등을 정의하면 Claude Code가 해당 지침을 따라 동작한다.

### 테스트한 3가지 역할

각각 다른 CLAUDE.md를 가진 환경(또는 레포)에서 Claude Code가 다음 역할을 수행하도록 설정했다:

| 역할 | 설명 |
|------|------|
| **PR Reviewer** | `gh pr view/diff/checkout` → 테스트/빌드 → `gh pr review`로 approve/comment/request-changes |
| **Issue Creator** | 백로그/로드맵 확인 → 스프린트 단위로 작업 분해 → `gh issue create`로 상세 이슈 생성 |
| **Developer** | `gh issue view`로 이슈 확인 → 브랜치 생성 → 개발 → 테스트 → PR 생성 |

### 멀티 에이전트 시스템(MAS) 관점

- 각 Claude Code 인스턴스가 CLAUDE.md에 정의된 역할에 따라 **전문화된 에이전트**로 동작
- Issue Creator가 만든 이슈를 Developer가 구현하고, PR Reviewer가 리뷰하는 **파이프라인 구조** 구현 가능
- 동일한 LLM이지만 컨텍스트(CLAUDE.md)에 따라 완전히 다른 행동 패턴을 보임

## 헷갈린 점 / 막힌 점

- 여러 Claude Code 인스턴스 간 **상태 공유**는 GitHub(이슈, PR, 코드)를 통해서만 이루어짐
- 실시간 협업이 아닌 **비동기 협업** 형태
- 하나의 레포에서 역할을 동적으로 전환하는 방식 vs 역할별로 레포를 분리하는 방식 중 어떤 게 더 나은지 아직 판단이 안 섬
