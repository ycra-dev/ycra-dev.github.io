## 핵심 개념

지식 포트폴리오(Knowledge Portfolio)는 개발자가 보유한 기술, 지식, 경험을 금융 투자 포트폴리오에 비유한 개념이다. The Pragmatic Programmer에서 제시되었으며, 투자 포트폴리오를 관리하듯 자신의 지식을 전략적으로 관리해야 한다는 원칙이다.

## 동작 원리

지식 포트폴리오는 투자 포트폴리오와 같은 원칙으로 관리해야 한다:

- **정기적 투자**: 꾸준히 새로운 지식에 투자. 매주 새로운 것을 학습
- **다각화**: 한 기술에만 의존하지 않고 다양한 분야의 지식 확보
- **리스크 관리**: 안정적인 기술(주력 언어)과 새로운 기술(신흥 기술)의 균형
- **저가 매입**: 아직 인기가 없는 유망 기술을 미리 학습
- **정기 점검**: 포트폴리오를 주기적으로 재평가하고 조정

학습 방법:
- 매년 새로운 프로그래밍 언어 하나를 배운다
- 분기마다 기술 서적을 읽는다
- 컨퍼런스와 사용자 그룹에 참여한다
- 다양한 플랫폼과 패러다임을 경험한다
- 코드 카타(kata)를 통한 의도적 연습

## 예시

```
# 지식 포트폴리오 관리 예시

안정적 투자 (70%):
  ├── 주력 언어 심화 (Python, Java 등)
  ├── 주요 프레임워크 숙련
  └── 핵심 CS 기초 (알고리즘, 자료구조)

성장 투자 (20%):
  ├── 새로운 패러다임 (함수형 프로그래밍)
  ├── 클라우드/DevOps 기술
  └── 다른 도메인 지식 (ML, 보안 등)

투기적 투자 (10%):
  ├── 신흥 언어 (Rust, Zig 등)
  ├── 새로운 아키텍처 패턴
  └── 실험적 사이드 프로젝트
```
