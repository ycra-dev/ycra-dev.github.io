## 핵심 개념

1:1 미팅(One-on-One Meeting)은 매니저와 팀원이 정기적으로(보통 주 1회) 갖는 개인 면담이다. 상태 업데이트나 업무 보고가 아닌, 큰 그림의 논의, 피드백 교환, 커리어 개발, 개인적 고충 등을 다루는 **팀원 중심의 시간**이다. 매니저는 일상적인 상태 업데이트를 스탠드업이나 프로젝트 도구에서 확인할 수 있으므로, 1:1에서는 그보다 더 깊고 중요한 주제를 다뤄야 한다.

## 동작 원리

1:1에서 다뤄야 할 주제들:
- **큰 그림**: 팀/회사의 방향성, 우선순위 변화, 전략적 의사결정에 대한 질문
- **피드백**: 본인이 받고 싶은 피드백 요청, 매니저나 팀에 대한 피드백 전달
- **커리어 개발**: 승진 경로, 스킬 갭, 성장 기회, 학습 계획
- **개인적 주제**: 번아웃, 팀 내 갈등, 워라밸 고충 등
- **블로커**: 혼자 해결하기 어려운 조직적/정치적 장애물

효과적인 1:1을 위한 실천 방법:

1. **어젠다를 직접 설정하라**: 미팅 전에 논의할 주제 목록을 준비한다. 매니저가 어젠다를 설정하길 기다리지 마라
2. **1:1 노트를 유지하라**: 매 미팅의 논의 내용과 액션 아이템을 기록한다
3. **취소가 반복되면 문제를 제기하라**: 정기적인 1:1 없이는 피드백 루프가 끊긴다
4. **어려운 대화를 피하지 마라**: 1:1은 불편한 주제를 안전하게 꺼낼 수 있는 공간이다

## 예시

좋은 1:1 어젠다 vs 나쁜 1:1 어젠다:

```
[나쁜 어젠다 - 상태 보고]
1. 로그인 버그 수정 완료
2. 결제 API 70% 진행
3. 다음 주 코드 리뷰 3건 예정
→ 이건 Jira/Slack에서 확인 가능한 정보. 1:1 시간 낭비.

[좋은 어젠다 - 의미 있는 대화]
1. Q2 팀 우선순위에서 우리 프로젝트 위치가 궁금합니다
   - 결제 시스템 리팩토링이 밀리고 있는데, 기술 부채를
     어떻게 균형 잡을 수 있을까요?
2. 최근 코드 리뷰에서 제가 개선할 점이 있다면 피드백 부탁드립니다
3. 시니어 승진을 목표로 하고 있는데, 현재 갭이 무엇인지 논의하고 싶습니다
4. 팀에 온콜이 한 사람에게 집중되고 있어서 걱정됩니다
```

1:1 공유 노트 예시:

```markdown
## 2026-02-26 1:1 노트

### 이전 액션 아이템 확인
- [완료] 김개발: 디자인 시스템 스터디 그룹 참여 등록
- [진행중] 매니저: 크로스팀 프로젝트 기회 물색 중

### 오늘 논의
**커리어:**
- 시니어 승진 기준 중 "기술적 리더십" 항목에서 갭이 있음
- 액션: 다음 프로젝트에서 설계 문서 작성 리드 역할 맡기로 함

**팀 이슈:**
- 온콜 로테이션이 공정하지 않다는 느낌
- 매니저가 다음 팀 미팅에서 데이터 기반으로 재배분 제안 예정

### 액션 아이템
- [ ] 김개발: 다음 주까지 설계 문서 초안 작성
- [ ] 매니저: 온콜 로테이션 히스토리 데이터 수집
```
