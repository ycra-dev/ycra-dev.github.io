## 핵심 개념

소프트웨어 표준은 세 가지 이유로 중요하다: (1) 조직의 경험에서 얻은 지혜를 포착하여 이전 실수를 반복하지 않도록 함, (2) 특정 환경에서 품질의 의미를 정의하는 프레임워크를 제공함, (3) 한 사람이 수행한 작업을 다른 사람이 이어받을 때 연속성을 보장함. 제품 표준(문서 구조, 코딩 표준 등)은 개발 중인 소프트웨어에 적용되고, 프로세스 표준(설계 검토 절차, 버전 릴리스 프로세스 등)은 개발 프로세스에 적용된다. 엔지니어들이 표준을 수용하도록 하려면 표준 선택에 참여시키고, 기술 변화를 반영하여 정기적으로 갱신하며, 도구 지원을 제공해야 한다.

## 예시

제품 표준의 예: 설계 리뷰 양식, 요구사항 문서 구조, 메서드 헤더 형식, Java 프로그래밍 스타일, 프로젝트 계획 형식, 변경 요청 양식. 프로세스 표준의 예: 설계 리뷰 수행 절차, 새 코드의 시스템 빌드 제출 절차, 버전 릴리스 프로세스, 프로젝트 계획 승인 프로세스, 변경 통제 프로세스, 테스트 기록 프로세스.
