## 핵심 개념

기술 문서의 대상 독자(audience)를 명확히 정의하는 것으로, 효과적인 문서 작성의 핵심 전제 조건이다.

## 동작 원리

좋은 문서의 핵심 원칙 중 하나는 **대상 독자를 아는 것**이다. 문서 작성자가 범하는 가장 흔한 실수는 "나 자신을 위해" 쓰는 것이다. 좋은 문서는 독자의 입장에서 작성되어야 한다.

대상 독자를 정의할 때 고려할 점:
- **경험 수준**: 초보자인가, 전문가인가?
- **도메인 지식**: 해당 영역에 대한 사전 지식이 있는가?
- **목적**: 독자가 문서를 통해 무엇을 얻으려 하는가?

Google에서는 모든 문서가 한 종류의 독자만을 대상으로 해야 한다고 권장한다. 한 문서에서 여러 수준의 독자를 동시에 만족시키려 하면 아무도 만족시키지 못하게 된다. 대신, 각 독자 유형에 맞는 별도의 문서를 작성하고 서로 링크한다.

## 예시

같은 API에 대해 세 가지 다른 문서를 작성할 수 있다:
- 초보 사용자를 위한 **튜토리얼**: "Getting Started with the Widget API"
- 기존 사용자를 위한 **참조 문서**: API 메서드와 파라미터 상세 설명
- 기여자를 위한 **개발자 가이드**: 내부 구조와 확장 방법 설명
