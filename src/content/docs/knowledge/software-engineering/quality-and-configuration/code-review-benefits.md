## 핵심 개념

코드 리뷰의 이점은 단순한 버그 검출을 넘어서 코드 정확성, 코드 이해도, 코드 일관성, 심리적 이점, 지식 공유 등 다양한 차원에서 조직에 가치를 제공하는 효과들이다.

## 동작 원리

**1. 코드 정확성(Code Correctness)**: 가장 명확한 이점. 다른 사람의 눈이 버그, 엣지 케이스 누락, 잘못된 가정을 발견한다. 그러나 이는 이점의 일부에 불과하다.

**2. 코드 이해도(Code Comprehension)**: 코드가 작성자뿐만 아니라 리뷰어에게도 이해 가능해야 한다는 요구는 코드의 가독성을 자연스럽게 높인다. "이 코드가 무슨 뜻인지 모르겠다"는 리뷰 코멘트는 코드를 개선하게 만든다.

**3. 코드 일관성(Code Consistency)**: 코드 리뷰는 스타일 가이드와 모범 사례가 실제로 적용되는 핵심 메커니즘이다. 가독성 리뷰어가 일관성을 보장한다.

**4. 심리적 이점(Psychological Benefits)**: 코드가 리뷰된다는 사실만으로 작성자는 더 신중하게 코드를 작성한다. 또한 리뷰 프로세스는 코드 소유권을 공유하여, 개인이 아닌 팀이 코드를 소유한다는 인식을 강화한다.

**5. 지식 공유(Knowledge Sharing)**: 리뷰 과정에서 작성자와 리뷰어 양쪽 모두 학습한다. 작성자는 새로운 기법을 배우고, 리뷰어는 코드베이스의 다른 부분을 이해하게 된다. 이는 Bus Factor를 높이는 효과가 있다.

이 이점들은 시간이 지남에 따라 복리로 누적된다. 오늘의 리뷰 코멘트가 내일의 더 나은 코드를 만든다.

## 예시

구글에서 코드 리뷰의 지식 공유 효과를 측정한 결과, 리뷰를 많이 수행한 엔지니어일수록 코드베이스의 다양한 영역에 기여할 수 있었다. 특히 readability 리뷰어는 수백 명의 엔지니어에게 모범 사례를 전파하는 "확산 효과"를 가졌다. 이는 공식적 교육이나 문서만으로는 달성하기 어려운 유기적 지식 전파이다.
