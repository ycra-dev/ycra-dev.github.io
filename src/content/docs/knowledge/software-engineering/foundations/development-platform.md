## 핵심 개념

프로그래밍 언어가 플랫폼을 결정하지 않는다. 예를 들어 C# 하나로 Windows, Mac, iOS, Android, Linux, 임베디드 시스템 모두 개발 가능하다. 기업들은 보통 특정 플랫폼을 위한 개발자를 채용하기 때문에, 무엇을 선택하느냐보다 **하나를 선택하고 깊이 파고드는 것**이 더 중요하다.

플랫폼의 종류:
- **모바일**: iOS, Android
- **데스크톱**: Windows, Mac, Linux
- **웹**: 웹 브라우저 기반 플랫폼
- **임베디드**: IoT 기기, 마이크로컨트롤러

## 동작 원리

플랫폼에 대한 전문성은 해당 플랫폼 자체뿐 아니라 세 가지를 포함한다:
1. **개발 도구**: IDE, 빌드 시스템, 디버거
2. **관용적 패턴**: 해당 플랫폼에서 코드를 작성하는 일반적인 방식
3. **일반적인 프레임워크**: 그 플랫폼에서 자주 쓰이는 라이브러리들

iOS 개발자로 취업하려면 Swift와 Xcode, iOS 앱의 라이프사이클, UIKit 또는 SwiftUI에 대한 전문성이 필요하다.

## 예시

같은 C# 언어로 다양한 플랫폼에서 개발하는 예:
- **Windows 데스크톱**: WPF, WinForms 프레임워크 사용
- **웹**: ASP.NET Core로 웹 애플리케이션 개발
- **모바일**: Xamarin으로 iOS/Android 앱 개발
- **게임**: Unity 엔진으로 크로스 플랫폼 게임 개발
- **클라우드**: Azure Functions로 서버리스 함수 개발
