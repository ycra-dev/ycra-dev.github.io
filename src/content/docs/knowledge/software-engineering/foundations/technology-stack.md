## 핵심 개념

**프레임워크**와 **스택**의 차이:

- **프레임워크(Framework)**: 특정 플랫폼에서 코드를 개발하는 데 사용되는 라이브러리 세트. 이미지 처리 같은 복잡한 작업을 처음부터 작성하지 않고도 수행할 수 있게 해준다.
- **스택(Stack)**: 완전한 애플리케이션을 만드는 데 일반적으로 함께 사용되는 기술들의 세트

스택을 아는 것이 가치 있는 이유:
1. 많은 개발자가 사용하는 공통 패러다임을 제공한다
2. 지식을 쉽게 공유할 수 있다
3. 특정 기술들이 함께 작동한다는 것이 검증되어 있다
4. 특정 스택을 사용하는 회사들이 해당 스택 개발자를 찾는다

## 동작 원리

스택은 보통 데이터베이스 + 서버 프레임워크 + 프론트엔드 프레임워크의 조합으로 구성된다. 각 계층이 서로 통신하며 하나의 완전한 애플리케이션을 이룬다.

## 예시

```
MEAN Stack (JavaScript 기반):
- MongoDB (NoSQL 데이터베이스)
- Express.js (Node.js 웹 프레임워크)
- AngularJS (프론트엔드 프레임워크)
- Node.js (런타임 환경)

LAMP Stack (전통적 웹 개발):
- Linux (운영체제)
- Apache (웹 서버)
- MySQL (관계형 데이터베이스)
- PHP (서버 사이드 언어)

.NET Stack (Microsoft 생태계):
- C# (프로그래밍 언어)
- ASP.NET (웹 프레임워크)
- SQL Server (데이터베이스)
- Azure (클라우드 플랫폼)
```
