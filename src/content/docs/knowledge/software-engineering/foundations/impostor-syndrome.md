## 핵심 개념

임포스터 신드롬(Impostor Syndrome)은 객관적 성취에도 불구하고 자신이 사기꾼이며 운이 좋았을 뿐이라고 믿는 심리적 패턴이다. 1978년 Clance & Imes가 처음 기술했다. 자기 강화 순환이 핵심이다 — 실수는 무능의 증거로, 성공은 운의 증거로 해석한다.

## 동작 원리

특히 기술 분야에서 빈번한 이유: 끊임없이 새로운 기술이 등장하여 "모르는 것"이 항상 존재하기 때문이다.

대처법:
1. **인식** — 패턴을 감지하면 의식적으로 깨뜨리기
2. **리프레이밍** — "도움 요청 = 무능"이 아니라 "도움 요청 = 학습"
3. **칭찬과 성취 기록하기** — 객관적 증거 축적
4. **신뢰하는 사람에게 피드백 요청** — 외부 시각 확보
5. **필요시 전문 상담** — 심각할 경우 전문가 도움

## 예시

```
승진 후에도 "기준이 낮았을 뿐"이라고 생각한다.
동료가 "잘했다"고 하면 "그냥 말해주는 것"이라고 무시한다.

대처:
- 성취 목록을 문서화하여 객관적 증거 축적
- "나는 X를 모른다" vs "나는 X를 아직 배우고 있다"
```

[Dunning-Kruger Effect](/knowledge/software-engineering/foundations/dunning-kruger-effect/)와 반대 현상:
- Dunning-Kruger: 능력 부족 → 과신 (무의식적 무능)
- Impostor Syndrome: 능력 충분 → 과소평가 (의식적 무능에서 벗어나지 못함)
