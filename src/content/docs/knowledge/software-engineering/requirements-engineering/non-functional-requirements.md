## 핵심 개념

비기능적 요구사항은 제품 요구사항(성능, 신뢰성, 사용성 등), 조직 요구사항(개발 프로세스 표준, 사용 도구 등), 외부 요구사항(법규, 윤리, 상호운용성 등)으로 분류된다. 비기능적 요구사항은 종종 기능적 요구사항보다 더 중요할 수 있는데, 하나의 비기능적 요구사항의 위반이 전체 시스템을 무용하게 만들 수 있기 때문이다. 가능하면 비기능적 요구사항은 정량적으로 측정 가능한 형태로 기술되어야 한다.

## 예시

비기능적 요구사항 측정 기준 예시: 성능 - "초당 처리 트랜잭션 수 300건 이상", 가용성 - "99.9% 가동 시간", 사용성 - "초보 사용자가 2시간 훈련 후 모든 기능 사용 가능", 보안 - "HIPAA 규정 준수".
