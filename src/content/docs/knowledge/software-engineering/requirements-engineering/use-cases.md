## 핵심 개념

유스 케이스는 시스템의 기능적 요구사항을 식별하고 문서화하는 데 사용되며, 액터(사용자 또는 외부 시스템)와 시스템 간의 상호작용을 기술한다. 유스 케이스 다이어그램은 전체 상호작용의 개요를 제공하고, 각 유스 케이스에 대한 상세 텍스트 설명은 구체적인 시나리오를 기술한다. 유스 케이스는 요구사항 도출, 시스템 설계, 테스트 케이스 도출의 기반으로 활용된다.

## 예시

병원 시스템의 유스 케이스 예시: 액터 - 의사, 유스 케이스 이름 - "약물 처방", 전제 조건 - 환자가 등록되어 있음, 기본 흐름 - 1) 의사가 환자 선택, 2) 약물 검색, 3) 용량 입력, 4) 시스템이 알레르기 확인, 5) 처방 확인, 대안 흐름 - 4a) 알레르기 경고 표시.
