## 핵심 개념

SDET는 주로 대규모 기업(Microsoft 등)에서 볼 수 있는 역할로, 자동화 테스트 코드 작성, 테스트 도구 개발, 개발 팀의 생산성을 높이는 도구 개발 등을 담당한다.

소프트웨어 개발의 기술적 역할 사다리:
1. **SDET/도구 개발자**: 테스트 코드와 도구 작성, 시작점으로 좋음
2. **개발자** (주니어/일반/시니어): 실제 제품 코드 작성
3. **테크 리드**: 개발팀 리딩, 기술 의사결정, 코드 리뷰
4. **아키텍트**: 시스템 설계, 기술 방향 결정
5. **디렉터/기술 펠로우**: 연구, 조직 리딩

SDET로 시작하면 테스트 프로세스를 이해하게 되어 더 나은 개발자가 될 수 있다는 장점이 있다.

## 동작 원리

대기업에서는 직함보다 **레벨(숫자)**이 더 중요하다. 레벨이 급여 범위를 결정하기 때문이다. 입사 시 가능한 높은 레벨을 확보하는 것이 장기적으로 유리하다.

```
대기업의 레벨 시스템 (예시):
Level 59-60: Software Development Engineer
Level 61-62: Software Development Engineer II
Level 63+: Senior Software Development Engineer
→ 레벨이 직함보다 중요 (급여 범위 결정)
```

## 예시

```
SDET의 일반적인 업무:
- Selenium을 사용한 웹 자동화 테스트 작성
- 성능 테스트 프레임워크 개발
- CI 파이프라인용 테스트 스크립트 작성
- 개발 팀 내부 도구 개발

경력 사다리:
SDET → Developer → Senior Developer → Tech Lead → Architect → Director
```
