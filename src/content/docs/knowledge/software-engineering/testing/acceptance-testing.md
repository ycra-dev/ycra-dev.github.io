## 핵심 개념

인수 테스트는 고객/사용자 테스트의 한 유형으로, 알파 테스트(개발 팀과 협력하여 초기 릴리즈 테스트), 베타 테스트(더 넓은 사용자 그룹에 릴리즈를 공개하여 테스트), 인수 테스트(계약적 인수 결정)로 구분된다. 인수 테스트의 6단계는 인수 기준 정의, 인수 테스트 계획, 인수 테스트 도출, 인수 테스트 실행, 테스트 결과 협상, 시스템 수락/거부이다. 애자일 방법론에서는 사용자가 개발 팀의 일원으로 참여하여 사용자 스토리 기반의 테스트가 인수 테스트 역할을 수행한다.

## 예시

인수 테스트 시나리오: 고객(보험 회사)이 자사의 실제 보험 청구 데이터를 새로 개발된 청구 처리 시스템에 입력하여, 기존 시스템과 동일한 처리 결과가 나오는지, 성능이 기준을 충족하는지 검증한 후 인수 여부를 결정한다.
