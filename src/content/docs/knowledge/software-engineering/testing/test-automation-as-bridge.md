## 핵심 개념

테스트 자동화는 QA에서 개발로 가는 가장 효과적인 브릿지다. 상사를 설득하기도 쉽다. "주니어 개발자가 되게 해달라"보다 "테스트 자동화를 맡겨달라"가 훨씬 수월한 제안이다.

이는 **윈-윈** 상황이다:
- 조직: 자동화된 테스트로 전체 효율화
- 본인: 실제 코딩 경험 획득

테스트 자동화 전문가는 채용이 어려운 니치 역할이라 수요가 높다.

## 동작 원리

"테스트 자동화 엔지니어" 또는 "Software Developer in Test(SDET)" 직함을 얻으면 일반 개발 역할로의 이직이 매우 쉬워진다.

다른 기술 직무에서도 유사한 브릿지가 존재한다:
- **Linux 관리자 → DevOps**: 스크립팅과 도구 개발을 통해 전환
- **기술 지원 → 개발**: 코드 분석을 포함하는 고급 지원 역할로 이동

## 예시

- 수동 테스터가 Selenium 기반 테스트 자동화를 배워 자동화 엔지니어로 전환
- Linux 관리자가 Ansible, Terraform 등 Infrastructure as Code를 익혀 DevOps로 전환
- 기술 지원 담당자가 API 디버깅과 코드 분석을 포함하는 고급 지원 역할로 이동
- "Software Developer in Test" 직함 획득 후 일반 개발자로 이직
