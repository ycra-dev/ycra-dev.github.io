## 핵심 개념

피싱(Phishing)은 은행, 정부기관 등 신뢰할 수 있는 기관을 사칭하여 이메일이나 가짜 웹사이트를 통해 사용자의 개인정보(비밀번호, 신용카드 번호 등)를 탈취하는 **사회공학적 공격**이다. "fishing(낚시)"에서 유래한 이름으로, 미끼를 던져 피해자를 유인하는 방식이다.

## 동작 원리

피싱은 기술적 취약점이 아닌 **인간의 심리적 취약점**을 이용하는 공격이다.

**일반적인 피싱 수법**:
1. 공격자가 은행, 쇼핑몰, IT 서비스를 사칭한 이메일을 대량 발송
2. "계정이 정지됩니다", "비밀번호를 변경하세요" 등 긴급한 메시지로 불안감 조성
3. 이메일 내 링크를 클릭하면 진짜와 거의 동일한 가짜 웹사이트로 이동
4. 사용자가 로그인 정보를 입력하면 공격자에게 전달

**스피어 피싱(Spear Phishing)**:
- 불특정 다수가 아닌 **특정 개인이나 조직**을 겨냥한 정교한 공격
- 대상의 이름, 직위, 업무 관계 등을 사전에 조사하여 맞춤형 메시지 작성
- 일반 피싱보다 성공률이 훨씬 높음
- 기업 임원을 겨냥하는 경우 "웨일링(whaling)"이라고도 함

**피싱 방어**:
- URL을 항상 직접 확인 (예: `amaz0n.com` vs `amazon.com`)
- HTTPS 사용 여부와 인증서 확인
- 이메일 내 링크 대신 직접 브라우저에 주소 입력
- 이중 인증(2FA) 활성화
- 의심스러운 요청에 대해 발신자에게 직접 확인

피싱은 데이터 유출 사건의 가장 흔한 시작점 중 하나이며, 기술적 보안이 아무리 강력해도 사용자 한 명의 실수로 무력화될 수 있다는 점에서 보안 교육의 중요성이 크다.

## 예시

일반 피싱 이메일 예시:
```
발신: security@bank-0f-america.com (가짜: O가 아닌 숫자 0)
제목: [긴급] 비정상 로그인 감지 - 즉시 확인 필요

고객님의 계좌에서 비정상적인 활동이 감지되었습니다.
24시간 내에 아래 링크에서 계정을 확인하지 않으면
계좌가 정지됩니다.

[계정 확인하기] → https://bank0famerica-security.com/verify
                  (실제 은행이 아닌 공격자의 사이트)
```

스피어 피싱 예시:
```
발신: ceo@company.com (위조된 발신자)
수신: 재무팀 김과장
제목: 긴급 송금 요청

김과장님, 오늘 중으로 아래 계좌로 3억원 송금 부탁드립니다.
해외 거래처 계약금입니다. 보안상 이메일로만 소통해주세요.

- 대표이사 드림
```
