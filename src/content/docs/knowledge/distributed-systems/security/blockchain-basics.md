## 핵심 개념

블록체인(Blockchain)은 모든 거래를 블록 단위로 기록하고, 각 블록이 이전 블록의 해시를 참조하여 체인을 형성하는 **탈중앙화된 공개 분산 원장(distributed ledger)**이다. 2008년 사토시 나카모토(가명)가 비트코인 논문에서 제안한 기술로, 은행이나 정부 같은 중앙 기관 없이 신뢰할 수 있는 거래를 가능하게 한다.

## 동작 원리

**구조**:
- **블록(Block)**: 일정 시간 동안의 거래 내역을 담은 데이터 묶음
- **체인(Chain)**: 각 블록이 이전 블록의 해시값을 포함하여 순서대로 연결
- 하나의 블록을 변조하면 이후 모든 블록의 해시가 달라지므로, 과거 기록의 위변조가 사실상 불가능

**핵심 기술 요소**:
1. **디지털 서명**: 각 거래는 발신자의 개인키로 서명되어 발신자를 인증
2. **암호학적 해시**: 블록 연결과 무결성 보장에 SHA-256 등을 사용
3. **합의 알고리즘**: 분산된 참여자들이 블록의 유효성에 합의하는 메커니즘
4. **P2P 네트워크**: 중앙 서버 없이 모든 참여자가 전체 원장의 사본을 보관

**비트코인에서의 작동**:
- 모든 거래가 공개적으로 기록됨 (하지만 실명이 아닌 주소로 기록)
- **채굴(Mining)**: 새 블록을 생성하기 위해 어려운 수학 문제를 풀어야 함 (작업 증명, Proof of Work)
- 채굴자에게 비트코인으로 보상 제공 → 네트워크 유지 인센티브

**탈중앙화의 의의**: 전통적으로 "이중 지불(double spending)" 문제를 해결하려면 은행 같은 신뢰할 수 있는 중개자가 필요했다. 블록체인은 암호학과 합의 알고리즘으로 이 문제를 중개자 없이 해결한다.

**한계와 비판**:
- 에너지 소비가 막대함 (비트코인 채굴)
- 거래 처리 속도가 느림 (비트코인: ~7건/초 vs Visa: ~24,000건/초)
- "익명"이지만 분석을 통해 실제 신원 추적 가능
- 가격 변동성이 극심하여 통화로서의 안정성 부족

## 예시

블록체인 구조:
```
블록 #0 (제네시스 블록)
┌─────────────────────┐
│ 이전 해시: 0000...  │
│ 거래: 최초 50 BTC   │
│ 해시: a1b2c3...     │
└─────────┬───────────┘
          │
블록 #1   ▼
┌─────────────────────┐
│ 이전 해시: a1b2c3...│  ← 블록 #0의 해시 참조
│ 거래:               │
│  A → B: 2.5 BTC     │
│  C → D: 1.0 BTC     │
│ 해시: d4e5f6...     │
└─────────┬───────────┘
          │
블록 #2   ▼
┌─────────────────────┐
│ 이전 해시: d4e5f6...│  ← 블록 #1의 해시 참조
│ 거래:               │
│  B → E: 0.5 BTC     │
│ 해시: g7h8i9...     │
└─────────────────────┘

→ 블록 #1을 변조하면 해시가 변경 → 블록 #2의 "이전 해시"와 불일치 → 감지
```

비트코인 거래 흐름:
```
1. Alice가 Bob에게 1 BTC 전송하려 함
2. Alice: 거래 내역을 자신의 개인키로 디지털 서명
3. 거래가 P2P 네트워크에 브로드캐스트
4. 채굴자들이 이 거래를 모아 새 블록에 포함
5. 채굴자가 작업 증명(Proof of Work) 완료 → 새 블록 생성
6. 새 블록이 네트워크에 전파 → 모든 참여자의 원장이 업데이트
7. Bob이 1 BTC 수신 확인
```
