## 핵심 개념

회사 내부에서만 열심히 해서는 "영원한 신입" 인식을 깨기 어렵다. 가족은 아무리 커도 여전히 어릴 때처럼 볼 수 있다. 해결책은 **외부에서 가치를 만드는 것**이다. 외부의 인정이 내부 인식을 바꾼다.

## 동작 원리

외부 브랜딩이 내부 가치를 높이는 메커니즘:

- 상사가 팟캐스트에서 당신의 인터뷰를 듣는다
- 당신이 쓴 기술 블로그 글이 업계에서 화제가 된다
- 오픈소스 프로젝트가 GitHub 스타를 많이 받는다

그러면 더 이상 "그냥 평범한 개발자"가 아니게 된다. 이것은 자연스럽게 "이 사람이 떠나면 회사가 손해"라는 인식을 형성한다.

**위협 vs 외부 브랜딩**

- 위협: 직접적, 적대적 ("연봉 올려줘, 안 그러면 이직할 게요")
- 외부 브랜딩: 간접적, 상사가 스스로 결론을 내리게 함

개발자 외부 브랜딩 채널:
- 기술 블로그 운영
- 지역/국제 컨퍼런스 발표
- 오픈소스 라이브러리 개발
- Stack Overflow 최상위 답변자
- YouTube 기술 채널

## 예시

- 기술 블로그: 자신이 해결한 기술 문제를 주 1회 포스팅
- 지역 개발자 모임 발표: "우리 팀이 마이크로서비스 전환한 경험"
- 오픈소스 라이브러리 개발 및 공개 → GitHub 스타 수가 외부 가치의 가시적 지표
