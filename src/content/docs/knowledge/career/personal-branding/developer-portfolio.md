## 핵심 개념

경험이 없는 개발자에게 포트폴리오는 가장 강력한 능력 증명 수단이다. 고용주가 개발자를 채용할 때 가장 큰 리스크는 **실제로 코딩할 줄 모르는 사람을 뽑는 것**인데, 포트폴리오는 이 리스크를 크게 줄여준다.

GitHub 프로필은 사실상 온라인 포트폴리오 역할을 하며, 다음이 평가된다:
- 어떤 프로젝트에 코드를 커밋했는지
- 얼마나 자주 커밋하는지
- 코드의 인기도 (스타, 포크)

최소 3-4개의 프로젝트를 포함해야 하며, 웹 서비스 호출, 데이터베이스 활용, 단위 테스트 포함 등으로 실력을 다각적으로 보여주는 것이 좋다.

## 동작 원리

지원하는 직무의 기술 스택에 맞는 프로젝트를 제작하라. 면접 시 노트북에 프로젝트를 로드해가서 코드와 설계 의도를 직접 설명할 수 있도록 준비하라.

## 예시

- GitHub에 3-4개의 완성된 프로젝트를 올리고 정기적으로 커밋
- 모바일 앱을 앱 스토어에 실제로 배포
- 면접 시 노트북에 프로젝트를 로드해가서 코드와 설계 의도를 직접 설명
- 웹 서비스 호출, DB 연동, 자동화 테스트를 포함한 프로젝트
