## 핵심 개념

기술 커뮤니티는 물리적·온라인 공간에서 정보를 공유하고, 전문적 관계를 발전시키며, 서로를 지원하는 사람들의 그룹이다. 커뮤니티에서 정보를 소비만 하는 것은 기생(parasite)과 같다. 기여자가 될 때 커리어가 크게 향상된다.

## 동작 원리

**기여할 방법은 매우 다양하다:**
- 기술 블로깅, 동영상 제작
- Q&A 사이트(Stack Overflow 등)에서 질문 답변
- 오픈소스 프로젝트: 문서화, 코드 리뷰, 번역, 이슈 중재, PR 리뷰
- 팟캐스트 지원, 컨퍼런스 자원봉사
- 로컬 밋업이나 스터디 그룹 운영

**기여의 효과:**
PowerShell 커뮤니티의 많은 리더들은 처음에 질문을 올리는 소비자였다가, 기여를 시작하면서 Microsoft 채용, 승진, 새로운 기회를 얻었다. 기여는 [Luck Surface Area](/knowledge/career/personal-branding/luck-surface-area/)를 넓히는 직접적인 방법이다.

**오픈소스 기여의 오해:**
코드 작성만이 기여가 아니다. 문서 업데이트, 유닛 테스트 작성, 이슈 중재, PR 리뷰, 번역 등도 동등하게 귀중한 기여다. 새로운 기여자에게는 이런 "코드 외 기여"가 오히려 진입 장벽이 낮아 시작하기 좋다.

## 예시

커뮤니티 기여 시작 로드맵:
```
Level 1 (소비자):
  - 블로그 글 읽기
  - 튜토리얼 따라하기
  - GitHub 프로젝트 Star

Level 2 (참여자):
  - Stack Overflow에서 아는 질문에 답변
  - 오픈소스 문서에서 오타 수정 PR
  - 로컬 밋업 참석

Level 3 (기여자):
  - 자신만의 기술 블로그 운영
  - 오픈소스 버그 수정 PR
  - 컨퍼런스 라이트닝 토크 발표

Level 4 (리더):
  - 오픈소스 프로젝트 메인테이너
  - 컨퍼런스 키노트 발표
  - 커뮤니티 주최
```
