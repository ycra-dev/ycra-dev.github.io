## 핵심 개념

Jason Roberts가 만든 개념. 세계 최고의 프로그래머라도 지하실에 혼자 앉아 아무도 모르게 일한다면 기회는 거의 찾아오지 않는다. **행운 표면적 = 능력(Doing) × 소통(Telling)**이다.

## 동작 원리

행운 표면적을 넓히는 방법:

```
행운 표면적 = Doing × Telling

Doing (능력 향상):
- 실제 기술 습득
- 프로젝트 완성
- 전문성 축적

Telling (소통/노출):
- 블로그 작성
- 팟캐스트 출연
- 컨퍼런스 발표
- 오픈소스 기여
- SNS 활동
```

**채널 확장 전략**

한 채널을 마스터한 후 체계화하여 매우 생산적으로 만들고, 그 다음 여러 채널로 확장한다.

노출 채널 목록:
- 개인 블로그, 게스트 포스팅, 책 출판, 잡지 기고
- 팟캐스트, YouTube, SNS
- 유저 그룹 발표, 컨퍼런스 발표
- 오픈소스 프로젝트

**반복 노출의 중요성**

4-5번 반복 노출이 되어야 사람들이 브랜드를 인식하기 시작한다.

## 예시

- 블로그 시작 → 팟캐스트 출연 → YouTube 채널 → 컨퍼런스 발표 순으로 채널을 확장
- 능력만 있고 노출 없음: Luck Surface Area = 작은 수 × 0 = 0
- 능력과 노출 모두 있음: 기회가 기하급수적으로 늘어남
