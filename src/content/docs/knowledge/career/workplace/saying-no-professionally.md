## 핵심 개념

Robert C. Martin의 비유: 환자가 "팔이 아프니 잘라주세요"라고 하면, 의사는 당연히 거절한다. 하지만 소프트웨어 개발자들은 비슷한 상황에서 "네, 코드를 절단하겠습니다"라고 답한다.

프로는 넘지 않는 선이 있다.

## 동작 원리

소프트웨어 개발자가 직면하는 윤리적 도전:
- 올바른 것과 편한 것 사이에서 선택해야 하는 순간
- 상사의 압력으로 품질 기준을 낮추라는 요구
- 불가능한 일정에 동의하라는 압박
- 기술적으로 잘못된 방향으로 가라는 지시

현실적 대응:
- "아니오"라고 말할 여유가 없는 상황이라면 일단 따르되, 다시는 그런 상황에 처하지 않도록 준비하라
- 저축으로 경제적 안전망을 만들어라
- 다른 직장을 알아보라

프로의 우선순위 설정:
- 프로는 스스로 작업의 우선순위를 정한다
- 아마추어는 항상 누군가에게 다음에 무엇을 해야 할지 묻는다
- Gold-plating(과잉 개발)은 우선순위를 못 정하는 증거다

## 예시

```
상황: PM이 테스트 없이 1주일 안에 기능을 출시하라고 요구

[아마추어 반응]
"네, 알겠습니다." → 버그 투성이 코드 배포 → 프로덕션 장애

[프로의 반응]
"1주일 안에 출시하려면 범위를 줄여야 합니다.
핵심 기능 A, B만 포함하면 테스트까지 1주일에 가능합니다.
기능 C, D는 다음 스프린트에서 추가하는 것이 어떨까요?

혹은 모든 기능을 원하신다면 2주가 필요합니다."

→ "아니오"만 하는 것이 아니라, 대안을 제시
→ 전문가로서의 의견을 분명히 하면서도 협력적
```
