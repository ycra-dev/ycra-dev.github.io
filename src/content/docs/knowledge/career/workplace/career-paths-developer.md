## 핵심 개념

소프트웨어 개발자는 커리어를 크게 세 가지 방향으로 선택할 수 있다. 너무 많은 초보 개발자가 이 선택을 의식적으로 하지 않고 표류한다. 커리어 경로를 명확히 선택하는 것 자체가 경쟁 우위가 된다.

## 동작 원리

**세 가지 커리어 유형**

| 유형 | 설명 | 특징 |
|------|------|------|
| 커리어 개발자 | 회사에 소속되어 정규 급여 | 안정성 높음, 상한선 존재 |
| 프리랜서 | 독립적으로 여러 클라이언트 | 자유, 더 높은 상한, 불확실성 |
| Entreprogrammer | 자신의 제품/서비스 판매 | 무제한 상한, 큰 위험 부담 |

**기술 전문화 경로**

- 웹 개발: 프론트엔드 / 백엔드 / 풀스택 (가장 큰 영역)
- 모바일: iOS / Android
- 데이터 사이언스: 가장 빠르게 성장, 높은 보수
- 임베디드 시스템: 높은 수요, 전문 기술 필요
- 클라우드: AWS / Azure / GCP
- 테스트 자동화: 심각한 인력 부족, 높은 수요
- 게임 개발, 데스크톱 애플리케이션, 도구/엔터프라이즈 등

**선택 시 고려사항**

의식적으로 경로를 선택하고, 그 경로에 맞는 전문성과 브랜드를 구축한다.

## 예시

- 웹 개발자 → "React 성능 최적화 전문가"로 특화
- 임베디드 시스템 개발자 → 높은 전문성으로 경쟁자 적고 수요 높은 포지션
- Entreprogrammer → 블로그/YouTube 오디언스 구축 후 온라인 강좌 판매
