## 핵심 개념

논쟁에서 이기는 유일한 방법은 논쟁 자체를 피하는 것이다. Dale Carnegie: "논쟁을 피하라. 방울뱀과 지진을 피하듯이."

소프트웨어 개발자는 모든 사람이 논리적으로 사고한다고 착각하기 쉽다. 하지만 인간은 본질적으로 감정적인 존재다.

## 동작 원리

논쟁이 비효과적인 이유:
- 논리적 반박은 울고 있는 아이에게 잠을 자야 한다고 설명하는 것만큼이나 효과가 없다
- 상대방이 감정적으로 상처를 받으면, 아무리 좋은 논리도 받아들이지 않는다
- 논쟁에서 이겨도 상대방의 마음은 잃는다

대신 해야 할 것:
1. 이 문제가 정말 중요한 전투인지 먼저 판단한다 ("hill worth dying on")
2. 자신에게는 작은 문제이지만 상대에게 큰 문제라면 기꺼이 양보한다
3. 양보할 때마다 미래에 사용할 수 있는 신뢰 자산이 쌓인다
4. 정말 중요한 문제에서는 쌓아둔 신뢰를 활용한다

## 예시

```
상황: 코드 포매팅 스타일에 대한 의견 충돌

[논쟁하는 방식]
"탭 대신 스페이스를 써야 합니다. 이유는..."
→ 30분 논쟁 → 서로 기분 상함 → 아무것도 해결 안됨

[양보하는 방식]
"그 방식도 괜찮네요. 팀 컨벤션에 맞추겠습니다."
→ 상대방이 감사함 → 다음에 정말 중요한 아키텍처 결정에서
  당신의 의견을 더 존중해줌
```

핵심: 작은 전투에서 양보하여 큰 전투에서 이겨라.
