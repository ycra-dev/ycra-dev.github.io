## 핵심 개념

사람의 가장 깊고 절박한 욕구 중 하나는 자신이 중요하다고 느끼는 것이다. 이것은 사회와 삶에서 대부분의 위대한 성취의 주된 동기이기도 하다.

이 원칙을 위반하면: 상대방의 아이디어를 무시하면 그는 당신의 아이디어에도 귀를 닫는다.

## 동작 원리

이 원칙을 활용하는 방법:
- 상대방의 아이디어를 먼저 인정하면, 당신의 아이디어도 가치 있게 받아들여진다
- 작은 것에서 양보하면, 큰 것에서 신뢰를 얻는다
- 경청하고 관심을 보이면, 강력한 유대감이 형성된다

이 원칙을 위반하면:
- 동료의 아이디어를 무시하고 자신의 아이디어를 내놓으면, 상대방은 당신의 아이디어에 귀를 닫는다
- 상대방을 작게 만들면, 산소가 차단된 사람처럼 격렬하게 반응한다

## 예시

코드 리뷰 상황:
```
[잘못된 방식]
"이 코드는 비효율적이야. 왜 이렇게 했어?"
→ 상대방의 노력을 무시하고 무능하다고 암시

[올바른 방식]
"이 부분 접근 방식이 흥미롭네요. 여기서 한 가지 생각이 있는데,
캐싱을 추가하면 성능이 더 좋아질 수 있을 것 같아요.
어떻게 생각하세요?"
→ 상대방의 작업을 인정하면서 개선안 제안
```

팀 미팅에서:
- 다른 사람이 말할 때 끝까지 경청한다
- "좋은 지적이네요"로 시작한 후 자신의 의견을 덧붙인다
- 팀원의 기여를 공개적으로 인정한다
