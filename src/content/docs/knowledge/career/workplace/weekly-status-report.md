## 핵심 개념

"눈에 보이지 않으면 존재하지 않는다." 아무리 열심히 일해도 상사가 모른다면 인사 평가에 반영되지 않는다. 주간 상태 보고서는 자신의 기여를 지속적으로 문서화하고 상사와 신뢰를 쌓는 방법이다.

## 동작 원리

주간 보고서 템플릿:

```
제목: 주간 업무 보고 - [이름] - [날짜]

이번 주 완료:
- [구체적 성과 1] (비즈니스 임팩트 포함)
- [구체적 성과 2]

다음 주 계획:
- [예정 작업 1]
- [예정 작업 2]

블로커/도움 필요:
- [있는 경우]

지표:
- PR 제출 수: X개
- 버그 수정: X개
- 코드 리뷰 참여: X개
```

**이중 효과:**
1. 즉각: 상사가 자신의 기여를 인식함
2. 장기: 평가 시즌에 "제 1년 성과를 정리해왔습니다" → 52주 보고서 → 반박 불가

## 예시

- 매주 금요일 오후 상사에게 이메일 또는 Slack 메시지
- "이번 주: 결제 API 성능 최적화 완료, 응답시간 40% 단축 (고객 만족도 향상 기대)"
- 평가 면담 시: "지난 1년간 주간 보고서를 모아두었습니다. 총 30개 기능 개발, 15개 버그 수정..."
