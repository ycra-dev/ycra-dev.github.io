## 핵심 개념

문화는 하루아침에 만들어지지 않는다. 초기 팀 멤버의 가치관에서 시작하여 매일의 결정, 이야기, 습관으로 진화한다. 모든 엔지니어가 형성에 기여한다.

Edmond Lau는 500명 이상의 엔지니어 인터뷰에서 반복적으로 질문했다: "전 회사의 엔지니어링 문화에서 좋았던 점과 싫었던 점 하나씩?"

## 동작 원리

최고의 엔지니어들이 찾는 문화 10가지:
1. **반복 속도 최적화** — 빠른 피드백 루프와 배포 사이클
2. **자동화를 끊임없이 추진** — 수동 작업을 시스템으로 대체
3. **올바른 소프트웨어 추상화 구축** — 복잡성을 관리하는 설계
4. **코드 리뷰로 높은 코드 품질 유지** — 버그 방지와 지식 공유
5. **존중하는 업무 환경** — 심리적 안전
6. **공유 코드 소유권** — 버스 팩터 관리
7. **자동화 테스트 투자** — 회귀 방지와 리팩토링 자신감
8. **20% 시간 또는 해커톤으로 실험 시간 확보** — 혁신의 여지
9. **학습과 지속적 개선 문화** — 성장 마인드셋
10. **최고의 인재 채용** — 문화 강화 선순환

**문화의 이점:**
- 엔지니어에게 권한 부여 → 더 행복하고 생산적
- 공유 의사결정 컨텍스트 → 더 빠른 적응
- 강한 문화 → 인재 유치 → 문화 강화 (선순환)

## 예시

```
# 좋은 엔지니어링 문화의 실천 예시
culture_practices = {
    "Google": "코드 리뷰 필수, 언어별 가독성 리뷰, 20% 타임, EngEDU",
    "Facebook": "Bootcamp 6주 온보딩, Move Fast, 강한 채용 문화",
    "Instagram": "Do the simple thing first, 검증된 기술 사용",
    "Etsy": "A/B 테스트 문화, 지속적 배포, 메트릭 기반",
    "Quora": "코드랩, 멘토링 프로그램, 포스트모템 문화"
}

# 문화 구축에 기여하는 방법 (매니저가 아니어도)
individual_actions = [
    "좋은 코드를 작성하여 모범 보이기",
    "코드 리뷰에서 건설적 피드백 제공",
    "온보딩 자료 개선",
    "테크 토크 발표",
    "포스트모템 주도",
    "도구 구축 및 공유"
]
```

Marc Hedlund(Etsy/Stripe): "팀 전체를 더 낫게 만들면 Staff Engineer, 회사 전체를 더 낫게 만들면 Principal Engineer, 산업 전체를 더 낫게 만들면 Distinguished."
