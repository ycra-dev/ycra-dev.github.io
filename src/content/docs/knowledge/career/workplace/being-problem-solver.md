## 핵심 개념

모든 조직에는 "이 아이디어가 왜 안 되는지" 설명하는 사람이 넘쳐난다. 그런 사람은 흔하다. 희소한 것은 **장애물을 만나도 해결책을 찾아내는 사람**이다.

## 동작 원리

문제 해결자의 특성:
- 문제를 만나면 불가능하다고 말하지 않고 대안을 찾는다
- 어려운 문제를 회피하지 않고 정면으로 맞선다
- 다른 사람이 포기한 문제를 해결한다
- 아이디어만 내는 것이 아니라 결과를 만들어낸다

정치적 전략보다 나은 이유:
- 정치적 기동으로 올라가면, 같은 방식으로 쉽게 떨어질 수 있다
- 실질적 가치에 기반한 승진은 견고한 기반 위에 선다
- 문제 해결 능력은 어떤 회사, 어떤 역할에서든 가치 있다

지속적 교육의 중요성: 기술 교육뿐 아니라 리더십, 경영, 비즈니스도 배워라. 배운 것을 공유하라 (블로그, 발표, 기사 등). 외부 노출은 내부에서의 가치도 높인다.

## 예시

```
상황: 레거시 시스템이 주기적으로 다운됨

[회피하는 사람]
"이건 코드가 너무 오래돼서 건드리면 안 됩니다.
새로 만들 때까지 기다려야 해요."

[해결하는 사람]
"로그를 분석해보니 메모리 누수가 원인입니다.
임시 해결책으로 자동 재시작을 설정하고,
이번 스프린트에서 해당 모듈의 메모리 관리를
리팩토링하겠습니다. 2주면 됩니다."

→ "이 사람에게 맡기면 된다"는 신뢰를 얻음
```
