## 핵심 개념

대기업은 최고의 기술 인재를 유지하기 위해 두 가지 별도 승진 경로를 제공한다. 핵심 질문은 "남은 커리어 동안 코드를 작성하지 않아도 괜찮은가?"이다.

## 동작 원리

**관리 트랙 (Management Track)**

```
Developer Manager
    → Section Manager
    → Director of Development
    → CTO
```

- 시간이 지날수록 코딩에서 멀어진다
- 주요 업무: 사람 관리, 예산, 전략, 조직 정치
- 개발자가 좋아하는 기술적 문제 해결이 줄어든다

**기술 트랙 (Technical Track)**

```
Senior Developer
    → Architect / Principal Engineer
    → Distinguished Engineer / Fellow
    → Senior Fellow / VP of Technology
```

- 계속 기술적 작업을 할 수 있다
- 주요 업무: 기술 방향 결정, 복잡한 기술 문제 해결, 아키텍처 설계
- 중소기업에는 이 트랙 자체가 없을 수 있다

**선택 기준**

- 사람 관리와 조직 정치를 즐기는가 → 관리 트랙
- 기술적 문제 해결을 즐기는가 → 기술 트랙
- 대부분의 개발자는 기술적 문제 해결을 좋아해서 이 직업을 택했으므로, 관리 트랙이 생각보다 맞지 않을 수 있다

## 예시

- 관리 트랙의 현실: 회의, 예산 협의, 채용/해고, 성과 관리가 대부분의 시간을 차지
- 기술 트랙의 가치: Google의 Distinguished Engineer는 관리직과 동등한 대우 받음
- 중소기업: 기술 트랙이 없어서 관리직만이 유일한 승진 경로일 수 있음
