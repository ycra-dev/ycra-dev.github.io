## 핵심 개념

Problem-Context-Solution(PCS) 구조는 비즈니스 커뮤니케이션에서 가장 효과적인 메시지 전달 방식이다. 이메일, 슬랙 메시지, 제안서, 장애 보고서 등 모든 형태의 비즈니스 문서에 적용 가능하다.

## 동작 원리

**4단계 구조:**

1. **문제를 제시**한다: 청자의 주의를 즉시 끈다. 사람은 자신에게 관련된 문제를 들으면 자연스럽게 집중한다.
2. **맥락(배경 정보)을 제공**한다: 이 문제를 이해하기 위한 최소한의 배경 정보만 제공. 과도한 맥락은 곁가지가 되어 핵심 메시지를 희석시킨다.
3. **맥락과 문제를 연결**한다: "이 배경 정보가 왜 이 문제와 관련이 있는가?"를 명시적으로 설명한다. 이 단계가 종종 간과되지만 매우 중요하다.
4. **해결책을 제안**한다: 앞서 제시한 문제에 직접적으로 대응해야 하며, 제공한 맥락에 의해 뒷받침되어야 한다.

이 구조를 따르면 청자가 자연스럽게 "그래, 이 해결책이 맞다"는 결론에 도달한다.

## 예시

```
# 실제 PCS 구조 적용 예시

문제: "배포 후 롤백이 월 평균 5회 발생하고 있습니다."
맥락: "현재 CI/CD 파이프라인에 통합 테스트가 없으며, 코드 리뷰도 선택 사항입니다."
연결: "통합 테스트와 필수 코드 리뷰가 없기 때문에, 결함이 프로덕션까지 도달하고 있습니다."
해결책: "통합 테스트 스위트를 추가하고, 코드 리뷰를 머지 필수 조건으로 설정하겠습니다."
```

- **배경 설명 먼저 시작할 경우**: "우리 시스템은 2018년에 설계되어..."로 시작하면, "이게 나와 무슨 관련이지?"라는 생각에 주의가 흩어진다
- **문제 먼저 시작할 경우**: "우리 시스템이 하루에 3번 다운됩니다"로 시작하면, 모든 이해관계자가 즉시 귀를 기울인다
