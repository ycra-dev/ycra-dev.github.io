## 핵심 개념

열심히 일하는 것만으로는 충분하지 않다. 당신의 노력과 성과를 알릴 방법이 필요하다.

팀에서 가장 뛰어나고 열심히 일하는 개발자라도 아무도 모르면 소용없다.

## 동작 원리

**주간 보고서(Weekly Report) 전략:**
- 매일 자신의 시간 사용과 성과를 기록한다
- 매주 금요일에 주간 요약을 매니저에게 보낸다
- 첫 보고서에는 "매니저님의 업무를 돕기 위해" 보내는 것임을 설명한다
- 효과: 매주 매니저의 레이더에 나타나며, 자랑 없이도 성과를 알릴 수 있다
- 부가 효과: 연말 성과 리뷰 시 정확한 성과 기록으로 활용

**기타 가시성 확보 방법:**
1. **프레젠테이션 제공**: 팀이 직면한 문제에 대해 발표 제안 (점심시간 활용 가능)
2. **미팅에서 발언하기**: 존재감을 드러내는 가장 쉬운 방법
3. **정기적 1:1 미팅 설정**: 상사와의 정기적 접촉 유지
4. **지식 공유**: 블로그, 사내 위키, 커뮤니티 발표 등

발표의 이중 효과: 가시성 확보 + 해당 주제에 대한 강제 학습

## 예시

주간 보고서 형식:
```
제목: [주간 보고] 2026년 2월 4주차 - 홍길동

## 이번 주 성과
- 결제 모듈 리팩토링 완료 (응답시간 40% 개선)
- 신규 팀원 온보딩 지원
- 프로덕션 배포 2건 (무중단)

## 다음 주 계획
- 사용자 대시보드 API 설계 착수

## 이슈/블로커
- 테스트 서버 디스크 용량 부족 (인프라팀에 요청 중)
```

이것만으로도 다른 개발자보다 훨씬 생산적으로 보이게 된다. 매니저는 당신의 업무만 듣고, 다른 개발자의 업무는 듣지 못하기 때문이다.
