## 핵심 개념

저자는 주당 8시간을 독서에 투자하며, 연간 50권 이상을 읽는다. 실제 멘토가 없었기에 책이 그 역할을 대신했다. 책은 가장 저렴하고 접근하기 쉬운 멘토십 형태이다.

## 동작 원리

효과적인 독서 습관 구축:

**책 선택**
- 높은 평가를 받은 양질의 책만 읽는다
- 다음에 읽을 책 목록을 항상 유지한다
- 이미 많이 알고 있는 주제보다 새로운 영역의 책을 포함한다

**시간 확보**
- 런닝, 운전, 운동 시 Audible 오디오북 활용
- 트레드밀 위에서 기술 서적 읽기 → 운동과 학습 동시에

**읽은 것을 실행에 옮기기**
행동 없는 지식은 무가치하다. 읽은 내용 중 즉시 적용할 수 있는 것을 하나라도 실행한다.

**개발자가 읽어야 할 분야**

| 분야 | 추천 도서 |
|------|----------|
| 좋은 코드 작성 | Code Complete, Clean Code |
| 필수 CS 지식 | SICP, Design Patterns |
| 레거시 코드 | Refactoring, Working Effectively with Legacy Code |
| 자기계발 | How to Win Friends and Influence People |
| 동기부여 | The 10x Rule, The War of Art |

## 예시

- 매일 런닝 중 오디오북 청취: 주 8시간 = 연 50권 이상
- "How to Win Friends & Influence People": 연 1-2회 재독 → 매번 새로운 인사이트
- 책을 읽고 즉시 실천 항목 1가지 작성 후 다음 주 실행
