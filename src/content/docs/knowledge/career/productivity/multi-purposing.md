## 핵심 개념

멀티태스킹(동시에 여러 일 하기)은 비생산적이지만, 멀티퍼포징(하나의 행위에서 여러 가치 추출)은 극도로 효율적이다. 같은 시간과 노력으로 더 많은 가치를 만드는 전략이다.

## 동작 원리

하나의 사이드 프로젝트가 동시에 달성할 수 있는 목적들:

```
하나의 사이드 프로젝트
    ├── 새 프로그래밍 언어 학습
    ├── 기존 기술 연마
    ├── 블로그 콘텐츠 시리즈 생성
    ├── 면접 포트폴리오 작품
    ├── 수입원 창출 (앱스토어 등)
    ├── 자기 규율 훈련
    └── 새로운 산업 학습
```

**실제 예시: 책 한 권**

저자의 "The Complete Software Developer's Career Guide":
1. 매일 1,000 단어 쓰기 습관 유지
2. 책 완성
3. 판매 수익
4. 블로그 콘텐츠 (각 챕터를 블로그 포스트로 먼저 발행)
5. 이메일 구독자 증가

나아가 eBook, 인쇄본, 오디오북, 영상 버전, YouTube 콘텐츠 등으로 확장 가능하다.

## 예시

- 이 책: 글쓰기 습관 + 책 + 블로그 콘텐츠 + 이메일 구독자 + 판매 수익 = 5가지 목적
- 사이드 프로젝트: React 학습 + 포트폴리오 작품 + 블로그 시리즈 + 앱스토어 수입
- 트레드밀에서 오디오북 듣기: 운동 + 학습 = 2가지 목적
