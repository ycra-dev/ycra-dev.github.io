## 핵심 개념

반복 속도(Iteration Speed)는 아이디어를 구현하고 결과를 확인하는 사이클의 빠르기다. 학습이 복리로 작용하므로, 반복 속도를 가속화할수록 학습률도 가속화된다.

Facebook의 핵심 문화: "MOVE FAST AND BREAK THINGS." Mark Zuckerberg의 IPO 서한: "빠르게 움직이면 더 많이 만들고 더 빨리 배울 수 있다. 아무것도 깨뜨리지 않는다면 충분히 빠르게 움직이지 않는 것이다."

## 동작 원리

**반복 속도를 높이는 4가지 전략:**

1. **시간 절약 도구 투자**: 도구는 시간 곱셈기(multiplier). Bobby Johnson(전 Facebook): "성공적인 사람은 도구를 많이 만든다"
2. **디버깅/검증 루프 단축**: 최소 재현 테스트 케이스, 커스텀 워크플로우 구축
3. **프로그래밍 환경 마스터**: 키보드 숏컷, 텍스트 에디터 숙달, UNIX 명령어
4. **비-엔지니어링 병목 해소**: 의사결정자 승인, 팀 간 의존성, 리뷰 프로세스

**도구의 복합 효과**: 20분 작업을 10분으로 줄이면 절약은 10분이 아니라 그 이상이다. 더 자주 사용하게 되고 새로운 워크플로우가 가능해지기 때문이다.

```
# 컴파일 시간의 영향
20분 컴파일 → 하루 2-3회 컴파일, 큰 코드 덩어리를 배치
2분 컴파일  → 하루 수십 회 컴파일, 작은 단위로 반복
수초 컴파일  → 증분 컴파일, 파일 저장 시 자동 검사
```

## 예시

**Sarah vs Mark 비유:**
- Mark: 바로 기능 개발 시작 → 2개월간 꾸준히 작업
- Sarah: 처음 2주 워크플로우 개선 (33% 속도 향상) → 2개월 후 Mark를 따라잡고, 이후 영구적으로 33% 빠름

**팀 확장 효과:**
- 1시간/일 절약 도구 × 10명 팀 = 10시간/일 절약
- Google: 빌드 시간 1분 단축 × 1000 엔지니어 × 12회/일 = 주당 ~1인년 절약
