## 핵심 개념

멀티태스킹은 존재하지 않는다. 컴퓨터의 단일 프로세서처럼 인간도 한 번에 한 가지만 할 수 있다. 실제로 일어나는 것은 태스크 스위칭(작업 전환)이며, 전환할 때마다 오버헤드가 발생한다.

연구에 따르면 작업 전환 후 원래 작업에 다시 깊이 집중하기까지 평균 **23분**이 걸린다. 슬랙 알림 하나에 반응하는 것만으로도 거의 30분의 생산성을 잃을 수 있다.

## 동작 원리

두 가지 작업 모드를 의식적으로 구분하여 사용한다:

**싱글태스크 모드:**
- 하나의 작업에만 깊이 집중하는 시간
- 알림을 끄고, 이메일/메신저를 닫고, 오직 한 가지 작업만
- 코딩, 설계, 문서 작성 같은 깊은 사고가 필요한 작업에 적합

**태스크 스위칭 모드:**
- 의도적으로 여러 가벼운 작업을 전환하며 처리하는 시간
- 이메일 확인, 코드 리뷰 피드백 응답, 미팅 등
- 이 모드는 집중적인 작업이 아닌 처리 가능한 작업들에 사용

두 모드를 혼합하지 않는 것이 핵심이다. "잠깐만" 이메일을 확인하는 것은 싱글태스크 모드에서 강제로 태스크 스위칭을 발생시키며, 두 모드의 장점을 모두 잃게 된다.

## 예시

- **나쁜 예**: 복잡한 알고리즘을 구현하는 중에 슬랙 알림이 울려 확인하고 다시 코딩으로 돌아왔을 때 "어디까지 했지?"라며 코드를 다시 읽어야 하는 상황
- **좋은 예**: 오전 9-11시는 슬랙 알림을 끄고 코딩에만 집중(싱글태스크 모드), 11시부터 점심까지는 코드 리뷰와 이메일 처리(태스크 스위칭 모드)
- **실용적 팁**: 컴퓨터 프로세서가 프로세스 간 전환할 때 레지스터 상태를 저장/복원하는 것처럼, 개발자도 작업 전환 시 "현재 상태 노트"를 작성해 두면 복귀 비용을 줄일 수 있다
