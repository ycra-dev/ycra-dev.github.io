## 핵심 개념

사이드 프로젝트는 소프트웨어 개발 커리어를 발전시키는 최고의 방법 중 하나이다. 6개월 사이드 프로젝트 하나가 9-to-5 직장에서 수 년 걸릴 경험을 줄 수 있다. 각 사이드 프로젝트는 작은 복권과 같다 -- 경제적 성공의 가능성이 있다.

## 동작 원리

효과적인 사이드 프로젝트를 위한 4가지 원칙:

**1. 작게 시작하라**
처음에는 기존 앱 클론처럼 쉬운 것에서 시작한다. 완성 경험이 중요하다.

**2. Multi-Purposing 설계**
하나의 프로젝트가 최소 두 가지 이상의 목적을 동시에 달성하도록 설계한다:
- 새 언어 학습 + 블로그 콘텐츠 + 포트폴리오 + 수입원

**3. 명확한 완료 기준 정의**
세 가지를 반드시 정의한다:
- 완료 기준 (scope)
- 마감일 (deadline)
- 작업 일정 (schedule)

**4. 수익화 방법 생각하기**
처음부터 수익화를 생각한다. 99센트라도. 수익화를 생각하면 더 진지하게 임하게 된다.

**99% 완성 = 0% 가치**

프로젝트가 완성되지 않은 것은 0%의 가치를 갖는다.

## 예시

- 저자의 "PaceMaker" 앱: 첫 완성 사이드 프로젝트 → 모든 것을 바꾼 전환점
- Multi-purposing: React 학습 + 블로그 포스트 시리즈 + 포트폴리오 + 앱스토어 수입
- 작게 시작: 할 일 목록 앱 → 나중에 기능 추가
