## 핵심 개념

우선순위화는 제한된 시간을 가장 높은 레버리지 활동에 지속적으로 재배분하는 프로세스다. 역설적이지만 이것이 나머지 시간의 레버리지를 결정하는 "가장 높은 레버리지 활동"이다.

인간의 작업 기억은 7±2 항목으로 제한된다(Miller's Law). 뇌를 저장소가 아닌 프로세서로 사용하기 위해, 모든 할 일을 하나의 접근 가능한 리스트에 기록해야 한다.

## 동작 원리

**효과적인 우선순위화의 5가지 전략:**

1. **단일 마스터 리스트**: 모든 할 일을 하나의 리스트에 기록. 뇌를 저장소로 사용하지 않는다.

2. **쌍별 비교(Pairwise Comparison)**: 전체 순위를 매기려 하지 말고, "지금 하는 것보다 더 높은 레버리지인 것이 있는가?"를 반복적으로 질문한다.

3. **두 가지 휴리스틱**:
   - 직접적으로 가치를 생산하는 것에 집중 (활동 ≠ 생산성)
   - 중요하지만 긴급하지 않은 것에 집중 (Eisenhower Matrix의 2사분면)

4. **WIP(Work in Progress) 제한**: 동시 진행 프로젝트를 제한하여 컨텍스트 스위칭을 최소화한다. "작업 부하를 선형적으로 증가시키면 실패 가능성은 기하급수적으로 증가한다."

5. **If-Then 계획**: "3시 회의 후라면, 이 오래된 버그를 조사하겠다" 같은 사전 계획. Gollwitzer 연구에서 if-then 계획 수립 시 완수율이 71% vs 32%로 높았다.

## 예시

```
# 주간 우선순위 워크플로우
월요일 아침:  주간 우선순위 설정 (backlog에서 "This Week"로 이동)
매일 아침:    "This Week" → "Today"로 이동
작업 시:      "Today" → "Doing"으로 이동

# 일간 원칙
아침:         인지 에너지가 높을 때 우선순위 결정 + 창의적 작업
20분 여유:    코드 리뷰/이메일 등 단시간 태스크
회의:         연속 배치하여 집중 시간 보호
```
