## 핵심 개념

미루기(procrastination)의 주요 원인 중 하나는 "문제 감상"(problem admiration)이다. 문제의 크기에 압도되어 해결하려는 시도조차 하지 않는 것이다.

큰 작업이 문제인 이유:
- 심리적으로 위압감을 준다
- 정의가 불명확하다
- 오류 가능성이 높다
- 추정이 거의 불가능하다

## 동작 원리

분해 방법:
1. 작업을 순차적인 단계로 나눈다
2. 각 단계에 명확한 목표를 설정한다
3. 각 단계가 30분~2시간 내에 완료 가능한 크기인지 확인한다
4. 분해가 어렵다면 정보가 부족하다는 신호다 → 먼저 조사가 필요

프로그래밍에도 동일하게 적용된다. 아무리 복잡한 알고리즘이라도 코드 한 줄 수준까지 분해할 수 있고, 한 줄의 코드는 어떤 프로그래머에게도 불가능하지 않다.

분해의 부가 효과: 프로젝트 초기에 분해 어려움을 발견하면, 나중에 발견하는 것보다 훨씬 적은 비용으로 문제를 해결할 수 있다.

## 예시

```
큰 작업: "CI/CD 파이프라인 구축"  (위압적)

분해:
1. 커맨드라인에서 프로젝트 빌드하기       (2시간)
2. 빌드 서버에서 코드 체크아웃 설정하기   (1시간)
3. 1번과 2번 통합하기                     (1시간)
4. 테스트 실행 단계 추가하기              (1시간)
5. 배포 단계 추가하기                     (2시간)
```

- 이 책(Soft Skills): 거대한 책 1권이 아니라 1,000~2,000단어의 80여 개 짧은 챕터로 분해
- Agile 팀: 고객 요구사항을 작은 단위로 분해하면 고객도 원하는 것을 더 잘 설명하게 됨
