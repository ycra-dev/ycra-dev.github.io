## 핵심 개념

기술은 빠르게 변한다. 어제 최신이던 것이 오늘 구식이 된다. 20년간 같은 회사에서 일하며 기술 투자를 소홀히 한 개발자들이 갑작스런 해고 후 20년 뒤처진 기술로 구직 시장에 내몰리는 사례가 있다. 고용 안정을 추구하는 대신, 자신의 능력을 끊임없이 최신 상태로 유지하는 것이 진정한 안정이다.

## 동작 원리

의도적 기술 유지의 세 가지 원칙:

**1. Write Code Every Day**

John Resig(jQuery 창시자)의 원칙. 매일 또는 매주 실제로 코드를 작성하는 시간을 확보한다. 단순히 읽거나 강의를 보는 것으로는 부족하다.

**2. 사이드 프로젝트로 새 기술 적용**

새로운 기술을 배우면서 코딩 실력도 유지할 수 있는 가장 효율적인 방법이다.

**3. 반드시 완료한다**

프로젝트는 반드시 완료해야 하며, "[끝내는 사람(Finisher)](/knowledge/career/productivity/finisher-mindset/)"이 되는 것이 핵심이다. 아무리 안정적인 직장이라도 너무 편안해지지 말아야 한다.

## 예시

- John Resig의 "Write Code Every Day" 원칙 실천 → 매일 최소 30분의 코딩
- 새로운 기술/언어를 배우면서 작은 프로젝트를 만들어 완성하기
- 매일 30분 기술 서적 읽기를 트레드밀 위에서 실천 (월 1-2권 완독)
- 안정적인 직장에서도 "내일 해고된다면?"을 상상하며 기술 투자 유지
