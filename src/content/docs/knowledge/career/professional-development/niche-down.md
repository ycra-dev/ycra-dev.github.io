## 핵심 개념

"C# 개발자"가 아니라 "C#의 특정 프레임워크 전문가"가 되어야 한다. 가능한 한 작고 구체적인 전문 분야를 선택하면 두각을 나타내기 쉽다. 나중에 항상 확장할 수 있지만, 처음부터 넓으면 경쟁에서 눈에 띄기 어렵다.

## 동작 원리

**니치 다운의 이중 이점**

현재 직장에서 사용하는 기술을 전문화하면:
1. 낮 시간에 실무를 통해 전문성을 쌓는다 (무료로)
2. 퇴근 후 블로그나 부업을 통해 명성을 쌓는다

**관심 분야가 현재 업무와 다를 때**

1. 아침/저녁 시간에 원하는 분야의 명성을 먼저 쌓는다
2. 그 명성을 기반으로 해당 분야의 직장으로 이직한다

**학습 대상 선택 기준**

- 전문 분야에 깊이 집중된 것
- 또는 시대를 초월할 만큼 넓게 적용 가능한 것 (클린 코드, 알고리즘 등)
- 둘 중 하나여야 한다

## 예시

블로그 주제 선택:

| 너무 넓음 | 적절한 니치 |
|-----------|-------------|
| "Android 개발" | "Android ListView 컨트롤 최적화" |
| "Python 개발" | "Python 컴퓨터 비전 (OpenCV)" |
| "클라우드 개발" | "AWS Lambda + Python 서버리스" |

- Adrian Rosebrock: Python 컴퓨터 비전의 특정 라이브러리만 전문화하여 큰 성공
- 현재 React를 쓰는 직장에서 "React 성능 최적화 전문가"로 브랜딩
