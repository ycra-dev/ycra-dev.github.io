## 핵심 개념

"칼로리를 줄이고 운동을 더 하겠다"는 것은 계획이 아니다. 진짜 계획은 구체적이어야 한다. 학습 계획은 막연한 의지가 아니라, 무엇을 언제 어떻게 배울지를 명시한 로드맵이다.

## 동작 원리

학습 계획에 포함할 요소:

- **매일 루틴**: 매일 아침 30분 프로그래밍 블로그 읽기, 항상 기술 서적 한 권을 진행 중이기
- **학습 목록**: 다음에 배울 기술들을 우선순위와 함께 정리 (예: React → Docker → Kubernetes)
- **실습 계획**: 사이드 프로젝트를 통한 실제 적용
- **이벤트 참석**: 컨퍼런스, 밋업 등 커뮤니티 참여

**전략적 학습 선택**

학습할 내용을 선택할 때:
- **즉각적 수익**: 현재 직무에 직접 관련된 기술 → 빠른 업무 적용
- **장기적 가치**: 시대를 초월한 코드 품질/설계 원칙 → 평생 유효

사용하지 않을 기술을 무목적으로 배우는 것은 시간 낭비다.

**뉴스 소비 제한**

Hacker News, Reddit 같은 기술 뉴스는 timeboxing하여 읽는다. 중독성 있는 정보 소비는 실제 학습을 방해한다.

## 예시

```
나의 학습 계획 (2026 Q1)
- 매일 아침 6-6:30: 기술 블로그 읽기 (RSS 리더 활용)
- 매일 저녁 9-9:30: "Designing Data-Intensive Applications" 읽기
- 사이드 프로젝트: Go 언어로 간단한 CLI 툴 만들기
- 다음 배울 기술 목록: TypeScript → Kubernetes → Rust
```
