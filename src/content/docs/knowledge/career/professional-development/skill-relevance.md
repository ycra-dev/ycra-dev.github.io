## 핵심 개념

기술 관련성(Skill Relevance)은 보유 기술이 현재 시장에서 얼마나 수요가 있는지의 척도다. 고용주가 요구하는 기술(fresh)과 시장이 요구하는 기술(relevant)을 구분하여 커리어 위험을 관리한다.

## 동작 원리

**기술의 두 축:**
- **Fresh**: 현재 직무에 필요한 기술의 최신 상태 — "지금 일자리를 지키는 기술"
- **Relevant**: 업계 전반에서 수요가 있는 기술 — "다음 일자리를 얻게 해주는 기술"

**4사분면 분석:**

| | Fresh (현재 직무에 최신) | Not Fresh |
|---|---|---|
| **Relevant (시장 수요)** | Awesome Zone | Career-Safe |
| **Not Relevant** | Job-Safe | Danger Zone |

- **Awesome Zone**: 현재 직무도 잘하고, 시장에서도 수요가 높음. 최상의 위치
- **Job-Safe**: 현재 직무는 안전하지만 시장에서 사라지는 중. 위험 신호
- **Career-Safe**: 시장 수요는 있지만 현재 깊이가 부족. 학습 필요
- **Danger Zone**: 현재 직무도 위태롭고, 시장에서도 수요가 없음. 즉시 전환 필요

**관련성 판단 방법:**
- 구인 공고를 분석하여 시장 선두 기술 파악
- 기본적인 친숙도(40~80시간의 학습)를 목표로 삼기
- 전문가가 될 필요 없이, "이 기술이 무엇인지, 어떨 때 쓰는지" 이해하면 충분

## 예시

```
# 4사분면 예시 (회사가 레거시 기술을 사용하는 경우)

기술 포트폴리오 분석:
  VBScript      → Job-Safe (회사에서 쓰지만 시장에서 사라짐)
  JavaScript    → Awesome Zone (회사+시장 모두 활발)
  Python        → Career-Safe (시장 수요 있지만 현재 깊이 부족)
  Access        → Danger Zone (회사에서도 줄이고, 시장에서도 수요 없음)

행동 계획:
  1. JavaScript (Awesome Zone): 현재 상태 유지
  2. VBScript (Job-Safe): 장기적으로 대체 계획 수립
  3. Python (Career-Safe): 깊이를 키우기 위한 학습 계획
  4. Access (Danger Zone): 즉시 다른 기술로 전환

관련성 확인 방법:
  - LinkedIn Jobs / Indeed에서 원하는 직무 10개 확인
  - 요구 기술 목록에서 공통 항목 추출
  - 현재 내 기술과 비교
```
