## 핵심 개념

어떤 기술이든 빠르게 배우려면 세 가지만 알면 된다:
1. **시작하는 방법**: 기본적으로 알아야 할 최소한의 것
2. **주제의 범위(breadth)**: 전체적으로 무엇이 가능한지 개요
3. **기본기(basics)**: 일상 사용의 80%를 커버하는 20%의 핵심 지식 (파레토 원칙)

## 동작 원리

```
10단계 구성:

[준비 단계 - 1회 실행]
1단계: 큰 그림 파악 (Big Picture)
2단계: 범위 결정 (Determine Scope)
3단계: 성공 정의 (Define Success)
4단계: 자료 수집 (Find Resources)
5단계: 학습 계획 수립 (Create Learning Plan)
6단계: 자료 필터링 (Filter Resources)

[실행 단계 - 모듈별 반복]
7단계: 시작할 만큼 배우기 (Learn Enough to Get Started)
8단계: 놀며 탐구 (Play Around)
9단계: 유용한 수준까지 배우기 (Learn Enough to Be Useful)
10단계: 가르치기 (Teach)
```

기존 방식(책 처음부터 끝까지 읽기)은 느리고 중요한 것과 사소한 것이 섞여 비효율적이다.

## 예시

Go 언어 학습 적용:
```
1단계: Go가 무엇인지, 어디에 쓰이는지 파악
2단계: "간단한 HTTP 서버 만들기"로 범위 좁힘
3단계: "Hello World API를 30분 안에 실행"이 성공 기준
4단계: Tour of Go, Go by Example, 유튜브 튜토리얼 수집
5단계: 기초 → goroutine → HTTP 순서의 학습 경로
6단계: Tour of Go + 유튜브 1개만 선택
7단계: Hello World 실행
8단계: 간단한 계산기 서버 만들기
9단계: goroutine, 에러 처리 학습
10단계: 블로그 "Go로 HTTP 서버 만들기" 작성
```
