## 핵심 개념

독학 프로그래머는 어떤 도전도 해낼 수 있다는 자신감이 있다. 하지만 때로는 과신하여 "카우보이 코더"로 불리기도 한다.

**장점:**
- **비용이 낮거나 무료**: 인터넷에 무료 튜토리얼, 블로그, 완전한 교재도 있음
- **자기 교육 능력 습득**: 인생에서 가장 가치 있는 기술 중 하나
- **자신의 속도로 학습**: 완전히 이해한 후 다음으로 넘어갈 수 있음
- **유연한 스케줄**: 직장을 유지하면서 학습 가능

**단점:**
- **무엇을 배울지 스스로 결정**: 절대적 유연성은 방향성 상실로 이어질 수 있음
- **취업 지원 없음**: 대학의 인턴십이나 부트캠프의 취업 배치 서비스가 없음
- **동기 상실 쉬움**: 새로움이 사라진 후가 진짜 시험
- **지식 공백**: 모범 사례나 정통 방법론을 놓칠 수 있음

## 동작 원리

```
전략적 자기 교육 계획:

1. 계획 수립:
   - 학습할 내용, 소요 기간, 단계별 목표 정의

2. 스케줄 설정 (가장 중요!):
   - 매일 오후 7:30-9:30 프로그래밍 학습
   - 누적 효과(cumulative effect)를 활용

3. 네트워킹 병행:
   - 밋업, 사용자 그룹 참석
   - 블로그 시작, 커뮤니티 참여

4. 멘토 확보:
   - 막힐 때 도움을 요청할 수 있는 개발자 1-2명 확보

5. 사이드 프로젝트 진행:
   - 실제 문제를 풀며 학습
   - 포트폴리오 구축
```

## 예시

Pluralsight, Udemy, Coursera 같은 유료 플랫폼도 대학이나 부트캠프보다 훨씬 저렴하다. 자기 교육을 마스터한 사람에게는 거의 불가능한 것이 없다.
