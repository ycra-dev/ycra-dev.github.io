## 핵심 개념

전문화와 다재다능함은 상호 배타적이지 않다. 다양한 기술을 보유하면서도 **마케팅과 포지셔닝에서는 특정 전문 분야를 내세워야** 한다.

"모든 것을 할 수 있는 사람"으로 자신을 마케팅하는 것은 효과적이지 않다. 팀에 만능 개발자가 있으면 좋지만, 기업이 채용 공고를 낼 때 "아무거나 다 할 수 있는 사람"을 찾지는 않는다.

## 동작 원리

핵심 원칙:
- **배움**: 가능한 한 많이, 넓게 배워라
- **마케팅**: 특정 전문 분야를 내세워라
- 두 가지 중 택해야 한다면, **전문화를 먼저** 하고 나중에 확장하라

전문 분야는 영구적이지 않다:
- 기술 변화에 따라 전문 분야를 바꿀 수 있다
- John Papa: Silverlight 전문가 → SPA(Single Page Application) 전문가
- 저자: 프린터 개발 → 온라인 교육 콘텐츠

전문 분야 선택 질문:
- 현재/이전 회사의 주요 문제점은 무엇인가?
- 아무도 하고 싶어하지 않거나 숙련자가 부족한 영역은?
- Stack Overflow에서 가장 자주 답변하는 질문 분야는?

## 예시

T자형 개발자 모델:
```
넓은 지식 (상단 가로줄)
─────────────────────────────────
Python | JS | Go | SQL | DevOps

깊은 전문성 (세로줄)
│
│  React + TypeScript
│  + 전자상거래 플랫폼
│  + 결제 시스템 통합
│
▼

마케팅 메시지:
"전자상거래 플랫폼의 React/TypeScript 프론트엔드 전문가"
(Python, Go 등도 할 수 있지만, 이것이 나의 포지셔닝)
```
