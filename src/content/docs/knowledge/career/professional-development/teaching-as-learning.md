## 핵심 개념

가르침은 표면적 학습(learning)을 진정한 이해(understanding)로 전환하는 가장 강력한 방법이다. 가르치려 할 때 비로소 자신의 지식 격차가 드러난다.

학습(learning) vs 이해(understanding):
- 학습: 일시적. 구구단을 외우는 것
- 이해: 영구적. 곱셈의 원리를 아는 것

## 동작 원리

**표면적 이해의 함정**: 패턴 인식으로 문제를 풀 수 있지만, "왜?"라고 물으면 막힌다. 가르침이 이 격차를 노출시킨다.

**뇌의 데이터 재조직**: 학습 시 정보는 비체계적으로 저장된다. 가르치려면 이를 논리적으로 재조직해야 하므로, 자신에게 다시 가르치는 효과가 있다.

**겸손하되 자신감 있게**: 가르칠 때 우월감을 보이지 않되, 확신을 가지고 전달해야 한다.

## 예시

가르침의 형태 (쉬운 것부터):
1. 동료에게 개념 설명하기
2. 포럼에서 질문에 답하기
3. 블로그 포스트 작성하기
4. 스크린캐스트/튜토리얼 제작하기 (시청각 + 시연)

"가르칠 자격이 없다"는 가장 흔한 변명이지만, 실제로는 이미 일상에서 가르치고 있다 (동료에게 프레임워크 사용법 설명 등).

C# 람다 표현식 예: 알고는 있지만 "왜 이렇게 작동하나요?"라는 질문에 답하려면 다시 공부해야 한다.
