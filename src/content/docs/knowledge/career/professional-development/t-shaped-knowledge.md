## 핵심 개념

제너럴리스트 vs 스페셜리스트 논쟁의 답은 "둘 다"이다. T자형 지식은 소프트웨어 개발의 넓은 기반 지식과 하나 이상의 전문 분야에 대한 깊은 지식을 동시에 갖추는 구조다.

## 동작 원리

T자형 지식 구조:

```
←────────── 넓은 기반 지식 (T의 가로줄) ──────────→
알고리즘 │ 자료구조 │ 아키텍처 │ 프론트엔드 │ 백엔드 │ DB │ DevOps

                          │
                          │ 깊은 전문성
                          │ (T의 세로줄)
                          ↓
                   React 성능 최적화
```

**가로줄: 넓은 기반**
- 모범 사례, 알고리즘, 자료구조, 아키텍처, 프론트엔드, 백엔드, 데이터베이스, DevOps 등
- 해마다 조금씩 성장시킨다

**세로줄: 깊은 전문성**
- 최소 하나의 전문 분야에서 깊이 파고든다
- 개인 브랜드를 구축하는 데 활용한다

**빗살형으로 발전 (Comb-Shaped)**

여러 전문 분야를 갖춘 "빗살형 지식"으로 발전할 수 있지만, 처음에는 하나의 전문 분야부터 시작해야 한다.

## 예시

- 구강외과의: 전문의가 되기 위해 일반 치과 과정을 먼저 이수 → 일반 치과 치료도 수행 가능
- T자형 개발자: 웹 개발 전반에 대한 이해 + React 성능 최적화 전문가
- Elon Musk: 전기차, 우주, 뇌-컴퓨터 인터페이스 등 여러 깊은 전문성을 갖춘 빗살형 지식
