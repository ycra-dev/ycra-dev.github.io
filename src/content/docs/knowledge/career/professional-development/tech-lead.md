## 핵심 개념

테크 리드는 일반 개발자에서 한 단계 올라간 역할이다. 개발자 팀을 이끌며 일부 관리 업무와 개발 업무를 동시에 수행한다.

주요 책임:
- 코드베이스 아키텍처에 기여
- 코드 리뷰 수행
- 다른 개발자 멘토링
- 프로젝트의 중요한 기술 결정
- 개발자 면접과 작업 배정

**테크 리드는 대부분의 업무 시간에 여전히 코드를 작성**하며, 이 점이 매니저와 구분된다.

## 동작 원리

| 항목 | 테크 리드 | 매니저 |
|------|-----------|--------|
| 코딩 | 대부분의 시간 | 거의 없음 |
| 기술 결정 | 직접 수행 | 위임 |
| 인사 관리 | 최소 | 주요 업무 |
| 아키텍처 | 직접 기여 | 방향 설정 |

숙련된 개발자의 가장 큰 가치는 개인의 코딩 능력이 아니라, **전체 개발팀의 역량을 향상시키는 교육과 멘토링 능력**이다.

## 예시

```
테크 리드의 일상:
- 오전: 팀 스탠드업, 기술 장애물 해결
- 오전: 주니어 개발자의 PR 코드 리뷰
- 오후: 새 기능의 아키텍처 설계 미팅
- 오후: 핵심 기능 코딩
- 수시: 팀원의 기술적 질문에 답변, 멘토링
```

경력 사다리:
Developer → Senior Developer → **Tech Lead** → Architect → Director/Fellow
