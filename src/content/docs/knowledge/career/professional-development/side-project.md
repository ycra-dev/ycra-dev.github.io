## 핵심 개념

사이드 프로젝트는 어떤 교육 경로를 선택하든 반드시 해야 하는 활동이다. Microsoft, Facebook, Yahoo, Dell, Google 모두 기숙사방이나 차고에서의 사이드 프로젝트로 시작했다.

**사이드 프로젝트의 동시 다중 혜택:**
1. 학습 내용을 적용할 **실제 문제** 제공
2. 면접에서 보여줄 **실제 작업 포트폴리오** 생성
3. **부수입** 또는 새로운 사업 가능성
4. 학습하면서 자신에게 **유용한 도구** 제작
5. 능력에 대한 **자신감** 구축
6. **재미있는** 활동

## 동작 원리

"이중 이익(double-dip)" 전략: 어차피 코드를 작성할 것이라면, 그 코드가 여러 방면으로 유용한 일을 하게 만들어라.

## 예시

```
초급 단계:
- 개인 할 일 관리 앱 (웹/모바일)
- 가계부 또는 지출 추적 앱
- 날씨 정보 API를 활용한 날씨 앱

중급 단계:
- 블로그 플랫폼 (직접 구축)
- 개인 포트폴리오 웹사이트
- 오픈소스 프로젝트 기여

고급 단계:
- SaaS 도구 (실제 사용자 대상)
- 모바일 앱 출시 (앱스토어)
- API 서비스 구축 및 운영

포트폴리오 효과 극대화:
├── GitHub에 소스 코드 공개
├── README에 기술 스택 및 아키텍처 설명
├── 라이브 데모 링크 포함
└── 면접에서 기술적 결정의 이유를 설명할 수 있도록 준비
```
