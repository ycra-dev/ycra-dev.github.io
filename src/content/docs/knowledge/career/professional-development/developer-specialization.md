## 핵심 개념

대부분의 개발자는 프로그래밍 언어로만 자신을 정의한다 ("저는 C# 개발자입니다"). 이것은 너무 광범위하다. 프로그래밍 언어는 도구일 뿐, 어떤 종류의 개발자인지를 말해주지 않는다.

전문화의 규칙: **더 깊이 전문화할수록 잠재적 기회는 줄어들지만, 그 기회를 얻을 확률은 크게 높아진다.**

## 동작 원리

전문화의 이점:
- 제네럴리스트 변호사를 고용할 사람이 거의 없듯이, 제네럴리스트 개발자의 수요도 제한적이다
- 전문가 풀이 작을수록 각 전문가의 가치는 높아진다
- 기업이 특정 전문가를 찾는 것은 어렵기 때문에 발견되면 높이 평가받는다

전문화 가능한 영역:
- 웹 개발 스택 (예: React + Node.js + PostgreSQL)
- 임베디드 시스템
- 특정 OS 환경
- 모바일 개발 (iOS, Android)
- 특정 프레임워크 또는 소프트웨어 시스템 (SAP, Salesforce 등)

## 예시

전문화 수준에 따른 기회 분석:
```
Java 개발자:              500개 일자리  | 경쟁률 높음
Java 웹 개발자:           250개 일자리  | 경쟁률 중간
Java + Spring Boot:        50개 일자리  | 경쟁률 낮음
Java + Spring Boot + 금융:  10개 일자리  | 경쟁자 거의 없음

→ 10개 중 1개를 확실히 얻는 것이
  500개 중 1개를 간신히 얻는 것보다 낫다
```

저자의 경험: 프린터 및 프린터 언어 전문 개발자로 특화하여, 소수의 프린터 제조업체에서 매우 높은 가치를 인정받음.
