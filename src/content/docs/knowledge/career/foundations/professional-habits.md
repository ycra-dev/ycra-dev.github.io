## 핵심 개념

프로페셔널이 되기 위해 일상적으로 수행하는 습관들. 우리가 매일 하는 일의 대부분은 습관에 의해 좌우되므로, 프로의 습관을 형성하는 것이 프로페셔널리즘의 기반이다.

좋은 습관이 일관성을 만들고, 일관성이 신뢰를 만든다.

## 동작 원리

프로 개발자가 갖추어야 할 핵심 습관:

1. **사전 준비 습관**: 스크럼 미팅 전에 발언 내용을 미리 정리하는 것처럼, 중요한 활동 전에 준비하는 습관
2. **시간 관리 습관**: 매일 무엇을 작업할지 미리 계획하고, 작업 소요 시간을 예측하는 습관
3. **지속적 학습 습관**: 매일 일정 시간을 학습에 투자하는 습관
4. **품질 확인 습관**: 코드를 제출하기 전에 반드시 리뷰하는 습관

습관 형성의 핵심:
- 작게 시작하라
- 일관되게 반복하라
- 환경을 설계하라 (습관을 실행하기 쉽게)

## 예시

```
프로 개발자의 아침 루틴 예시:

08:30 - 출근, 오늘의 작업 목록 검토 및 우선순위 설정
08:45 - 이메일/Slack 확인 (15분 제한)
09:00 - 스탠드업 미팅 (사전 준비된 발언)
09:15 - 딥 워크 시작 (가장 중요한 작업부터)
11:30 - 코드 리뷰 시간
12:00 - 점심 + 기술 아티클 1개 읽기

[스크럼 미팅 사전 준비]
"어제: 결제 API 에러 핸들링 완료
 오늘: 결제 실패 시 재시도 로직 구현
 블로커: 없음"
→ 즉석에서 말하는 다른 사람들보다 일관되고 전문적으로 보임
```

추천 도서: Charles Duhigg의 "The Power of Habit"
