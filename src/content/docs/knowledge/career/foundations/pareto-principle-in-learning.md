## 핵심 개념

기술을 가르칠 때 세 가지에 집중하라:
1. **전체 그림(Big Picture)**: 기술로 무엇을 할 수 있는지 개요 파악. "알려지지 않은 미지(unknown unknowns)"를 제거하는 것이 목적.
2. **시작하는 방법**: 다운로드, 설치, 첫 프로젝트 생성, 컴파일까지. 이 장벽이 너무 높으면 손을 대지 않게 된다.
3. **핵심 20%**: 80% 이상의 시간에 사용될 20%의 지식.

대부분의 책이나 튜토리얼은 레퍼런스 매뉴얼처럼 작성되어 가장 중요한 20%에 특별한 강조를 하지 않는다.

## 동작 원리

실제로 기술을 사용하면 무엇을 가장 많이 쓰는지 자연스럽게 알게 된다. 그래서 "실행을 통한 학습"이 중요하다. 전문가를 섀도잉하거나 그들의 작업을 관찰하는 것도 핵심 20%를 파악하는 효과적인 방법이다.

## 예시

SQL 학습에 파레토 원칙 적용:

```
전체 SQL 기능 (100%):
├── SELECT 문 (핵심 20%)      -> 실무 사용 빈도 매우 높음
├── JOIN (핵심 20%)           -> 실무 사용 빈도 매우 높음
├── WHERE 절 (핵심 20%)       -> 실무 사용 빈도 매우 높음
├── INSERT                    -> 보통
├── UPDATE                    -> 보통
├── DELETE                    -> 낮음
├── 인덱싱                     -> 가끔
├── 저장 프로시저              -> 가끔
└── 트리거                     -> 매우 드물게

-> 상위 3개(핵심 20%)만 잘 알아도 일상 업무의 80% 처리 가능
```
