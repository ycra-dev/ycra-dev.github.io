## 핵심 개념

기술을 가르칠 때 학생들이 당장 필요하지 않거나 필요할 때 스스로 배울 수 있는 내용으로 지루하게 만들지 마라. 대신 즉시 가치 있는 것을 가르치는 데 집중하고, 더 깊이 들어가야 할 때 "적시 학습"을 실천할 수 있는 자원을 제공하라.

이것은 제조업의 **JIT(Just-In-Time) 재고 관리** 방식과 유사하다. 필요한 것을 필요한 시점에 정확히 공급받아 낭비를 최소화하는 원리다.

핵심 원리:
- 당장 사용할 필요가 없는 기술이라면 배울 필요가 있는지 먼저 자문하라
- 핵심 20%를 먼저 익히고, 나머지는 필요할 때 배운다
- Google의 힘을 활용하되, 먼저 무엇이 존재하는지는 알아야 한다

## 동작 원리

```
전통적 방법 (비효율적):
1. 프레임워크 문서를 처음부터 끝까지 읽기 (3주)
2. 모든 기능을 예제로 연습하기 (2주)
3. 프로젝트 시작 (이미 배운 것의 대부분 잊음)

적시 학습 방법 (효율적):
1. 전체 그림 파악 (2시간)
2. 핵심 20% 학습 (3일): 라우팅, 미들웨어, 기본 CRUD
3. 실제 프로젝트 시작
4. 필요할 때마다 해당 주제 심화 학습
```

## 예시

웹 프레임워크를 배울 때:
- "인증이 필요하네?" → 인증 관련 문서 학습
- "파일 업로드가 필요하네?" → 파일 처리 학습
- "성능이 느리네?" → 캐싱 전략 학습
