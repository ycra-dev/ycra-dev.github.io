## 핵심 개념

화이트보드 코딩 인터뷰는 FAANG(Facebook/Meta, Amazon, Apple, Netflix, Google) 등 대형 기술 기업에서 표준화된 채용 방식이다. 알고리즘 지식보다 문제 해결 과정을 소통하는 능력을 본다. 실제 업무와 관련이 없다는 비판이 있지만, 여전히 많은 기업에서 사용한다.

## 동작 원리

화이트보드 인터뷰 최적 접근법:

```
1. 문제 이해 확인 (요구사항 재확인, 엣지 케이스 질문)
2. 예시 입출력으로 문제 구체화
3. 브루트포스 접근 먼저 설명
4. 최적화 방안 논의
5. 코드 작성 (생각을 소리 내어)
6. 테스트 케이스로 검증
7. 시간/공간 복잡도 분석
```

준비 방법:
- LeetCode Easy 50개 → Medium 100개 → Hard 일부
- 핵심 패턴: 투 포인터, 슬라이딩 윈도우, BFS/DFS, DP, 힙
- 시스템 설계 인터뷰는 별도 준비 필요

## 예시

- 문제 받은 후: "입력이 정렬되어 있나요? 중복값이 있을 수 있나요? 빈 배열은요?"
- 코딩 중: "지금 반복문으로 O(n)인데, 해시맵을 쓰면 O(1)로 줄일 수 있을 것 같아요"
- 면접 준비 도구: LeetCode, AlgoExpert, Blind 75 리스트, NeetCode YouTube
