## 핵심 개념

구직은 영업과 같다. 제품은 자신이고, 고객은 고용주이며, 거래는 채용 계약이다. 영업팀이 파이프라인을 관리하듯 구직 활동도 단계별로 추적하면 어느 단계에서 막히는지 파악하고 전략을 조정할 수 있다.

## 동작 원리

구직 파이프라인 단계:

```
[리드 발굴] → [이력서 제출] → [전화 스크리닝]
    → [기술 인터뷰] → [최종 인터뷰] → [오퍼] → [협상] → [수락]
```

각 단계별 성공률 추적:
- 지원 50개 → 전화 스크리닝 10개 (20%) → 기술 인터뷰 5개 → 오퍼 2개
- 전환율이 낮은 단계를 찾아 집중 개선

핵심 지표:
- 주간 지원 수
- 단계별 전환율
- 응답 없는 지원의 비율 (이력서 문제 신호)
- 기술 인터뷰 통과율 (기술 준비 문제 신호)

## 예시

- Google Sheets나 Notion으로 지원 현황 추적: 회사명, 직무, 지원일, 현재 단계, 다음 액션
- 50개 지원 후 2개 이하만 응답 → 이력서 개선 필요
- 기술 인터뷰 10번 중 1번만 통과 → 코딩 연습 강화 필요
- CRM 도구(Huntr, JobHero) 사용으로 파이프라인 시각화
