## 핵심 개념

대부분의 이력서는 "무엇을 했는가(duties)"를 나열한다. 채용 담당자가 보고 싶은 것은 "어떤 가치를 만들었는가(value)"이다. 같은 경험이라도 가치 중심으로 표현하면 전혀 다른 인상을 준다.

## 동작 원리

의무 중심 vs 가치 중심 비교:

| 의무 중심 (나쁜 예) | 가치 중심 (좋은 예) |
|--------------------|---------------------|
| "API 개발 담당" | "결제 API 개발로 결제 완료율 23% 향상" |
| "데이터베이스 최적화" | "쿼리 최적화로 응답시간 4초 → 0.3초 단축, 서버 비용 월 $2,000 절감" |
| "팀 리드 역할" | "5인 팀 리드로 3개 분기 연속 스프린트 목표 100% 달성" |

가치를 표현하는 공식:
```
[행동] + [무엇을] + [결과/임팩트] + [측정 지표]
```

숫자가 없으면 추정값이라도 사용: "약 30%" "2배 이상" "대규모(500만 사용자 이상)"

## 예시

- "레거시 코드 리팩토링" → "3년된 모놀리식 결제 모듈을 마이크로서비스로 전환, 배포 주기 2주 → 1일 단축"
- "고객 지원 응대" → "기술 지원 티켓 24시간 평균 해결로 고객 만족도(CSAT) 4.2 → 4.8점 향상"
- XYZ Formula: "X를 하여 Y를 Y만큼 달성 (측정 지표 Z 기준)"
