## 핵심 개념

일반 구직(아웃바운드)은 내가 회사를 찾아다닌다. 인바운드 마케팅은 회사와 리크루터가 나를 찾아오게 만든다. 기술 블로그, 오픈소스 프로젝트, 컨퍼런스 발표, 강의 영상 등으로 온라인 존재감을 쌓으면, 리크루터가 먼저 LinkedIn으로 연락을 해온다.

## 동작 원리

인바운드 채용 파이프라인:

```
[콘텐츠 생성] → [검색 노출 + 공유]
    → [전문가 인지도] → [리크루터/채용담당자 연락]
    → [인터뷰 없이 바로 채용 오퍼] (최상위 시나리오)
```

인바운드를 만드는 채널:
- **기술 블로그**: SEO를 통해 자연 유입, 관련 기업 리크루터 독자
- **GitHub**: star가 많은 오픈소스 프로젝트
- **LinkedIn 게시물**: 업계 동향이나 학습 공유
- **컨퍼런스 발표**: "XX 컨퍼런스 발표자"라는 강력한 신뢰 신호
- **Stack Overflow**: 특정 기술 분야 최상위 답변자

## 예시

- 저자 경험: Simple Programmer 블로그가 유명해진 후 면접 없이 바로 채용 제안을 받음
- React 성능 최적화 블로그를 운영 → 해당 기술을 쓰는 스타트업 CTO가 직접 연락
- GitHub에 인기 라이브러리 공개 → "당신 코드 잘 봤습니다, 우리 팀에 관심 있으세요?"
