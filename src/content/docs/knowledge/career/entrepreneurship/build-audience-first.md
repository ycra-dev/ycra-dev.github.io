## 핵심 개념

많은 기업가가 훌륭한 제품을 만든 후 고객을 찾지 못해 실패한다. 관계 없이 누군가에게 무언가를 팔기는 매우 어렵다. 반대로, 먼저 오디언스를 구축하면 제품 출시 시 이미 구매 준비된 사람들이 있다.

## 동작 원리

오디언스 먼저 구축의 장점:

1. **제품 출시 시 즉각적 구매자**: 이미 당신을 신뢰하는 사람들이 새 제품을 먼저 산다
2. **제품 방향 파악**: 오디언스가 무엇을 원하는지 파악할 수 있다
3. **다중 제품 기회**: 하나의 오디언스에 여러 제품을 판매할 수 있다
4. **마케팅 비용 절감**: 오디언스가 스스로 입소문을 낸다

**오디언스 vs 제품 먼저**

```
제품 먼저:
완벽한 앱 개발 → 고객 찾기 실패 → 자금 고갈

오디언스 먼저:
블로그/YouTube → 관계 구축 →
오디언스가 원하는 것 파악 → 제품 개발 → 즉시 판매
```

오디언스 구축에는 시간이 걸리고 대량의 무료 콘텐츠를 만들어야 하지만, 한번 확보하면 단일 제품에 의존하지 않는 지속 가능한 사업이 된다.

## 예시

- 저자의 Simple Programmer: 이메일 목록 70,000명, YouTube 구독자 100,000명 → 신제품 출시 시 즉시 수천 명이 구매
- 블로그 독자에게 "어떤 강좌가 필요한가요?" 설문 → 응답 기반 강좌 개발 → 출시 즉시 판매
- 제품 출시 전 이메일 사전 등록 → 관심도 측정 + 초기 고객 확보
