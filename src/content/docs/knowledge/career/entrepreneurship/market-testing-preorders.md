## 핵심 개념

아이디어가 좋아 보여도 실제로 사람들이 돈을 낼지는 다른 문제다. 가장 확실한 검증 방법은 **제품을 만들기 전에 사전 주문을 받는 것**이다.

## 동작 원리

사전 판매 페이지 구성:
```
1. 제품 설명 (무엇을 만들 것인지)
2. 해결할 문제 (고객의 고충)
3. 출시 예정일
4. 할인가 (출시 전 특별가)
5. 환불 보장
```

사전 주문이 거의 없다면:
- 소수의 구매자에게 환불하고 방향을 전환한다
- 이것은 3개월 이상 제품을 만들고 나서 아무도 안 사는 것보다 훨씬 낫다

사전 구매자의 추가 가치: 제품 개발 중 피드백을 줄 수 있는 초기 고객이 된다.

## 예시

- 저자: 사전 판매 페이지 공개 → 첫날 7건 판매 → 제품 개발 진행 확신
- 사전 주문이 적으면 → 환불 후 방향 전환 (3개월 낭비보다 낫다)
- Eric Ries의 "The Lean Startup" 참고
