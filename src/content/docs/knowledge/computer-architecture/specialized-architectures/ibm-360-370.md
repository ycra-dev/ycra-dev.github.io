## 핵심 개념

IBM 360은 Amdahl, Blaauw, Brooks가 설계하여 컴퓨터 아키텍처의 여러 중요한 혁신을 도입했다:

**핵심 혁신**:
1. **ISA 개념 확립**: 동일 아키텍처로 성능/비용이 다른 여러 구현
2. **32비트 아키텍처**: 처음으로 32비트 설계 채택
3. **바이트 주소 지정 메모리**: 8비트 바이트 단위 접근
4. **다양한 데이터 크기**: 8, 16, 32, 64비트 데이터 지원
5. **부동소수점**: 32비트 단정밀도, 64비트 배정밀도

**명령어 형식** (6가지):
- **R-R (Register-Register)**: 레지스터 간 연산
- **RX (Register-indeXed)**: 레지스터-메모리 인덱스 연산
- **RS (Register-Storage)**: 레지스터-스토리지 연산
- **SI (Storage-Immediate)**: 스토리지-즉시값 연산
- **SS (Storage-Storage)**: 스토리지 간 연산 (BCD/문자열)

**발전 과정**:
- 1964: System/360 - 6개 모델, 25:1 성능 비
- 1971: System/370 - 가상 메모리 추가
- 1984: 370-XA - 31비트 주소로 확장 (24비트에서)

IBM 360/370은 80x86보다 훨씬 오래된 아키텍처이지만, 더 나은 기반 설계와 일관된 확장 덕분에 메인프레임 시장을 지배하며 여전히 사용되고 있다.

## 예시

```
IBM 360 명령어 형식:

R-R Format (2바이트):
[Op(8) | R1(4) | R2(4)]
예: AR R1,R2  (R1 = R1 + R2)

RX Format (4바이트):
[Op(8) | R1(4) | X2(4) | B2(4) | D2(12)]
예: A R1,D2(X2,B2)  (R1 = R1 + Mem[X2+B2+D2])

RS Format (4바이트):
[Op(8) | R1(4) | R3(4) | B2(4) | D2(12)]

SI Format (4바이트):
[Op(8) | I2(8) | B1(4) | D1(12)]

SS Format (6바이트):
[Op(8) | L(8) | B1(4) | D1(12) | B2(4) | D2(12)]

명령어 사용 빈도 (4개 프로그램 평균):
제어:     27%
산술:     16%
데이터 전송: 29%
논리:     12%
비교:     9%
```
