## 핵심 개념

패킷(Packet)은 네트워크에서 전송되는 데이터의 **기본 단위**로, 발신자와 수신자 주소, 실제 데이터, 그리고 오류 검출 등의 부가 정보를 포함하는 구조화된 데이터 블록이다.

## 동작 원리

데이터는 한 번에 통째로 보내지 않고 **작은 조각(패킷)으로 나누어** 전송된다. 이는 편지 봉투에 비유할 수 있다. 봉투에는 보내는 사람 주소, 받는 사람 주소가 적혀 있고 안에 내용물이 들어 있듯이, 패킷에도 헤더(주소 정보)와 페이로드(실제 데이터)가 포함된다.

**패킷 교환(Packet Switching)의 장점**:
- **효율성**: 회선 교환(전화처럼 전용 경로 점유)과 달리, 네트워크 자원을 여러 사용자가 공유할 수 있다
- **견고성**: 하나의 경로가 실패해도 패킷은 다른 경로로 우회할 수 있다
- **공정성**: 큰 데이터도 작은 패킷으로 나뉘므로, 한 사용자가 네트워크를 독점하지 않는다

**패킷의 구성 요소**:
- **헤더(Header)**: 출발지 주소, 목적지 주소, 프로토콜 정보, 시퀀스 번호 등
- **페이로드(Payload)**: 실제 전송하려는 데이터
- **트레일러(Trailer)**: 오류 검출 코드(CRC 등)

이더넷 패킷(프레임)의 최대 크기는 약 1500바이트이다. 큰 데이터를 보내려면 여러 패킷으로 분할(fragmentation)해야 하며, 수신 측에서 올바른 순서로 재조립(reassembly)한다.

각 패킷은 독립적으로 라우팅되므로, 같은 메시지의 패킷들이 서로 다른 경로를 거쳐 도착할 수 있고, 도착 순서도 뒤바뀔 수 있다. TCP 프로토콜이 이를 바로잡아 올바른 순서로 재조립하는 역할을 한다.

## 예시

1GB 파일 전송 시 패킷 분할:

```
1 GB = 1,073,741,824 바이트
패킷당 페이로드 ≈ 1,460 바이트 (TCP/IP 헤더 40바이트 제외)
필요한 패킷 수 ≈ 735,590개
```

패킷의 논리적 구조:

```
+------------------------------------------+
| 이더넷 헤더 (14B)                         |
|   목적지 MAC | 출발지 MAC | 타입          |
+------------------------------------------+
| IP 헤더 (20B)                             |
|   출발지 IP | 목적지 IP | TTL | ...       |
+------------------------------------------+
| TCP 헤더 (20B)                            |
|   출발지 포트 | 목적지 포트 | 시퀀스 번호  |
+------------------------------------------+
| 데이터 (페이로드)                          |
+------------------------------------------+
```
