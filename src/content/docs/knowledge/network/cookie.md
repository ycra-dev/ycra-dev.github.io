## 핵심 개념

쿠키(Cookie)는 웹 서버가 사용자의 브라우저에 저장하는 작은 텍스트 데이터로, 세션 관리, 사용자 식별, 온라인 활동 추적 등에 사용된다. HTTP는 **무상태(stateless)** 프로토콜이므로, 쿠키는 서버가 이전 요청을 기억하지 못하는 문제를 해결한다.

## 동작 원리

쿠키는 1994년 Netscape에서 HTTP의 무상태 문제를 해결하기 위해 도입되었다.

**동작 흐름**:
1. 사용자가 웹사이트에 처음 접속하면, 서버가 응답 헤더에 `Set-Cookie`를 포함하여 쿠키를 설정한다.
2. 브라우저는 이 쿠키를 로컬에 저장한다.
3. 이후 같은 사이트에 요청을 보낼 때마다 브라우저가 자동으로 쿠키를 함께 전송한다.
4. 서버는 쿠키를 통해 사용자를 식별한다.

**주요 용도**:
- **세션 관리**: 로그인 상태 유지, 장바구니 기억
- **개인화**: 사용자 선호 설정(언어, 테마) 저장
- **추적**: 사용자의 방문 기록, 행동 패턴 수집

**제1자 쿠키 vs 제3자 쿠키**:
- **제1자 쿠키**: 방문 중인 사이트가 설정. 로그인 유지 등 유용한 기능.
- **제3자 쿠키**: 방문 중인 사이트가 아닌 다른 도메인(광고 네트워크 등)이 설정. 여러 사이트에 걸친 사용자 추적에 핵심적으로 사용되며, 프라이버시 침해의 주요 원인이다.

제3자 쿠키를 통해 광고 네트워크는 사용자가 어떤 사이트를 방문하고, 무엇을 검색하고, 무엇에 관심을 보이는지 광범위하게 추적할 수 있다.

## 예시

```
HTTP 응답 (서버 → 브라우저):
Set-Cookie: session_id=abc123; Expires=Thu, 01 Jan 2027 00:00:00 GMT; Secure; HttpOnly

HTTP 요청 (브라우저 → 서버):
Cookie: session_id=abc123
```

실제 시나리오:
1. 온라인 쇼핑몰에 로그인 → 서버가 session_id 쿠키 설정
2. 상품 페이지 이동 → 브라우저가 자동으로 쿠키 전송
3. 서버가 쿠키로 로그인 상태 확인 → 개인화된 페이지 표시

제3자 쿠키 추적 시나리오:
```
뉴스 사이트 A 방문 → 광고 네트워크 X의 쿠키 저장
쇼핑 사이트 B 방문 → 같은 광고 네트워크 X의 쿠키 전송
광고 네트워크 X는 사용자가 A와 B 모두 방문한 것을 알게 됨
```
