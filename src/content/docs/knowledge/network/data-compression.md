## 핵심 개념

데이터 압축은 정보의 중복을 제거하거나 인간의 인지 한계를 이용하여 데이터의 크기를 줄이는 기술이다. 원본을 완벽히 복원할 수 있는 **무손실 압축**과 일부 정보를 버리는 **손실 압축**으로 나뉜다.

## 동작 원리

### 무손실 압축 (Lossless Compression)
원본 데이터를 100% 복원할 수 있다. 텍스트, 프로그램 코드, 데이터베이스 등 정확한 복원이 필요한 경우에 사용한다.

- **런 길이 부호화 (Run-Length Encoding, RLE)**: 연속 반복되는 데이터를 횟수로 표현. "AAAAABBB" → "5A3B"
- **허프만 코딩 (Huffman Coding)**: 자주 나타나는 문자에 짧은 코드를, 드물게 나타나는 문자에 긴 코드를 할당.
- **LZ77/LZW**: 이전에 나온 패턴을 참조하여 압축. ZIP, gzip의 기반
- **일반적인 압축률**: 텍스트 약 60-80% 감소

### 손실 압축 (Lossy Compression)
인간이 감지하기 어려운 정보를 제거하여 높은 압축률을 달성한다. 원본 복원은 불가능하다.

- **JPEG**: 이미지 압축. 인간의 눈이 색상 변화보다 밝기 변화에 민감한 점을 이용.
- **MP3/AAC**: 오디오 압축. 인간의 귀가 듣지 못하는 주파수 제거.
- **H.264/H.265**: 비디오 압축. 프레임 간 차이만 저장.
- **일반적인 압축률**: 이미지 80-95%, 오디오 75-90% 감소

**허프만 코딩의 원리**: 영어에서 'e'는 가장 빈번한 문자이고 'z'는 가장 드문 문자이다. 빈번한 문자에 짧은 코드를 할당하면 전체 데이터 크기가 줄어든다.

## 예시

허프만 코딩 개념:

```
문자 빈도:  A=45%, B=20%, C=15%, D=20%

고정 길이: 각 문자 2비트 → 평균 2비트/문자

허프만:
  A → "0"     (1비트, 가장 빈번)
  B → "10"    (2비트)
  C → "110"   (3비트)
  D → "111"   (3비트)

평균 코드 길이: 0.45×1 + 0.20×2 + 0.15×3 + 0.20×3 = 1.9비트/문자
→ 약 5% 개선
```

파일 크기 비교:

```
12MP 사진 원본: ~36MB
JPEG 압축 후: ~3-5MB (약 10배 압축)
JPEG 품질이 낮을수록 더 작지만 화질 저하
```
