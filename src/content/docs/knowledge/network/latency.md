## 핵심 개념

지연 시간(Latency)은 데이터가 한 지점에서 다른 지점으로 이동하는 데 걸리는 시간이다. 물리적으로 빛의 속도가 이론적 하한이며, 실제로는 여러 요인으로 인해 그보다 훨씬 길다.

## 동작 원리

대역폭이 "한 번에 얼마나 많이"의 문제라면, 지연 시간은 "**얼마나 빨리 도착하는가**"의 문제이다.

**물리적 한계**: 빛의 속도는 약 초속 30만 km이다. 미국 횡단(약 4,000km) 기준으로 빛의 속도로도 약 13ms가 걸린다. 광섬유에서는 빛이 유리 속을 지나가므로 진공보다 약 30% 느려진다. 실제 인터넷 경로에서는 라우터 처리 시간, 큐잉 지연, 경로 우회 등으로 미국 횡단 지연이 약 40ms 정도가 된다.

**트럭에 실린 디스크 비유**: 10TB 디스크를 실은 트럭이 미국을 횡단하는 데 하루가 걸린다면, 지연 시간은 24시간(매우 높음)이지만 대역폭은 약 926 Mbps(상당히 높음)이다.

**지연 시간이 중요한 이유**:
- 대화형 응용(게임, 화상 통화)에서는 낮은 지연이 필수적이다
- 금융 거래에서 밀리초 단위 차이가 수백만 달러의 차이를 만든다
- 웹 페이지 로딩 시 수십 개의 요청이 발생하므로, 각 요청의 지연이 누적된다

**왕복 시간(RTT, Round-Trip Time)**: 실제로는 데이터를 보내고 응답을 받기까지의 왕복 시간이 중요하다. 지연 시간의 약 2배이다.

## 예시

다양한 상황에서의 지연 시간:

| 경로 | 이론적 최소 (빛) | 실제 지연 |
|------|-----------------|----------|
| 같은 건물 (LAN) | < 0.01 ms | ~1 ms |
| 같은 도시 | ~0.1 ms | ~5-10 ms |
| 미국 횡단 | ~13 ms | ~40 ms |
| 대서양 횡단 | ~20 ms | ~70-80 ms |
| 지구 반대편 | ~67 ms | ~150-300 ms |
| 위성 (정지궤도) | ~120 ms | ~600 ms |

```bash
# ping 명령으로 지연 시간 측정
$ ping google.com
PING google.com: 56 data bytes
64 bytes from 142.250.80.46: time=12.3 ms
64 bytes from 142.250.80.46: time=11.8 ms
```
