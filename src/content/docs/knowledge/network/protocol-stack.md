## 핵심 개념

프로토콜 스택은 네트워크 통신을 기능별 계층으로 나누어 구성한 구조로, 각 계층이 독립적으로 동작하며 아래 계층의 서비스를 이용하여 위 계층에 서비스를 제공하는 **계층적 추상화** 모델이다.

## 동작 원리

프로토콜 스택의 핵심 아이디어는 **계층적 추상화**이다. 복잡한 네트워크 통신을 여러 계층으로 나누면, 각 계층은 자신의 역할에만 집중하고 다른 계층의 내부 동작은 알 필요가 없다.

**인터넷 프로토콜 스택 (TCP/IP 모델)**:

| 계층 | 역할 | 프로토콜 예시 |
|------|------|-------------|
| **응용 계층** | 사용자 수준의 서비스 | HTTP, SMTP, DNS, SSH |
| **전송 계층** | 종단 간 신뢰성 있는 전송 | TCP, UDP |
| **인터넷 계층** | 패킷 라우팅과 주소 지정 | IP |
| **링크 계층** | 물리적 네트워크 내 전송 | 이더넷, WiFi |
| **물리 계층** | 비트를 전기/광/무선 신호로 변환 | 케이블, 광섬유, 전파 |

**편지 비유**: 프로토콜 스택은 편지 교환에 비유할 수 있다.
- 두 CEO가 편지를 주고받는다 (응용 계층)
- 비서가 편지를 봉투에 넣고 주소를 쓴다 (전송 계층)
- 사내 우편실이 외부 배송을 준비한다 (인터넷 계층)
- 우체국이 물리적으로 배달한다 (링크/물리 계층)
- 각 단계의 사람은 자기 역할만 수행하며, 편지 내용을 몰라도 된다

**캡슐화 (Encapsulation)**: 데이터가 각 계층을 내려가면서 해당 계층의 헤더가 추가된다.

```
[HTTP 데이터]
→ [TCP 헤더][HTTP 데이터]
→ [IP 헤더][TCP][HTTP]
→ [이더넷 헤더][IP][TCP][HTTP][CRC]
```

수신 측에서는 역순으로 헤더를 벗겨내며 각 계층이 처리한다.

**계층화의 장점**:
- 각 계층이 독립적으로 발전 가능 (이더넷 → WiFi로 바꿔도 TCP/IP는 변경 불필요)
- 문제 발생 시 해당 계층만 수정
- 표준화를 통해 서로 다른 제조사 제품 간 호환

## 예시

웹 페이지 요청의 계층별 처리:

```
[브라우저]   → HTTP GET 요청 생성                  (응용 계층)
[TCP 모듈]   → TCP 헤더 추가, 연결 관리           (전송 계층)
[IP 모듈]    → IP 헤더 추가, 목적지 IP로 라우팅   (인터넷 계층)
[이더넷 드라이버] → 이더넷 프레임으로 캡슐화      (링크 계층)
[네트워크 카드] → 전기 신호로 변환하여 전송       (물리 계층)
```
