## 핵심 개념

라우팅(Routing)은 네트워크에서 패킷이 출발지로부터 목적지까지 전달되는 경로를 결정하는 과정이다. 라우터가 패킷의 목적지 IP 주소를 보고 다음으로 전달할 노드(**다음 홉, next hop**)를 결정한다.

## 동작 원리

인터넷은 수십만 개의 네트워크가 연결된 거대한 그래프이다. 패킷이 출발지에서 목적지까지 도달하려면 여러 중간 라우터를 거쳐야 하며, 각 라우터는 패킷을 목적지에 가까운 다음 라우터로 전달한다.

**라우터의 동작**:
1. 패킷의 목적지 IP 주소를 확인
2. 라우팅 테이블을 참조하여 최적의 다음 홉(next hop) 결정
3. 해당 인터페이스로 패킷 전달

우편 시스템에 비유하면, 각 우체국(라우터)은 전체 배달 경로를 알 필요 없이 "이 지역 편지는 저 우체국으로 보내라"는 규칙만 알면 된다.

**라우팅 테이블**: 라우터가 가지고 있는 "다음 홉 결정 표"이다. 목적지 네트워크 주소와 그 네트워크로 가기 위한 다음 라우터의 주소가 매핑되어 있다.

```
목적지 네트워크        다음 홉          인터페이스
10.0.0.0/8           직접 연결         eth0
172.16.0.0/16        10.0.0.1         eth0
0.0.0.0/0            10.0.0.254       eth0  ← 기본 경로 (default)

→ 10.x.x.x는 직접 전달
→ 172.16.x.x는 10.0.0.1로 전달
→ 그 외 모든 것은 기본 게이트웨이(10.0.0.254)로 전달
```

**동적 라우팅**: 라우팅 테이블은 고정되지 않고 동적으로 갱신된다.
- 라우터들이 서로 정보를 교환하여 네트워크 상태 변화에 적응
- 링크가 끊어지면 대체 경로를 자동으로 찾음
- 트래픽 부하에 따라 경로를 분산시킬 수 있음
- **BGP(Border Gateway Protocol)**: 인터넷의 자율 시스템(AS) 간 경로 정보를 교환하는 핵심 프로토콜

**자율 시스템 (Autonomous System, AS)**: 인터넷은 하나의 거대한 네트워크가 아니라 수만 개의 독립적 네트워크(AS)가 연결된 것이다. 각 ISP, 대학, 기업이 하나의 AS를 운영하며, AS 간 라우팅은 BGP가 담당한다.

**인터넷의 견고성**: 동적 라우팅 덕분에 인터넷은 부분적 장애에 강하다. 하나의 경로가 끊어져도 패킷은 다른 경로를 찾아간다.

## 예시

`traceroute`로 실제 라우팅 경로 확인:
```bash
$ traceroute google.com
 1  192.168.1.1      1.2 ms   ← 가정용 라우터
 2  10.0.0.1         5.3 ms   ← ISP 라우터
 3  72.14.215.85     8.1 ms   ← ISP 백본
 4  108.170.250.33  10.5 ms   ← Google 네트워크 진입
 5  142.250.80.46   11.2 ms   ← 목적지 (google.com)

각 줄 = 하나의 "홉" (라우터를 거침)
```

동적 경로 변경:
```
정상 경로: A → B → C → D (최단)
B-C 링크 장애 발생!
새 경로:   A → B → E → F → D (우회)
→ 라우팅 프로토콜이 자동으로 대체 경로 탐색
```
