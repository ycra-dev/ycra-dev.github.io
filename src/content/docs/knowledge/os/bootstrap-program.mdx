## 핵심 개념

부트스트랩 프로그램은 컴퓨터 전원이 켜지거나 재부팅될 때 실행되는 초기 프로그램으로, 운영체제를 로드하는 역할을 한다.

> RAM은 휘발성이므로 부트스트랩 프로그램을 저장할 수 없다. 대신 **펌웨어(firmware)** 에 저장된다.

## 부트스트랩의 역할

1. CPU 레지스터부터 장치 컨트롤러, 메모리 내용까지 **시스템의 모든 측면을 초기화**
2. 운영체제 커널을 찾아 메모리에 로드
3. 운영체제 실행을 시작

## 저장 위치

| 저장 매체 | 특징 |
|----------|------|
| **펌웨어** | 드물게 쓰여지고 비휘발성인 저장장치 |
| **EEPROM** | 변경 가능하지만 자주 변경 불가, 느림 |

예: iPhone은 EEPROM을 사용하여 장치의 시리얼 번호와 하드웨어 정보를 저장한다.

| 4. 커널 로드 | 운영체제 커널을 찾아 메모리에 로드 |
| 5. 시스템 데몬 시작 | 커널 실행 후 시스템 서비스 시작 |
| 6. 부팅 완료 | 시스템과 사용자에게 서비스 제공 시작 |

## 시스템 데몬

커널 외부에서 일부 서비스는 부팅 시 메모리에 로드되는 **시스템 데몬(system daemons)** 에 의해 제공된다.

- 커널이 실행되는 전체 시간 동안 실행됨
- Linux에서 첫 번째 시스템 프로그램은 **systemd**이며, 이것이 다른 많은 데몬을 시작함
