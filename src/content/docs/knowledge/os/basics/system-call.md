## 핵심 개념

시스템 콜은 프로세스가 운영체제에게 어떤 동작을 요청하는 방법이다.

> 사용자 프로그램은 user mode에서 실행되므로 직접 하드웨어나 특권 작업에 접근할 수 없다. 시스템 콜을 통해 안전하게 OS 서비스에 접근한다.

## 동작 원리

1. 사용자 프로그램이 시스템 콜 실행
2. 인터럽트 벡터의 특정 위치로 trap 발생
3. mode bit이 kernel mode로 설정됨
4. 커널이 어떤 시스템 콜인지 판단
5. 파라미터의 정확성과 적법성 검증
6. 요청 실행 후 제어 반환

## 파라미터 전달 방식

| 방식 | 설명 |
|------|------|
| 레지스터 | 파라미터를 레지스터에 직접 전달 |
| 스택 | 파라미터를 스택에 push |
| 메모리 | 메모리에 저장하고 포인터를 레지스터에 전달 |

## 예시

- 파일 읽기/쓰기
- 네트워크 통신
- 새 프로세스 생성
