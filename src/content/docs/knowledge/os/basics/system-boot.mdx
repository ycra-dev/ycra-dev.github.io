## 핵심 개념

커널을 로드하여 컴퓨터를 시작하는 과정으로, 부트스트랩 프로그램이 커널을 찾아 메모리에 적재하고 실행한다. 운영체제가 생성된 후 하드웨어가 커널 위치와 로드 방법을 알아야 한다.

## BIOS vs UEFI

| 특성 | BIOS | UEFI |
|------|------|------|
| 부팅 방식 | 다단계 (느림) | 단일 완전 부트 매니저 (빠름) |
| 64비트 지원 | 제한적 | 우수 |
| 대용량 디스크 | 제한적 | 우수한 지원 |
| 장점 | 레거시 호환성 | 속도, 현대적 하드웨어 지원 |

## 부트 블록과 MBR

- 고정 디스크 위치에 저장된 프로그램
- 단일 디스크 블록에 맞아야 하므로 단순한 코드
- 부트스트랩 프로그램의 나머지 부분의 디스크 주소와 길이만 알고 있음

- **UEFI 방식**: EFI 시스템 파티션에서 부트 로더를 직접 로드 (MBR 부트 코드 불필요)

## 부트스트랩 프로그램의 역할

- 커널 프로그램을 메모리에 로드
- 진단 실행: 메모리, CPU 상태 검사, 장치 검색
- 시스템 초기화: CPU 레지스터, 장치 컨트롤러, 메인 메모리 내용 등
- 운영체제 시작 및 루트 파일 시스템 마운트

## GRUB

Linux 및 UNIX 시스템용 **오픈소스 부트스트랩 프로그램**:
- GRUB 설정 파일에서 부트 파라미터 설정
- 유연성: 부팅 시 커널 파라미터 수정 가능, 다른 커널 선택 가능

예시 커널 파라미터 (`/proc/cmdline`):
```
BOOT_IMAGE=/boot/vmlinuz-4.4.0-59-generic
root=UUID=5f2e2232-4e47-4fe8-ae94-45ea749a5c92
```

## initramfs (초기 RAM 파일 시스템)

- 커널 로드 직후 사용되는 임시 루트 파일 시스템
- 실제 루트 파일 시스템 마운트에 필요한 드라이버 포함
- 루트 파일 시스템이 마운트되면 initramfs는 해제됨

## 모바일 시스템 부팅

- **LK** (little kernel): Android용 부트 로더
- **Android init**: 첫 번째 사용자 프로세스로 시작

## 트레이드오프

| 방식 | 장점 | 단점 |
|------|------|------|
| **BIOS** | 레거시 호환성 | 느림, 현대 하드웨어 지원 제한 |
| **UEFI** | 빠름, 64비트/대용량 디스크 지원 우수 | 복잡성, 보안 부트 설정 필요 |
