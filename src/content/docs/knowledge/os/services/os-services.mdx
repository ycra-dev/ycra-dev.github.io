## 핵심 개념

운영체제가 프로그램 실행을 위한 환경을 제공하며, 프로그램과 사용자에게 특정 서비스를 제공하는 것.

운영체제 서비스는 프로그래머의 작업을 더 쉽게 만들어 준다.

## 사용자에게 유용한 서비스

| 서비스 | 설명 |
|--------|------|
| **사용자 인터페이스 (UI)** | GUI, CLI, 터치스크린 인터페이스 제공 |
| **프로그램 실행** | 프로그램을 메모리에 적재하고 실행, 정상/비정상 종료 |
| **I/O 연산** | 파일이나 I/O 장치 관련 I/O 제공. 효율성과 보호를 위해 사용자가 직접 제어 불가 |
| **파일 시스템 조작** | 파일/디렉토리 읽기, 쓰기, 생성, 삭제, 검색, 권한 관리 |
| **통신** | 프로세스 간 정보 교환 - 공유 메모리 또는 메시지 전달 방식 |
| **오류 감지** | CPU, 메모리, I/O 장치, 사용자 프로그램의 오류를 지속적으로 감지하고 수정 |

## 시스템 효율을 위한 서비스

| 서비스 | 설명 |
|--------|------|
| **자원 할당** | 여러 프로세스 실행 시 자원(CPU 사이클, 메모리, 파일 저장소 등)을 각각에 할당 |
| **로깅** | 어떤 프로그램이 얼마나 많은 자원을 사용하는지 기록 (과금 또는 통계용) |
| **보호와 보안** | 시스템 자원 접근 제어, 외부로부터의 보안 유지, 사용자 인증 |

## 예시

사용자가 파일을 복사하려면:
1. 화면에 프롬프트 출력
2. 키보드 입력 읽기
3. 입력 파일 열기
4. 출력 파일 생성
5. 반복적으로 읽기/쓰기
6. 파일 닫기

→ 수많은 시스템 콜 필요

## 트레이드오프

- **장점**: 프로그래머의 작업 용이, 시스템 효율성 향상, 보안 및 보호 제공
- **단점**: 서비스 제공에 따른 오버헤드, 복잡성 증가
