## 핵심 개념

거의 모든 운영체제가 가지고 있는, 사용자가 운영체제와 상호작용할 수 있게 해주는 인터페이스.

## 세 가지 기본 인터페이스 유형

| 인터페이스 | 입력 방식 | 주요 사용 환경 |
|------------|----------|---------------|
| **CLI (명령줄)** | 키보드로 텍스트 명령 입력 | 서버, 시스템 관리 |
| **GUI** | 마우스, 키보드 | 데스크톱, 랩톱 |
| **터치스크린** | 손가락 제스처 | 스마트폰, 태블릿 |

## 인터페이스 선택

CLI vs GUI 선택은 대부분 개인 선호의 문제이다.

- **시스템 관리자와 파워 유저**: CLI 선호 - 더 효율적이고 빠른 접근 제공
- **일반 사용자**: GUI 선호 - 사용하기 쉽고 직관적
- 일부 시스템에서는 GUI로 일부 기능만 제공, 나머지는 CLI로만 접근 가능

## 실제 시스템 예시

| 시스템 | 인터페이스 |
|--------|-----------|
| **macOS** | Aqua GUI와 명령줄 인터페이스 모두 제공 |
| **Windows** | 데스크톱용 표준 GUI와 태블릿용 터치스크린 제공 |
| **iOS, Android** | CLI 앱이 존재하지만 거의 사용되지 않음, 대부분 터치스크린 사용 |

## 트레이드오프

| 인터페이스 | 장점 | 단점 |
|------------|-----|------|
| **GUI** | 사용하기 쉽고 직관적 | 리소스 소비가 큼 |
| **CLI** | 반복 작업에 효율적, 셸 스크립트로 자동화 가능 | 학습 곡선 존재 |
| **터치스크린** | 모바일 시스템에 적합 | 정밀한 입력이 어려움 |
