## 핵심 개념

스마트폰과 태블릿 컴퓨터에서 사용자가 터치스크린에 **제스처**(gestures)를 만들어 상호작용하는 인터페이스.

대부분의 모바일 시스템에서 명령줄 인터페이스나 마우스-키보드 시스템은 비실용적이다.

## 동작 원리

- 사용자가 터치스크린에서 제스처를 만들어 상호작용
- 제스처 예시: 손가락으로 화면을 누르거나 밀기
- 초기 스마트폰은 물리적 키보드를 포함했으나, 현재 대부분은 터치스크린에 키보드를 시뮬레이션

## 터치스크린 인터페이스 예시

| 시스템 | 터치스크린 인터페이스 |
|--------|----------------------|
| **iPhone, iPad** | Springboard 터치스크린 인터페이스 사용 |
| **Windows** | 태블릿용 터치스크린 제공 |

## 모바일 시스템의 인터페이스 사용

- iOS와 Android 모바일 시스템에는 명령줄 인터페이스를 제공하는 앱이 있으나 거의 사용되지 않음
- 대부분의 모바일 시스템 사용자는 터치스크린 인터페이스로 기기와 상호작용

## 트레이드오프

- **장점**:
  - 모바일 시스템에 적합
  - 직관적이고 물리적 키보드 불필요
- **단점**:
  - 정밀한 입력이 어려울 수 있음
  - 긴 텍스트 입력에 비효율적
