## 핵심 개념

분산 파일 시스템(DFS)은 클라이언트, 서버, 스토리지 장치가 분산 시스템의 여러 머신에 분산되어 있으면서 사용자에게는 단일 파일 시스템처럼 보이는 파일 서비스 시스템입니다. 중앙집중식 파일 서버의 단일 장애점과 병목 문제를 해결합니다.

## 동작 원리

### 기본 용어

| 용어 | 설명 |
|------|------|
| **Service** | 특정 기능을 클라이언트에 제공하는 소프트웨어 엔티티 |
| **Server** | 단일 머신에서 실행되는 서비스 소프트웨어 |
| **Client** | 클라이언트 인터페이스를 통해 서비스를 호출하는 프로세스 |

### 두 가지 아키텍처 모델

```
                        DFS
           _____________|_____________
          |                           |
   클라이언트-서버 모델          클러스터 기반 모델
   (NFS, OpenAFS)               (GFS, HDFS)
          |                           |
    투명한 파일 공유          대규모 병렬 처리
```

1. **클라이언트-서버 모델**: 투명한 파일 공유 (NFS, OpenAFS)
2. **클러스터 기반 모델**: 대규모 병렬 데이터 처리 (GFS, HDFS)

### 이름 지정 체계

| 체계 | 예시 | 설명 |
|------|------|------|
| host:path | ibis:/home/file | 위치 노출, 비투명 |
| NFS 마운트 | /mnt/nfs/path | 머신별 마운트 포인트 다를 수 있음 |
| 전역 네임스페이스 | /afs/org/path | OpenAFS, 모든 클라이언트 동일 |

### 성능 측정

서비스 요청 처리 시간 = 스토리지 접근 시간 + CPU 처리 시간 + **네트워크 오버헤드**. 이상적인 DFS는 로컬 파일 시스템과 유사한 성능을 목표로 합니다.

## 예시

분산 도서관 네트워크와 유사합니다. 어느 지점에서든 같은 카탈로그로 책을 검색하고 대출할 수 있으며, 실제 책은 여러 지점에 분산되어 있지만 사용자는 인식하지 못합니다.

```
사용자: /shared/project/report.txt 열기

[DFS 내부 처리]
1. 이름 해석: report.txt가 어느 서버에 있는지 확인
2. 인증/권한: 사용자 접근 권한 확인
3. 데이터 전송: 파일 내용을 클라이언트로 전달
4. 캐싱: 성능을 위해 로컬 캐시에 저장
```
