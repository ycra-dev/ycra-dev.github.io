## 핵심 개념

네트워크 운영체제(NOS)는 사용자가 원격 자원에 명시적으로 접근하는 환경을 제공하고, 분산 운영체제(DOS)는 원격 자원 접근을 로컬 자원처럼 투명하게 제공한다. 현재 모든 범용 OS(Windows, macOS, Linux, Android, iOS)는 NOS이다.

## 동작 원리

### 네트워크 운영체제 (NOS)

사용자가 **명시적으로** 원격 자원에 접근한다.

- **원격 로그인(SSH)**: 원격 시스템에 로그인하여 명령 실행
- **원격 파일 전송(FTP, SFTP)**: 파일을 명시적으로 다운로드/업로드
- **클라우드 스토리지**: 웹 인터페이스나 앱으로 파일 접근
- 사용자가 원격 시스템의 명령어를 알아야 함

### 분산 운영체제 (DOS)

사용자가 원격 자원을 **로컬처럼** 접근한다. 운영체제가 데이터/계산/프로세스 마이그레이션을 관리한다.

| 마이그레이션 유형 | 이동 대상 | 사용 시점 |
|----------|----------|----------|
| 데이터 마이그레이션 | 데이터 | 원격 데이터 접근 필요 시 |
| 계산 마이그레이션 | 연산 (RPC) | 데이터가 크고 계산이 작을 때 |
| 프로세스 마이그레이션 | 프로세스 전체 | 부하 분산, HW/SW 선호도 |

## 예시

- NOS = 다른 도시의 도서관에서 책을 대출하려면 직접 가거나 택배 요청해야 함
- DOS = 어느 도서관에 가든 모든 책이 있는 것처럼 보임 (시스템이 알아서 가져옴)

```bash
# NOS: 명시적 접근
ssh user@server.example.com
ls /home/user/documents
sftp user@server.example.com
get report.pdf

# DOS: 투명한 접근
open /network/server/documents/report.pdf
# 사용자는 파일이 어디 있는지 몰라도 됨
```
