## 핵심 개념

운영체제는 컴퓨터 하드웨어를 관리하고, 응용 프로그램의 실행 기반을 제공하며, 사용자와 하드웨어 사이의 중재자 역할을 하는 소프트웨어다.

> 운영체제는 정부(government)와 유사하다. 그 자체로는 유용한 기능을 수행하지 않지만, 다른 프로그램이 유용한 작업을 할 수 있는 환경을 제공한다.

## 왜 필요한가

- 하드웨어만으로는 사용하기 어려움
- 응용 프로그램들이 공통적으로 필요로 하는 I/O 제어 등의 기능이 있음
- 컴퓨터 시스템의 근본 목표: 프로그램을 실행하고 사용자 문제 해결을 쉽게 만드는 것

## 핵심 역할

컴퓨터 시스템은 4가지 구성요소로 나뉜다: **하드웨어 + 운영체제 + 응용 프로그램 + 사용자**

| 역할 | 설명 |
|------|------|
| **Resource Allocator** | CPU 시간, 메모리 공간, 저장 공간, I/O 장치 등 자원을 프로그램과 사용자에게 효율적이고 공정하게 할당 |
| **Control Program** | 사용자 프로그램의 실행을 관리하여 오류와 부적절한 사용을 방지 |

## 운영체제의 구성

| 계층 | 설명 |
|:---:|------|
| 응용 프로그램 | 사용자가 직접 사용하는 프로그램 |
| 미들웨어 | 모바일 OS에서 앱 개발자에게 추가 서비스 제공 |
| 시스템 프로그램 | OS와 연관되지만 커널의 일부는 아닌 프로그램 |
| **커널 (Kernel)** | 컴퓨터에서 항상 실행되는 핵심 프로그램 |
| 하드웨어 | CPU, 메모리, 저장장치, I/O 장치 등 |
