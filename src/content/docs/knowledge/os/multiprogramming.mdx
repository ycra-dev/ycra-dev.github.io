## 핵심 개념

멀티프로그래밍은 CPU가 항상 실행할 작업을 갖도록 여러 프로그램을 조직하여 CPU 활용률을 높이고 사용자를 만족시키는 운영체제의 중요한 측면이다.

> 단일 프로그램은 일반적으로 CPU나 I/O 장치를 항상 바쁘게 유지할 수 없다.

## 왜 필요한가

- 사용자는 일반적으로 한 번에 하나 이상의 프로그램을 실행하기를 원함
- 단일 프로그램만으로는 CPU가 I/O 대기 중 유휴 상태가 됨
- CPU 활용률을 최대화해야 함

## 동작 원리

1. 운영체제가 여러 프로세스를 메모리에 동시에 유지
2. 운영체제가 이 프로세스들 중 하나를 선택하여 실행 시작
3. 프로세스가 I/O 연산 같은 작업 완료를 기다려야 할 때:
   - **비-멀티프로그래밍 시스템**: CPU가 유휴 상태
   - **멀티프로그래밍 시스템**: 운영체제가 다른 프로세스로 전환
4. 최소 하나의 프로세스가 실행을 필요로 하는 한, CPU는 결코 유휴 상태가 되지 않음

## 멀티프로그래밍의 요구사항

| 요구사항 | 설명 |
|---------|------|
| **메모리 관리** | 여러 프로세스를 메모리에 동시에 유지 |
| **CPU 스케줄링** | 여러 프로세스가 실행 준비 시 다음 프로세스 선택 |
| **상호 영향 제한** | 여러 프로세스의 동시 실행 시 서로 영향 최소화 |
