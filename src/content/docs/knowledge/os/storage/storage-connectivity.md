## 핵심 개념

컴퓨터가 저장장치에 접근하는 방식은 크게 **DAS, NAS, SAN, Cloud**로 나뉜다. 각 방식은 접근 수준(블록/파일/객체), 네트워크 종류, 공유 가능 여부가 다르며, 용도에 따라 적합한 방식이 다르다.

쉽게 말해, DAS는 "내 컴퓨터에 꽂은 하드디스크", NAS는 "네트워크로 공유하는 파일 서버", SAN은 "전용 고속 네트워크로 연결된 디스크 풀", Cloud는 "인터넷으로 접근하는 저장소"이다.

## 동작 원리

### 연결 방식 비교

| 방식 | 접근 수준 | 네트워크 | 공유 | 주요 용도 |
|------|-----------|----------|------|-----------|
| **DAS (Host-Attached)** | 블록 수준 | 로컬 연결 (SATA/SAS/USB/NVMe) | 불가 | 단일 서버 |
| **NAS** | 파일 수준 | LAN (NFS/CIFS) | 가능 | 파일 공유 |
| **iSCSI** | 블록 수준 | LAN | 가능 | VM/DB |
| **SAN** | 블록 수준 | 전용 FC 네트워크 | 가능 | 엔터프라이즈 |
| **Cloud** | 객체/API 수준 | WAN (S3/Azure/GCS) | 가능 | 백업/앱 |

### DAS (Direct-Attached Storage)

저장장치가 호스트에 **직접 연결**된 방식이다. 가장 단순하고 빠르지만, 해당 호스트만 접근할 수 있다.

```
┌──────────┐  SATA/NVMe  ┌──────────┐
│  서버 A   │ ──────────→ │  디스크   │
└──────────┘              └──────────┘
```

### NAS (Network-Attached Storage)

네트워크 파일 서버로, **기존 LAN 인프라를 활용**하여 파일 수준의 공유 스토리지를 제공한다.

```
┌──────────┐              ┌──────────┐
│  서버 A   │ ─── LAN ──→ │          │
└──────────┘              │  NAS     │
┌──────────┐              │  서버    │
│  서버 B   │ ─── LAN ──→ │          │
└──────────┘              └──────────┘
  NFS/CIFS 프로토콜로 파일 수준 접근
```

서버는 NAS를 마치 네트워크 드라이브처럼 사용한다. 파일 수준 접근이므로 블록 수준 제어가 필요한 DB에는 적합하지 않다.

### SAN (Storage Area Network)

**전용 고속 네트워크(Fibre Channel)**로 스토리지를 연결하는 방식이다. 서버에게는 **로컬 디스크처럼** 보인다.

```
┌──────────┐              ┌──────────┐
│  서버 A   │ ─── FC ───→ │          │
└──────────┘              │  SAN     │
┌──────────┐              │  스위치   │ ──→ 디스크 어레이
│  서버 B   │ ─── FC ───→ │          │
└──────────┘              └──────────┘
  블록 수준 접근, 전용 네트워크
```

SAN은 일반 LAN과 분리된 전용 네트워크를 사용하므로 LAN 트래픽에 영향을 주지 않는다. 블록 수준 접근이 가능하여 DB, 가상화 등 고성능이 필요한 환경에 적합하다.

### Cloud Storage

인터넷(WAN)을 통해 **API 기반**으로 접근하는 저장소이다. 객체 수준(Object Storage)으로 접근하며, 사실상 무제한 확장이 가능하다.

## 예시

실무에서의 선택 기준:

```
개인 PC          → DAS (내장 SSD/HDD)
팀 파일 공유      → NAS (Synology, QNAP)
가상화 환경(VM)   → SAN 또는 iSCSI
데이터베이스       → SAN (고성능 블록 I/O)
백업/아카이브      → Cloud (S3, Azure Blob)
웹 애플리케이션    → Cloud (S3 + CDN)
```
