## 핵심 개념

파일 디스크립터(File Descriptor)는 열린 파일을 참조하는 음이 아닌 정수로, 프로세스가 파일 I/O를 수행하는 핸들입니다. `read()`/`write()` 시스템 콜 때마다 파일 경로를 전달하면 매번 경로 탐색이 필요하므로, 파일을 한 번 열면 이후에는 정수 인덱스로 빠르게 접근하는 추상화를 제공합니다.

## 동작 원리

각 프로세스는 자신만의 파일 디스크립터 테이블을 가집니다.

1. **`open()`**: 경로명으로 파일을 열고 파일 디스크립터 반환 (0부터 시작하는 작은 정수)
2. **세 단계 테이블 구조**:
   - **프로세스별 열린 파일 테이블**: fd → file 구조체 포인터
   - **시스템 전역 파일 테이블**: file 구조체 (파일 오프셋, 모드)
   - **in-core 아이노드 테이블**: 아이노드 정보 + 참조 카운트
3. **`read()`/`write()`**: fd를 사용해 현재 오프셋에서 I/O 수행, 오프셋 자동 갱신
4. **`close()`**: fd 해제, 참조 카운트 감소

```
프로세스 A                  시스템 전역
┌──────────────┐          ┌──────────────┐      ┌──────────────┐
│ fd 테이블     │          │ file 테이블   │      │ inode 테이블  │
├──────────────┤          ├──────────────┤      ├──────────────┤
│ 0 → ─────────┼─────────>│ offset: 100  │─────>│ /etc/passwd  │
│ 1 → ─────────┼───┐      │ mode: R      │      │ refcnt: 2    │
│ 2 → ─────────┼─┐ │      └──────────────┘      └──────────────┘
│ 3 → ─────────┼─┼─┼─────>│ offset: 0    │
└──────────────┘ │ │      │ mode: W      │
                 │ │      └──────────────┘
프로세스 B       │ │
┌──────────────┐ │ │
│ 0 → ─────────┼─┘ │ (fork 후 공유)
│ 1 → ─────────┼───┘
└──────────────┘
```

### 표준 파일 디스크립터

| fd | 이름 | 용도 |
|----|------|------|
| 0 | stdin | 표준 입력 |
| 1 | stdout | 표준 출력 |
| 2 | stderr | 표준 에러 |

리다이렉션 `ls > file`은 stdout(fd 1)을 file로 교체하는 것입니다.

## 예시

파일 디스크립터는 "티켓 번호"와 같습니다. 식당(커널)에 줄 서서 번호표(fd)를 받으면, 이후 서비스(I/O)를 받을 때 번호만 부르면 됩니다.

- 작은 정수로 빠른 인덱싱
- 디바이스, 파이프, 소켓 등도 동일 인터페이스로 처리 ("모든 것이 파일")
- `fork()` 시 자식에게 상속 → IPC 용이
- 단점: `fork()` 후 부모-자식 간 오프셋 공유 (의도치 않은 동작 가능), 프로세스당 fd 개수 제한
