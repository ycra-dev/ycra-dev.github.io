## 핵심 개념

CPU 스케줄링 알고리즘의 성능을 비교·평가하기 위한 정량적 지표들입니다. 시스템 목적(서버, 데스크톱, 실시간 등)에 따라 최적화할 기준이 다릅니다.

## 동작 원리

### 최대화해야 할 기준 (↑)

| 기준 | 정의 | 측정 |
|------|------|------|
| **CPU 이용률 (Utilization)** | CPU가 실제로 작업을 수행하는 비율 | 40~90% (실제 시스템) |
| **처리량 (Throughput)** | 단위 시간당 완료된 프로세스 수 | 프로세스/초 |

### 최소화해야 할 기준 (↓)

| 기준 | 정의 | 공식 |
|------|------|------|
| **반환 시간 (Turnaround Time)** | 프로세스 제출부터 완료까지 총 시간 | 완료 시간 - 도착 시간 |
| **대기 시간 (Waiting Time)** | Ready Queue에서 대기한 총 시간 | 반환 시간 - (CPU 시간 + I/O 시간) |
| **응답 시간 (Response Time)** | 제출부터 첫 응답까지 시간 | 첫 실행 시간 - 도착 시간 |

### 기준 간 관계

```
                제출        첫 응답      완료
                  │            │          │
시간축 ──────────┼────────────┼──────────┼──────▶
                  │◀──응답시간──▶│          │
                  │                        │
                  │◀────── 반환 시간 ──────▶│

반환 시간 = 대기 시간 + CPU 버스트 시간 + I/O 시간
```

### 시스템 유형별 우선순위

| 시스템 유형 | 중요 기준 | 이유 |
|-------------|-----------|------|
| **배치 시스템** | 처리량, CPU 이용률 | 최대한 많은 작업 처리 |
| **대화형 시스템** | 응답 시간 | 사용자 경험 |
| **실시간 시스템** | 데드라인 준수 | 예측 가능한 응답 |

## 예시

- CPU 이용률: 공장 기계가 돌아가는 시간 비율
- 처리량: 하루에 생산되는 제품 수
- 반환 시간: 주문부터 배송 완료까지 시간
- 대기 시간: 대기실에서 기다린 시간
- 응답 시간: 전화 걸고 상담원이 받을 때까지 시간

기준 간 충돌:
- **처리량 vs 응답 시간**: 문맥 교환을 줄이면 처리량↑ 하지만 응답 시간↑
- **CPU 이용률 vs 응답 시간**: CPU를 최대로 쓰면 대기 프로세스 응답↓
- 평균값뿐 아니라 **분산**도 중요 — 대화형 시스템에서는 일관된 응답 시간이 더 중요
