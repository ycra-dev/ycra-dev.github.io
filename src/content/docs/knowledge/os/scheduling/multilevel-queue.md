## 핵심 개념

다단계 큐 스케줄링은 프로세스를 특성에 따라 여러 개의 별도 큐로 분류하고, 각 큐에 다른 스케줄링 알고리즘을 적용하는 방식입니다. 프로세스 유형(시스템, 대화형, 배치 등)마다 요구사항이 다르므로 차별화된 스케줄링이 필요합니다.

## 동작 원리

프로세스는 **영구적으로** 하나의 큐에 할당됩니다 (큐 간 이동 없음).

### 일반적인 큐 구성 (높은 → 낮은 우선순위)

```
[실시간 큐]      → 절대 우선순위, 선점형
[시스템 큐]      → 우선순위 기반
[대화형 큐]      → Round-Robin (q=20ms)
[배치 큐]        → FCFS
```

### 큐 간 스케줄링 방식

1. **고정 우선순위 선점형**: 상위 큐가 비어야 하위 큐 실행
2. **시간 분할(Time Slicing)**: 각 큐에 CPU 시간 비율 할당 (예: 전경 80%, 배경 20%)

## 예시

공항 탑승과 같습니다. 퍼스트클래스, 비즈니스, 이코노미 별도 줄로, 각각 다른 대기 방식을 적용합니다.

- 대화형 프로세스가 도착하면 배치 프로세스는 즉시 선점됨
- 배치 프로세스는 상위 큐가 모두 비어야 실행됨

- 장점: 프로세스 특성에 맞는 차별화된 스케줄링, 효율적 우선순위 탐색
- 단점: 비유연성(큐 간 이동 불가), 하위 큐에서 기아(Starvation) 발생 가능
