## 핵심 개념

Linux 시스템은 **커널, 시스템 라이브러리, 시스템 유틸리티**의 3계층으로 구성된 **모놀리식(monolithic)** 운영체제이다. 성능(특히 커널 내부 통신 속도)이 가장 중요한 설계 목표이며, 적재 가능 커널 모듈(LKM)을 통해 유연성을 확보한다.

## 동작 원리

### 3계층 구조

```
┌─────────────────────────────────────────┐
│        사용자 프로세스 / 유틸리티           │  User Mode
├─────────────────────────────────────────┤
│           시스템 공유 라이브러리            │
├─────────────────────────────────────────┤
│              Linux 커널                  │  Kernel Mode
│  ┌─────────────────────────────────┐    │
│  │     Loadable Kernel Modules     │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

1. **커널(Kernel)**: 모든 코드가 단일 주소 공간에서 실행. 커널 모드(privileged mode)에서 하드웨어에 직접 접근
2. **시스템 라이브러리**: 사용자 모드에서 실행. 시스템 콜 인터페이스 제공 (예: libc)
3. **시스템 유틸리티**: 개별 관리 작업 수행 프로그램. 데몬(daemon) 포함

비유하면, 커널은 회사의 **핵심 경영진**(모든 자원에 접근 가능), 시스템 라이브러리는 **접수 창구**(사용자 요청을 커널에 전달), 유틸리티는 **각 부서 직원**(특정 업무 담당)이다.

## 예시

파일 읽기 요청 흐름:
1. 사용자 프로그램이 libc의 `read()` 호출
2. libc가 시스템 콜 발생
3. 커널이 실제 I/O 수행 후 결과 반환
