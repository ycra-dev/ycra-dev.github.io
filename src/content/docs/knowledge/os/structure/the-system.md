## 핵심 개념

THE 시스템은 운영체제를 계층(layer)으로 분리하여 설계하고, 동시 프로세스 간 동기화에 세마포어를 사용한 최초의 시스템입니다. 1960년대 중반 네덜란드 Technische Hogeschool Eindhoven에서 Dijkstra가 설계했습니다.

## 동작 원리

### 설계 원칙

시스템을 독립적인 계층으로 분리하면 각 계층을 개별적으로 검증할 수 있습니다.

1. **정적 프로세스 집합**: 동적 생성 없이 고정된 프로세스 사용
2. **협력 프로세스**: OS 자체가 협력하는 프로세스 집합으로 설계
3. **세마포어 동기화**: 프로세스 간 동기화에 세마포어 사용
4. **5개 사용자 프로세스**: 사용자 프로그램을 컴파일, 실행, 출력하는 에이전트 역할

### CPU 스케줄링

- 우선순위 기반 스케줄링
- 2초마다 우선순위 재계산
- 최근 8~10초간 CPU 사용량에 반비례
- 결과: I/O-bound 프로세스와 새 프로세스에 높은 우선순위

### 메모리 관리

- Algol 전용 시스템이므로 소프트웨어 페이징 사용
- 컴파일러가 자동으로 시스템 루틴 호출 코드 생성
- 512워드 페이지, LRU 교체 알고리즘
- 백업 스토어: 512KB 드럼

### 교착상태 제어

- Banker's Algorithm으로 교착상태 회피(deadlock avoidance) 구현

## 예시

건물의 층 구조와 유사합니다. 1층(하드웨어)부터 최상층(사용자)까지, 각 층은 아래 층의 서비스만 사용합니다.

### Venus 시스템과의 비교

| 구분 | THE | Venus |
|------|-----|-------|
| 구조 | 레이어 설계 | 레이어 설계 |
| 동기화 | 세마포어 | 세마포어 |
| 하위 레벨 | 소프트웨어 | 마이크로코드 (더 빠름) |
| 메모리 | 페이징 | 페이지드 세그먼테이션 |
| 용도 | 배치 시스템 | 시분할 시스템 |
