## 핵심 개념

커널의 모든 기능을 **단일 정적 바이너리 파일**에 넣고, **단일 주소 공간**에서 실행하는 운영체제 구조이다. 가장 단순한 구조이며, 원래 UNIX 운영체제의 구조였다.

## Linux 구조

Linux는 UNIX 기반이며 유사한 구조를 가진다:
- 애플리케이션은 **glibc** 표준 C 라이브러리를 사용하여 시스템 콜 인터페이스와 통신
- **모놀리식**: 전체가 커널 모드에서 단일 주소 공간으로 실행
- **모듈식 설계**: 런타임에 커널 수정 가능

## Tightly Coupled System

모놀리식 접근법은 **밀접하게 결합된 시스템**(tightly coupled system)이다:
- 한 부분의 변경이 다른 부분에 광범위한 영향을 미칠 수 있음

## 트레이드오프

| 장점 | 단점 |
|------|------|
| 시스템 콜 인터페이스 오버헤드가 매우 적음 | 구현이 어려움 |
| 커널 내 통신이 빠름 | 확장이 어려움 |
| 성능상의 뚜렷한 이점 | 한 부분의 변경이 시스템 전체에 영향 |
