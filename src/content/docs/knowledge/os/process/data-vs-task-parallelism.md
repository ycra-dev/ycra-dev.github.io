## 핵심 개념

데이터 병렬성은 동일 연산을 데이터 부분집합에 분산 적용하고, 태스크 병렬성은 서로 다른 연산(태스크)을 여러 코어에 분산 실행하는 것입니다. 병렬화를 설계할 때 "무엇을 분할할 것인가"에 대한 전략이 성능을 좌우합니다.

## 동작 원리

### 데이터 병렬성 (Data Parallelism)

같은 연산을 데이터의 부분집합에 적용합니다. SIMD 패러다임과 유사합니다.

```
배열 [1,2,3,4,5,6,7,8] 의 합 계산 (4코어):
Core 0: sum([1,2]) = 3
Core 1: sum([3,4]) = 7    → 최종: 3+7+11+15 = 36
Core 2: sum([5,6]) = 11
Core 3: sum([7,8]) = 15
```

### 태스크 병렬성 (Task Parallelism)

서로 다른 연산을 여러 코어에 분산합니다.

```
동일 배열에 대해 서로 다른 통계 계산 (4코어):
Core 0: 평균 계산
Core 1: 분산 계산     → 각각 다른 연산 수행
Core 2: 최댓값 찾기
Core 3: 정렬
```

### 비교

| 구분 | 데이터 병렬성 | 태스크 병렬성 |
|------|---------------|---------------|
| 분할 대상 | 데이터 | 태스크(연산) |
| 각 코어의 작업 | 동일한 연산 | 서로 다른 연산 |
| 적합한 문제 | 대용량 데이터, 행렬 연산 | 독립적 작업 집합, 파이프라인 |
| 부하 균형 | 데이터 균등 분할 시 자연스러움 | 태스크 수행 시간 차이로 불균형 가능 |

## 예시

- **데이터 병렬성**: 100개의 사과를 4명이 나눠서 깎는다 (같은 작업, 데이터 분할)
- **태스크 병렬성**: 한 명은 사과 깎고, 한 명은 반죽 만들고, 한 명은 오븐 예열 (다른 작업 동시 수행)

실제 애플리케이션에서는 두 방식을 혼합합니다. 전체 파이프라인은 태스크 병렬성으로, 각 태스크 내부는 데이터 병렬성으로 처리할 수 있습니다.
