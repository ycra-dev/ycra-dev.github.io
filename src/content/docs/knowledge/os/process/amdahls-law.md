## 핵심 개념

암달의 법칙(Amdahl's Law)은 **프로그램에서 병렬화할 수 없는 순차적 부분이 전체 성능 향상의 상한을 결정한다**는 법칙이다. 1967년 Gene Amdahl이 발표했다.

비유하자면, 10명이 함께 청소를 한다고 해도 **좁은 복도는 한 명씩만 지나갈 수 있다**. 아무리 사람을 늘려도 복도 구간의 시간은 줄어들지 않는다. 이 "복도 구간"이 바로 순차적 부분이며, 전체 속도 향상의 **하한을 결정**한다.

## 동작 원리

### 공식

```
                    1
Speedup  ≤  ─────────────────
             S + (1 - S) / N
```

- **S**: 프로그램에서 반드시 순차적으로 실행해야 하는 비율 (0 ≤ S ≤ 1)
- **N**: 코어(프로세서) 수
- **1 - S**: 병렬화 가능한 비율

### N → ∞ 일 때

코어 수를 무한히 늘리면 `(1-S)/N`이 0에 수렴하므로:

```
Speedup_max = 1 / S
```

즉, 순차 비율이 5%라면 아무리 코어를 늘려도 **최대 20배**까지만 빨라진다.

### 성능 향상 계산표

| 순차 비율 (S) | N=2 | N=4 | N=8 | N=16 | N=∞ (상한) |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **5%** | 1.90x | 3.48x | 5.93x | 9.14x | **20.00x** |
| **10%** | 1.82x | 3.08x | 4.71x | 6.40x | **10.00x** |
| **25%** | 1.60x | 2.29x | 2.91x | 3.37x | **4.00x** |
| **50%** | 1.33x | 1.60x | 1.78x | 1.88x | **2.00x** |

표에서 볼 수 있듯이:
- 순차 비율이 높을수록 코어를 아무리 추가해도 효과가 미미하다.
- 순차 비율이 50%이면 코어를 16개로 늘려도 고작 1.88배 향상에 그친다.
- 순차 비율이 5%라면 16코어에서 약 9배 향상이 가능하다.

### 핵심 통찰

**코어 수를 늘리는 것보다 순차 부분을 줄이는 것이 더 효과적이다.**

예를 들어 순차 비율 25%인 프로그램을:
- 코어 4→16개로 늘리면: 2.29x → 3.37x (47% 추가 향상)
- 순차 비율 25%→10%로 줄이면 (4코어): 2.29x → 3.08x (35% 추가 향상, 코어 추가 없이!)

### Gustafson의 법칙과 대비

암달의 법칙은 **문제 크기가 고정**이라고 가정한다. 하지만 현실에서는 코어가 늘어나면 **더 큰 문제를 풀려고** 한다.

| | 암달의 법칙 | Gustafson의 법칙 |
|------|------------|-----------------|
| **가정** | 문제 크기 고정 | 문제 크기가 코어 수에 비례하여 증가 |
| **관점** | 고정된 작업의 속도 향상 | 고정된 시간에 더 많은 작업 처리 |
| **결론** | 병렬화 한계 강조 | 병렬화 이점 강조 |

## 예시

웹 서버에서 요청 처리를 생각해보자:

- **순차 부분(S = 10%)**: 요청 파싱, DB 커넥션 초기화, 응답 직렬화
- **병렬화 가능 부분(90%)**: 데이터 처리, 비즈니스 로직

```
4코어:  Speedup = 1 / (0.1 + 0.9/4) = 1 / 0.325 ≈ 3.08x
8코어:  Speedup = 1 / (0.1 + 0.9/8) = 1 / 0.2125 ≈ 4.71x
16코어: Speedup = 1 / (0.1 + 0.9/16) = 1 / 0.15625 ≈ 6.40x
∞코어:  Speedup = 1 / 0.1 = 10x (상한)
```

8코어에서 16코어로 2배를 투자해도 4.71x → 6.40x, 약 36%밖에 개선되지 않는다. 이것이 "코어를 무한정 늘리는 것은 비효율적"이라는 암달의 법칙의 교훈이다.
