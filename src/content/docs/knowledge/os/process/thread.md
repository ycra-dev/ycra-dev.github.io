## 핵심 개념

스레드(Thread)는 CPU 이용의 기본 단위로, 프로세스 내에서 실행되는 독립적인 제어 흐름입니다. 단일 스레드 프로세스는 한 번에 하나의 작업만 수행 가능하여, 동시에 여러 요청을 처리해야 하는 환경에서 비효율적입니다. 프로세스 생성은 비용이 크므로(메모리, 자원 할당), 같은 작업을 수행하는 여러 실행 흐름이 필요할 때 스레드를 사용합니다.

## 동작 원리

### 스레드의 구성 요소

각 스레드마다 고유한 요소:
- **Thread ID**: 스레드 식별자
- **Program Counter (PC)**: 현재 실행 중인 명령어 위치
- **Register Set**: CPU 레지스터 상태
- **Stack**: 함수 호출, 지역 변수 저장

프로세스 내 공유 자원 (모든 스레드가 공유):
- Code Section (코드 영역)
- Data Section (전역 변수)
- Open Files, Signals 등 OS 자원

```
┌─────────────────────────────────────────┐
│              Process                     │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│  │ Thread1 │ │ Thread2 │ │ Thread3 │    │
│  │   PC    │ │   PC    │ │   PC    │    │
│  │ Regs    │ │ Regs    │ │ Regs    │    │
│  │ Stack   │ │ Stack   │ │ Stack   │    │
│  └────┬────┘ └────┬────┘ └────┬────┘    │
│       └───────────┴───────────┘          │
│                   │                      │
│     ┌─────────────┴─────────────┐        │
│     │   공유: Code, Data, Files  │        │
│     └───────────────────────────┘        │
└─────────────────────────────────────────┘
```

### 멀티스레딩의 이점

| 이점 | 설명 |
|------|------|
| **응답성(Responsiveness)** | 일부 스레드가 블로킹되어도 다른 스레드가 계속 실행 |
| **자원 공유(Resource Sharing)** | 같은 주소 공간 내에서 코드/데이터 공유, 별도 IPC 불필요 |
| **경제성(Economy)** | 프로세스보다 생성/컨텍스트 스위칭 비용이 적음 |
| **확장성(Scalability)** | 멀티코어에서 병렬 실행으로 성능 향상 |

## 예시

한 사무실(프로세스)에서 일하는 여러 직원(스레드)과 같습니다. 같은 사무실 공간, 복사기, 서류함(공유 자원)을 함께 사용하지만, 각자 자기 책상(스택)과 업무 진행 상황(PC, 레지스터)은 별도로 관리합니다.

- 웹 브라우저: 한 스레드는 이미지 표시, 다른 스레드는 네트워크 데이터 수신
- 워드 프로세서: 화면 표시, 키 입력 처리, 맞춤법 검사를 각각 별도 스레드로 처리

- 장점: 프로세스보다 빠르고 메모리 효율적, 통신 오버헤드 감소, 병렬 처리 가능
- 단점: 공유 데이터 동기화 문제(Race Condition), 하나의 스레드 오류가 전체 프로세스에 영향, 디버깅 복잡
