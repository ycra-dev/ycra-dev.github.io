## 핵심 개념

쿠버네티스는 컴퓨터 클러스터를 위한 운영체제와 같은 역할을 수행한다. 개별 서버의 하드웨어 세부사항을 추상화하여 전체 클러스터를 하나의 거대한 배포 영역으로 제공한다. 이를 통해 개발자는 인프라 세부사항에 신경 쓰지 않고 애플리케이션 개발에 집중할 수 있다.

쿠버네티스가 제공하는 핵심 기능은 다음과 같다:
- **서비스 디스커버리**: 애플리케이션이 다른 애플리케이션을 찾아 통신할 수 있게 해주는 메커니즘
- **수평적 확장(Horizontal Scaling)**: 부하 변동에 따라 애플리케이션 복제본을 조절
- **로드 밸런싱**: 모든 애플리케이션 복제본에 부하를 분산
- **자가 치유(Self-healing)**: 장애 발생 시 자동으로 애플리케이션을 재시작하고 건강한 노드로 이동
- **리더 선출(Leader Election)**: 활성 인스턴스를 결정하는 메커니즘

쿠버네티스는 선언적(declarative) 모델을 사용하여 애플리케이션을 정의하고 배포한다. 사용자가 원하는 상태를 기술하면 쿠버네티스가 이를 실현하고 유지한다.

주요 관리형 쿠버네티스 서비스:
- Google Kubernetes Engine (GKE)
- Amazon Elastic Kubernetes Service (EKS)
- Azure Kubernetes Service (AKS)
