## 핵심 개념

알고리즘은 입력을 출력으로 변환하는 일련의 계산 단계(computational steps)로 구성된다. 잘 정의된 계산 문제(computational problem)를 해결하는 도구로 볼 수 있으며, 문제의 명세(specification)는 원하는 입출력 관계를 일반적 용어로 기술한다.

알고리즘은 영어, 컴퓨터 프로그램, 하드웨어 설계 등 다양한 형태로 명세할 수 있다. 유일한 요구사항은 따라야 할 계산 절차를 정확하게 기술해야 한다는 것이다.

알고리즘은 하드웨어, GUI, 네트워킹과 같은 하나의 기술(technology)로 간주해야 한다. 시스템의 전체 성능은 빠른 하드웨어 선택만큼이나 효율적인 알고리즘 선택에 달려 있다.

## 예시

정렬 문제(Sorting Problem)는 알고리즘의 대표적 예시이다:

```
입력: n개의 수열 ⟨a₁, a₂, ..., aₙ⟩
출력: 입력의 순열(재배열) ⟨a'₁, a'₂, ..., a'ₙ⟩ (단, a'₁ ≤ a'₂ ≤ ... ≤ a'ₙ)

예: 입력 ⟨31, 41, 59, 26, 41, 58⟩
    출력 ⟨26, 31, 41, 41, 58, 59⟩
```
