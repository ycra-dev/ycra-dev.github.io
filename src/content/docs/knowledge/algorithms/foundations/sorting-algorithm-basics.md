## 핵심 개념

정렬 알고리즘은 데이터의 모음을 특정 기준(숫자 크기, 알파벳 순서 등)에 따라 순서대로 배열하는 알고리즘이다. 정렬은 이진 탐색 등 효율적인 알고리즘의 전제 조건이며, 알고리즘에 따라 성능이 크게 달라진다.

## 동작 원리

### 선택 정렬 (Selection Sort) - O(N²)

1. 전체에서 가장 작은 값을 찾아 첫 번째 위치에 놓는다.
2. 나머지에서 가장 작은 값을 찾아 두 번째 위치에 놓는다.
3. 이 과정을 반복한다.

N개의 항목을 정렬할 때 약 N×N/2 번의 비교가 필요하므로 O(N²)이다. 이해하기 쉽지만 데이터가 많아지면 매우 느리다.

### 퀵소트 (Quicksort) - O(N log N)

1. 하나의 기준값(pivot)을 선택한다.
2. 기준값보다 작은 것은 왼쪽, 큰 것은 오른쪽으로 분류한다.
3. 각 그룹에 대해 재귀적으로 같은 과정을 반복한다.

평균적으로 O(N log N)의 시간이 걸리며, 이는 비교 기반 정렬의 이론적 하한이기도 하다.

**성능 차이의 실제적 의미**:

| 데이터 수 (N) | 선택 정렬 (N²) | 퀵소트 (N log N) |
|---|---|---|
| 1,000 | 1,000,000 | 10,000 |
| 1,000,000 | 1,000,000,000,000 | 20,000,000 |

100만 개의 데이터에서 퀵소트는 선택 정렬보다 약 5만 배 빠르다.

## 예시

선택 정렬 과정 (배열: [64, 25, 12, 22, 11]):

```
[64, 25, 12, 22, 11]  → 최소값 11 찾음
[11, 25, 12, 22, 64]  → 나머지에서 최소값 12 찾음
[11, 12, 25, 22, 64]  → 나머지에서 최소값 22 찾음
[11, 12, 22, 25, 64]  → 나머지에서 최소값 25 찾음
[11, 12, 22, 25, 64]  → 정렬 완료
```

퀵소트 과정 (배열: [64, 25, 12, 22, 11], 피벗=22):

```
피벗 22 선택
왼쪽(< 22): [12, 11]    오른쪽(> 22): [64, 25]
  [11, 12]  ← 재귀 정렬    [25, 64] ← 재귀 정렬
결과: [11, 12] + [22] + [25, 64] = [11, 12, 22, 25, 64]
```
