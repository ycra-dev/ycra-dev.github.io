## 핵심 개념

적절한 자료구조의 선택은 알고리즘 설계의 중요한 부분이다. 모든 목적에 적합한 단일 자료구조는 존재하지 않으므로, 여러 자료구조의 장점과 한계를 알아야 한다.

자료구조는 알고리즘과 밀접하게 연관된다:
- 알고리즘의 효율성은 사용하는 자료구조에 크게 의존한다
- 같은 문제라도 자료구조 선택에 따라 시간 복잡도가 극적으로 달라질 수 있다
- 자료구조 자체가 알고리즘의 핵심 구성요소가 되기도 한다

이 책에서 다루는 주요 자료구조는 배열, 연결 리스트, 스택, 큐, 해시 테이블, 이진 탐색 트리, 레드-블랙 트리, B-트리 등이 있다.

## 예시

```
자료구조 선택의 영향 예시:

검색 연산의 시간 복잡도:
  - 정렬되지 않은 배열: O(n)
  - 정렬된 배열 (이진 탐색): O(lg n)
  - 해시 테이블: O(1) 평균
  - 이진 탐색 트리: O(lg n) 평균

삽입 연산의 시간 복잡도:
  - 배열 끝에 추가: O(1)
  - 정렬된 배열에 삽입: O(n)
  - 해시 테이블: O(1) 평균
  - 이진 탐색 트리: O(lg n) 평균
```
