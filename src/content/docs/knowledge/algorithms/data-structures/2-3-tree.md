## 핵심 개념

2-3 트리는 각 내부 노드가 2개 또는 3개의 자식을 가질 수 있는 균형 탐색 트리로, 차수 m=3인 B-트리의 특수 경우다. 모든 리프 노드가 동일한 레벨에 있으며, 삽입과 삭제 후에도 자동으로 균형이 유지된다.

**노드 구조:**
- 2-노드: 키 1개, 자식 2개 (왼쪽 < 키 ≤ 오른쪽)
- 3-노드: 키 2개, 자식 3개 (왼쪽 < 키₁ ≤ 중간 < 키₂ ≤ 오른쪽)

**핵심 불변식:**
- 모든 리프 노드는 동일한 깊이에 위치
- 탐색 트리 성질 유지 (대칭 순서)

**성능:**
- 탐색, 삽입, 삭제 모두 O(log N)
- AVL 트리보다 구현이 직관적이며 회전 규칙이 단순

## 동작 원리

**삽입 알고리즘:**
1. 탐색으로 삽입 위치(리프)를 찾음
2. 리프 노드가 2-노드면: 3-노드로 변환 (쉬움)
3. 리프 노드가 3-노드면: 분할(split)
   - 3개의 키 중 중간값을 부모로 올림
   - 노드를 두 개의 2-노드로 분할
   - 부모도 3-노드였다면 재귀적으로 분할
   - 루트가 분할되면 새 루트 생성 → 트리 높이 증가

**삭제 알고리즘:**
1. 삭제할 키가 내부 노드라면 인오더 후계자(또는 전임자)와 교환 후 삭제
2. 리프 노드가 3-노드: 단순 삭제 후 2-노드가 됨
3. 리프 노드가 2-노드: 형제 노드에서 빌리거나(rotation) 형제와 병합(merge)
   - 병합 시 부모에서 키 내려옴 → 부모가 2-노드였다면 재귀 처리

**AVL 트리와의 비교:**

| 특성 | AVL 트리 | 2-3 트리 |
|------|---------|---------|
| 구현 복잡도 | 높음 | 중간 |
| 노드 타입 | 단일 | 2-노드/3-노드 |
| 분기 수 | 최대 2 | 최대 3 |
| 균형 방법 | 회전 | 분할/병합 |

## 예시

```
2-3 트리 삽입: 5, 3, 7, 1, 4, 6, 8 순서로 삽입

삽입 5: [5]
삽입 3 (3-노드 형성): [3 | 5]
삽입 7 → 분할, 5를 루트로:
    [5]
   /   \
[3]   [7]

삽입 1 (3-노드): [1|3] [7]
삽입 4 → 왼쪽 분할, 3을 루트로:
      [3 | 5]
     /   |   \
   [1] [4]  [7]

삽입 6 → 3-노드:
      [3 | 5]
     /   |   \
   [1] [4] [6|7]

삽입 8 → 오른쪽 분할, 7을 루트로:
       [3 | 5 | 7]  ← 루트가 3-노드
      /   |   |  \
   [1]  [4] [6] [8]
```
