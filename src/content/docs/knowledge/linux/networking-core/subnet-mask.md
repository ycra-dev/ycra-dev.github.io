## 핵심 개념

서브넷 마스크를 사용하면 하나의 큰 네트워크 주소를 여러 개의 작은 서브넷으로 분할할 수 있다. 예를 들어 Class B 주소(기본 마스크 255.255.0.0)에 255.255.255.0 마스크를 적용하면 256개의 Class C 규모 네트워크로 나눌 수 있다.

서브넷 마스크는 두 가지 표기법이 있다:
- **점 십진법**: 255.255.255.192
- **CIDR 표기법**: /26 (네트워크 비트 수)

서브넷 계산의 실용적 공식:
- `마지막 넷마스크 바이트 = 256 - 네트워크 크기`
- `/26` = 64개 주소(62개 사용 가능, 네트워크 주소와 브로드캐스트 주소 제외)
- 실제 네트워크 주소의 마지막 바이트는 네트워크 크기로 나누어 떨어져야 함

최소 의미 있는 네트워크는 /30(4개 주소, 실제 호스트 2개)이지만, /31은 특수한 경우(RFC3021)로 네트워크/브로드캐스트 주소 없이 2개 호스트를 모두 사용할 수 있다. 같은 네트워크의 모든 호스트는 반드시 동일한 넷마스크를 사용해야 한다.

## 예시

```bash
# 서브넷 마스크 예시
# /24 = 255.255.255.0   -> 256주소, 254호스트
# /25 = 255.255.255.128  -> 128주소, 126호스트
# /26 = 255.255.255.192  -> 64주소, 62호스트
# /27 = 255.255.255.224  -> 32주소, 30호스트
# /30 = 255.255.255.252  -> 4주소, 2호스트 (point-to-point)

# 128.138.243.0/26 네트워크의 4개 서브넷:
# 128.138.243.0/26   (0-63)
# 128.138.243.64/26  (64-127)
# 128.138.243.128/26 (128-191)
# 128.138.243.192/26 (192-255)

# ipcalc로 서브넷 계산
ipcalc 128.138.243.100/26

# IP 주소 설정 시 넷마스크 지정
sudo ip address add 192.168.1.10/24 dev eth0

# bc를 이용한 진법 변환
echo "obase=2; 192" | bc
# 출력: 11000000
```
