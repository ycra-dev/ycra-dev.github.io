## 핵심 개념

전통적인 IP 클래스 시스템(A/B/C)은 고정된 바이트 경계에서만 네트워크를 분할할 수 있어 주소 낭비가 심했다. CIDR는 이 제한을 제거하고 네트워크 부분을 클래스가 암시하는 것보다 더 작게 만들 수 있어, 여러 네트워크를 하나의 라우팅 항목으로 통합(aggregation)할 수 있다.

CIDR의 핵심 이점:
1. **라우팅 정보 간소화**: 여러 네트워크를 하나의 라우팅 항목으로 집약
2. **유연한 주소 할당**: 다양한 크기의 네트워크를 자유롭게 정의
3. **주소 공간 절약**: 실제 필요에 맞는 크기로 네트워크 할당

예를 들어, 192.144.0.0/21 블록은 내부적으로 /24 8개, /25 16개, /26 32개 등으로 다양하게 분할 가능하지만, 인터넷 백본에서는 단일 /21 라우팅 항목으로 충분하다. 이를 **가변 길이 서브네팅(Variable Length Subnetting)**이라 한다.

CIDR는 IPv6에서도 IPv4 이후로 계속 사용되는 핵심 기법이다. IPv6의 계층적 프리픽스 할당은 CIDR가 성공적으로 입증한 원리를 기반으로 한다.

## 예시

```bash
# CIDR 표기법
# 192.168.1.0/24    -> 넷마스크 255.255.255.0
# 10.0.0.0/8        -> 넷마스크 255.0.0.0
# 192.144.0.0/21    -> 넷마스크 255.255.248.0

# 192.144.0.0/21 블록의 유연한 분할:
# 1개 /21 네트워크: 2046 호스트
# 8개 /24 네트워크: 각 254 호스트
# 16개 /25 네트워크: 각 126 호스트
# 32개 /26 네트워크: 각 62 호스트

# ISP의 가변 길이 서브네팅 예시
# /30 - 포인트 투 포인트 고객
# /24 - 대형 고객
# /27 - 소규모 고객

# CIDR 표기법으로 라우팅 추가
sudo ip route add 192.168.45.128/25 via 10.0.0.1

# CIDR 계산
ipcalc 192.144.0.0/21
```
