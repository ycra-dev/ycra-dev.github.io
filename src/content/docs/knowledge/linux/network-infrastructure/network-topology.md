## 핵심 개념

**네트워크 설계의 핵심 요소:**
- 사용할 매체 유형 (UTP, 광섬유, 무선)
- 케이블의 토폴로지와 경로
- 스위치와 라우터의 사용

**계층적 네트워크 구조:**
- 건물 간 백본 네트워크
- 백본에 연결된 부서 서브넷
- 부서 내 그룹 서브넷
- 외부 세계 연결 (인터넷, VPN)

**혼잡 제어**: 네트워크는 체인처럼 가장 약한 링크에 의해 성능이 결정된다. 이더넷의 성능은 부하가 증가하면 비선형적으로 저하된다. 서브넷을 생성하고 라우터로 로컬 트래픽을 격리하여 혼잡을 완화한다.

**물리적 제약**: 네트워크 설계는 건물 구조와 공존해야 한다. 방화벽(물리적)의 무결성을 유지하고, 환기 덕트에서는 적절한 케이블 피복을 사용해야 한다. 논리적으로 좋은 설계가 물리적으로 구현 불가능할 수 있으므로 사전에 확인이 필요하다.

**문서화**: 네트워크 유지보수성은 문서화 품질과 높은 상관관계를 가진다. 모든 종단점에서 케이블을 라벨링하고, 통신 클로짓에 로컬 케이블 맵을 게시하며, 변경 사항을 즉시 업데이트해야 한다.

**중앙 관리 대상**: 네트워크 설계, 백본 네트워크, 호스트 IP 주소/호스트명, 프로토콜, 인터넷 라우팅 정책은 사이트 전체 차원의 중앙 통제가 필요하다.

## 예시

```bash
# 일반적인 엔터프라이즈 네트워크 토폴로지
#
# [인터넷] --- [경계 라우터] --- [코어 스위치]
#                                    |
#              +-----+------+--------+--------+
#              |     |      |        |        |
#           [서버] [부서A] [부서B] [게스트] [관리]
#           VLAN10 VLAN20  VLAN30  VLAN99   VLAN1

# 사무실당 권장 네트워크 연결 수: 2개
# - 유선 VoIP 전화기 등 특수 장비용
# - 무선으로 대부분의 사용자 장비 연결

# EIA-606 케이블 색상 코드
# 주황색 - 중앙 사무실 분기점
# 녹색   - 네트워크 연결
# 보라색 - 전화 시스템
# 파란색 - 워크스테이션 연결
```
