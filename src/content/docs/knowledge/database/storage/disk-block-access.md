## 핵심 개념

디스크 I/O 요청은 디스크 식별자와 논리적 블록 번호를 지정한다. 블록 크기는 일반적으로 4~16 킬로바이트이다. 순차 접근 패턴과 랜덤 접근 패턴의 성능 차이가 크므로, 블록 접근 속도를 개선하기 위한 여러 기법이 개발되었다.

주요 최적화 기법:

1. **버퍼링(Buffering)**: 디스크에서 읽은 블록을 인메모리 버퍼에 임시 저장하여 미래의 요청을 충족시킨다. 운영체제와 데이터베이스 시스템 양쪽에서 수행된다.

2. **미리 읽기(Read-ahead)**: 디스크 블록에 접근할 때 같은 트랙의 연속 블록을 버퍼에 함께 읽어온다. 순차 접근 시 블록당 탐색/회전 대기 시간을 최소화한다.

3. **스케줄링(Scheduling)**: 여러 블록 요청이 있을 때 디스크 암 이동을 최소화하도록 접근 순서를 조정한다. 엘리베이터 알고리즘은 디스크 암이 한 방향으로 이동하면서 경로상의 요청을 처리하고, 끝에 도달하면 방향을 바꾸는 방식이다.

4. **파일 조직(File Organization)**: 블록을 디스크에 연속적으로 배치하여 순차 접근 시 탐색을 최소화한다. 연속 블록의 그룹인 익스텐트(extent) 단위로 파일에 할당한다.

5. **비휘발성 쓰기 버퍼**: NVRAM(또는 플래시 메모리)을 사용하여 디스크 쓰기를 가속화한다. 데이터베이스가 블록 쓰기를 요청하면 비휘발성 버퍼에 먼저 쓰고 즉시 완료를 보고한다. 실제 디스크 쓰기는 나중에 최적화된 순서로 수행된다.

랜덤 접근 감소는 자기 디스크에서 특히 중요하다. SSD는 랜덤 접근이 훨씬 빠르지만 위 기법들의 일부 혜택은 여전히 유효하다.

## 예시

엘리베이터 알고리즘 동작 예시:

```
디스크 암 현재 위치: 트랙 50 (바깥쪽으로 이동 중)
대기 중인 요청: 트랙 30, 65, 20, 80, 45, 90

처리 순서 (엘리베이터 알고리즘):
1. 트랙 65 (바깥 방향)
2. 트랙 80 (바깥 방향)
3. 트랙 90 (바깥 방향, 더 이상 바깥 요청 없음)
4. 방향 전환 (안쪽으로)
5. 트랙 45 (안쪽 방향)
6. 트랙 30 (안쪽 방향)
7. 트랙 20 (안쪽 방향)

총 암 이동: 50→90→20 = 40 + 70 = 110 트랙
(FCFS 방식보다 효율적)
```
