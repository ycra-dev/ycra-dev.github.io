## 핵심 개념

슬롯 페이지 구조의 블록 헤더에는 다음 정보가 포함된다:
- 헤더의 레코드 엔트리 수
- 블록 내 여유 공간의 끝 위치
- 각 레코드의 위치와 크기를 담은 배열

실제 레코드는 블록의 끝에서부터 연속적으로 할당된다. 여유 공간은 헤더 배열의 마지막 엔트리와 첫 번째 레코드 사이에 연속적으로 존재한다.

레코드 삽입 시 여유 공간 끝에 공간을 할당하고, 헤더에 위치와 크기 엔트리를 추가한다. 레코드 삭제 시 해당 레코드가 차지하던 공간을 해제하고, 삭제된 레코드 앞의 레코드들을 이동시켜 여유 공간이 다시 연속적이 되도록 한다. 레코드 크기 변경도 유사한 기법으로 처리한다.

슬롯 페이지 구조의 핵심 설계 원칙은 간접 참조(indirection)이다. 외부 포인터가 레코드를 직접 가리키지 않고 헤더의 엔트리를 가리킨다. 이 간접 참조 덕분에 블록 내에서 레코드를 자유롭게 이동시켜 단편화를 방지하면서도, 외부의 레코드 참조를 유지할 수 있다.

블록 크기가 제한적(일반적으로 4~8KB)이므로 레코드 이동 비용은 크지 않다.

## 예시

슬롯 페이지 구조:

```
블록 (4KB):
┌─────────────────────────────────────────┐
│ 헤더                                      │
│ ┌─────────────────────────────────────┐ │
│ │ 레코드 수: 3                          │ │
│ │ 여유 공간 끝: 오프셋 2800             │ │
│ │ [슬롯0: 위치=3800, 크기=196]          │ │
│ │ [슬롯1: 위치=3500, 크기=300]          │ │
│ │ [슬롯2: 위치=2800, 크기=700]          │ │
│ └─────────────────────────────────────┘ │
│                                           │
│         ← 여유 공간 (연속적) →             │
│                                           │
│ [레코드2: 700바이트]  (오프셋 2800~)       │
│ [레코드1: 300바이트]  (오프셋 3500~)       │
│ [레코드0: 196바이트]  (오프셋 3800~)       │
└─────────────────────────────────────────┘

외부 포인터 → 슬롯 번호 → 실제 레코드 위치
(레코드 이동 시 슬롯만 갱신하면 됨)
```
