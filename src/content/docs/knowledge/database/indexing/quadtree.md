## 핵심 개념

**PR 쿼드트리(Point-Region Quadtree)**: 점 데이터를 저장하며, 공간을 영역(region) 기반으로 분할한다. 리프 노드는 0개부터 정해진 최대 수의 점을 포함한다. 리프 노드의 점 수가 최대값을 초과하면 4개의 자식 노드로 분할된다.

**영역 쿼드트리(Region Quadtree)**: 배열(래스터) 정보를 저장한다. 노드가 커버하는 영역의 모든 배열 값이 동일하면 리프 노드가 된다. 그렇지 않으면 4개의 동일 면적 자식으로 세분화되는 내부 노드가 된다.

**공간 분할 과정**: 루트 노드는 전체 대상 공간에 대응한다. 각 비리프 노드는 자신의 영역을 4개의 동일 크기 사분면으로 분할하며, 각 사분면에 4개의 자식 노드가 대응한다. 분할은 재귀적으로 수행된다.

**R-트리와의 비교**:
- 쿼드트리는 공간을 고정된 방식으로 분할하므로, 같은 영역에 대한 모든 쿼드트리가 동일한 분할을 가진다. 이는 공간 조인(spatial join)을 단순화한다.
- R-트리는 B+-트리와 유사한 균형 트리로, 경계 상자(bounding box)를 사용하여 객체를 인덱싱한다. 객체를 한 번만 저장하며 최소 50% 채움률을 보장하므로 저장 효율이 좋다.
- R-트리의 경계 상자는 겹칠 수 있어, 검색 시 여러 경로를 탐색해야 할 수 있다. 쿼드트리의 사분면은 겹치지 않는다.
- 실용적으로 R-트리와 그 변형(R*-트리, R+-트리)이 데이터베이스 시스템에서 더 널리 사용된다.

## 예시

```
-- PR 쿼드트리 예시 (최대 점 수 = 1)
-- 점: A(2,7), B(6,8), C(8,3), D(3,4), E(5,2)

전체 공간: (0,0) ~ (10,10)

           Root (0,0)-(10,10)
          /    |    |    \
       NW     NE    SW    SE
     (0,5)-  (5,5)- (0,0)- (5,0)-
     (5,10)  (10,10) (5,5) (10,5)

NW 사분면: A(2,7) → 1개 점, 리프
NE 사분면: B(6,8) → 1개 점, 리프
SW 사분면: D(3,4), E(5,2) → 2개 점 → 분할!
SE 사분면: C(8,3) → 1개 점, 리프

SW 분할:
     SW (0,0)-(5,5)
    /    |    |    \
  NW   NE    SW    SE
(0,2.5)- (2.5,2.5)- (0,0)- (2.5,0)-
(2.5,5) (5,5)    (2.5,2.5) (5,2.5)

SW-NE: D(3,4) → 리프
SW-SE: E(5,2) → 리프 (경계에 있는 경우 규칙에 따라 할당)

-- 범위 질의: (1,1) ~ (7,6) 영역의 점 찾기
1. Root → NW, NE, SW, SE 중 질의 영역과 겹치는 사분면 탐색
2. NW (0,5)-(5,10): 질의 영역 (1,1)-(7,6)과 겹침 → A(2,7) 확인 → y=7>6 → 제외
3. NE (5,5)-(10,10): 겹침 → B(6,8) 확인 → y=8>6 → 제외
4. SW (0,0)-(5,5): 겹침 → 자식 탐색 → D(3,4) ∈ 범위 → 포함!
5. SE (5,0)-(10,5): 겹침 → C(8,3) → x=8>7 → 제외

결과: {D(3,4)}

공간 분할 시각화:
10 +---------+---------+
   |    A    |    B    |
 5 +----+----+---------+
   |    | D  |         |
2.5|----+----|    C    |
   |    | E  |         |
 0 +----+----+---------+
   0  2.5  5        10
```
