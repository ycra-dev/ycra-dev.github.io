## 핵심 개념

전통적인 B+트리나 해시 인덱스는 1차원 데이터에 최적화되어 있어, 2차원 이상의 공간 데이터에 대한 범위 검색(예: 반경 500m 내 식당 찾기)이나 최근접 이웃 검색에는 비효율적이다.

R-트리는 B+트리와 유사하게 리프 노드에 인덱싱된 객체를 저장하며, 균형을 유지한다. 핵심 차이점은 값의 범위 대신 직사각형 경계 상자(bounding box)를 사용한다는 것이다:
- **리프 노드의 경계 상자**: 리프에 저장된 모든 객체를 포함하는 축 평행 최소 직사각형
- **내부 노드의 경계 상자**: 자식 노드들의 경계 상자를 포함하는 축 평행 최소 직사각형

각 내부 노드는 자식 노드의 경계 상자와 자식 포인터를 저장한다. 범위 검색 시 쿼리 영역과 경계 상자가 겹치는 자식만 탐색하여 검색 범위를 축소한다.

R-트리의 대안으로 k-d 트리(공간을 한 차원씩 이분할), k-d-B 트리(k-d 트리의 다중 자식 확장), 쿼드트리(2차원 공간을 4분할) 등이 있다. R-트리는 각 객체를 한 번만 저장하고 노드가 최소 절반 이상 차는 것을 보장하여 저장 효율이 높다. R*-트리, R+-트리 등 변형이 존재하며, PostgreSQL의 GiST 인덱스에서 R-트리가 구현되어 있다.

R-트리는 시간 데이터 인덱싱에도 활용된다. 속성 값을 한 차원으로, 유효 시간 간격을 다른 차원으로 사용하여 시공간 범위 검색을 지원한다.

## 예시

R-트리 구조 예시:

```
2차원 공간에 9개 직사각형 객체 (A~I):

     ┌─────────BB1──────────┐
     │  ┌──┐  ┌──┐  ┌──┐  │
     │  │ A│  │ B│  │ C│  │
     │  └──┘  └──┘  └──┘  │
     └──────────────────────┘
     ┌─────────BB2──────────┐
     │  ┌──┐  ┌──┐  ┌──┐  │
     │  │ D│  │ E│  │ F│  │
     │  └──┘  └──┘  └──┘  │
     └──────────────────────┘
     ┌─────────BB3──────────┐
     │  ┌──┐  ┌──┐  ┌──┐  │
     │  │ G│  │ H│  │ I│  │
     │  └──┘  └──┘  └──┘  │
     └──────────────────────┘

R-트리:
        [BB1, BB2, BB3]        ← 루트 (내부 노드)
       /       |       \
  [A,B,C]   [D,E,F]   [G,H,I]  ← 리프 노드

범위 검색: 특정 직사각형 영역 Q와 겹치는 객체 찾기
1. 루트에서 Q와 BB1, BB2, BB3의 교집합 확인
2. Q가 BB1과만 겹치면 → [A,B,C] 리프만 탐색
3. 리프에서 Q와 실제 겹치는 객체만 반환
```
