## 핵심 개념

희소 인덱스에서 각 엔트리는 검색 키 값과 해당 값을 가진 첫 번째 데이터 레코드에 대한 포인터를 포함한다. 레코드를 찾으려면, 검색하려는 값 이하의 가장 큰 검색 키를 인덱스에서 찾고, 그 포인터가 가리키는 레코드부터 파일을 순차 탐색한다.

희소 인덱스는 밀집 인덱스보다 공간을 덜 차지하고 삽입/삭제 유지 보수 비용이 낮다. 대신 검색 시 해당 블록 내에서 추가 순차 탐색이 필요하여 접근 시간이 약간 더 길다. 블록당 하나의 엔트리를 두는 것이 일반적인 절충안이다.

인덱스 자체가 커져서 메모리에 다 담기 어려울 때, 인덱스를 순차 파일처럼 취급하여 그 위에 희소 외부 인덱스(outer index)를 구성할 수 있다. 이를 다단계 인덱스(multilevel index)라 한다. 외부 인덱스에서 이진 탐색으로 검색 범위를 크게 줄인 뒤, 내부 인덱스의 해당 부분을 탐색한다.

다단계 인덱스의 개념은 B+트리의 트리 구조와 밀접하게 관련된다. B+트리는 사실상 동적으로 관리되는 다단계 인덱스라 할 수 있다.

사전(dictionary)의 각 페이지 상단에 표시된 첫 번째 단어가 희소 인덱스의 좋은 비유이다. 페이지가 블록에 해당하고, 상단의 단어가 인덱스 엔트리에 해당한다.

## 예시

희소 인덱스와 다단계 인덱스:

```
instructor 파일 (ID 기준 정렬):
블록0: [10101, ...] [12121, ...] [15151, ...]
블록1: [22222, ...] [32343, ...] [33456, ...]
블록2: [45565, ...] [58583, ...]
블록3: [76543, ...] [76766, ...]
블록4: [83821, ...] [98345, ...]

희소 인덱스 (블록당 1개 엔트리):
10101 → 블록0
22222 → 블록1
45565 → 블록2
76543 → 블록3
83821 → 블록4

검색: ID = 33456
1. 인덱스에서 33456 이하의 최대값: 22222
2. 블록1로 이동
3. 블록1 내에서 순차 탐색: 22222 → 32343 → 33456 발견!

다단계 인덱스:
외부 인덱스 (희소):  [10101 → 내부 인덱스 블록A]
                     [45565 → 내부 인덱스 블록B]

내부 인덱스 블록A:   [10101 → 블록0] [22222 → 블록1]
내부 인덱스 블록B:   [45565 → 블록2] [76543 → 블록3] [83821 → 블록4]
```
