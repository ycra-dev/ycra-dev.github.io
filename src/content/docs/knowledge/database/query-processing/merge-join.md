## 핵심 개념

병합 조인은 등가 조인과 자연 조인에 사용되며, 두 릴레이션이 조인 속성에 대해 정렬되어 있어야 한다. 이미 정렬되어 있지 않다면 외부 정렬-병합을 사용하여 먼저 정렬한다.

**동작 원리:** 두 릴레이션의 정렬된 튜플을 동시에 스캔한다. 조인 속성 값이 같은 튜플들을 매칭한다. 한쪽 릴레이션의 현재 값이 다른쪽보다 작으면 해당 릴레이션의 포인터를 전진시킨다. 같은 조인 속성 값을 가진 튜플이 여러 개 있으면, 해당 그룹의 모든 조합을 생성한다.

**비용 분석:** 두 릴레이션이 이미 정렬되어 있는 경우, 병합 조인은 각 릴레이션을 한 번만 스캔하면 된다. 비용은 `b_r + b_s` 블록 전송이며, 정렬이 필요한 경우 정렬 비용이 추가된다. 각 블록 읽기에 b_b 블록을 사용하면 탐색 수는 `⌈b_r/b_b⌉ + ⌈b_s/b_b⌉`이다.

병합 조인의 장점은 각 릴레이션이 정렬되어 있으면 매우 효율적이라는 점이다. 특히 릴레이션이 이미 정렬되어 있거나, 클러스터링 인덱스가 있는 경우에 유리하다. 또한 결과도 조인 속성에 대해 정렬된 상태로 출력되므로, 후속 정렬이 필요한 연산(ORDER BY 등)에서 추가 비용이 발생하지 않는다.

병합 조인은 중첩 루프 조인에 비해 대부분의 경우 훨씬 효율적이며, 해시 조인과 비교했을 때도 데이터가 이미 정렬되어 있으면 더 나은 성능을 보인다. 다만 정렬이 필요한 경우 외부 정렬의 비용이 추가되어야 한다.

## 예시

student ⋈ takes 자연 조인(ID 속성으로)에서, 두 릴레이션이 이미 ID로 정렬되어 있다고 가정하면:

```
비용:
  블록 전송 = b_student + b_takes = 100 + 400 = 500
  디스크 탐색 = ⌈100/b_b⌉ + ⌈400/b_b⌉

b_b = 1인 경우: 500 블록 전송 + 500 탐색
```

정렬되지 않은 경우, 정렬 비용 추가:

```
student 정렬: b_r=100, M=20
  런 수 = ⌈100/20⌉ = 5, 병합 패스 = ⌈log_19(5)⌉ = 1
  정렬 비용 = 100 * (2*1 + 1) = 300 블록 전송

takes 정렬: b_r=400, M=20
  런 수 = ⌈400/20⌉ = 20, 병합 패스 = ⌈log_19(20)⌉ = 2
  정렬 비용 = 400 * (2*2 + 1) = 2000 블록 전송

총 비용 = 300 + 2000 + 500 = 2,800 블록 전송
```
