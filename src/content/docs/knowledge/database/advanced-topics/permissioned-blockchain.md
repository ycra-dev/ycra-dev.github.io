## 핵심 개념

공개 블록체인(Bitcoin 등)은 누구나 참여할 수 있어 완전한 탈중앙화를 추구하지만, 이로 인해 시빌 공격 방어를 위한 작업 증명과 같은 비용이 큰 합의 메커니즘이 필요하다. 허가형 블록체인은 참여를 통제함으로써 이러한 비효율성을 크게 줄일 수 있다.

**허가형 블록체인의 특성**:
- 허가 기관이 노드의 참여를 승인하므로 시빌 공격이 불가능하다.
- 중앙 통제는 접근 인가와 신원 관리에만 제한되며, 그 외의 모든 행동은 탈중앙화 방식으로 이루어진다.
- 참여자의 신원이 알려져 있으므로 익명성은 제한적이거나 없을 수 있다.
- 비잔틴 합의(Byzantine Consensus)와 같은 메시지 기반 합의 알고리즘을 사용할 수 있어, 작업 증명이나 지분 증명보다 훨씬 빠르고 효율적이다.

**악의적 행동의 가능성**: 허가형 블록체인에서도 모든 사용자가 완전히 정직하다고 가정할 수 없다. 예를 들어, 공급망 블록체인에서 공급업체가 자신에게 유리하게 데이터를 위조하거나, 사기 조사가 시작되면 블록체인을 포크하여 증거를 은폐하려 할 수 있다. 따라서 허가형 블록체인에서도 비잔틴 장애 모델을 적용한다.

**성능 향상 기법**:
1. **샤딩(Sharding)**: 블록체인의 계정을 샤드로 분할하여 병렬로 마이닝한다. 샤드 간 트랜잭션은 별도의 교차 샤드 트랜잭션으로 처리한다.
2. **오프체인 트랜잭션 처리**: 신뢰할 수 있는 시스템이 트랜잭션을 내부적으로 처리하고, 그룹화하여 블록체인에 기록한다. Lightning 네트워크가 대표적이다.

**데이터 관리**: 허가형 블록체인은 전통적 데이터베이스나 NoSQL 데이터베이스를 사용하여 상태를 저장하고 쿼리를 지원할 수 있다.
- **Hyperledger Fabric**: NoSQL 데이터베이스를 사용하여 상태를 저장한다.
- **Corda**: 임베디드 SQL 데이터베이스를 사용한다.
- **BigchainDB**: NoSQL 데이터베이스를 사용하며 블록체인 데이터베이스에 특화되어 있다.

이러한 데이터베이스 통합은 스마트 컨트랙트가 범용 데이터베이스 쿼리를 실행할 수 있게 하지만, 진정한 블록체인의 엄격한 암호학적 보호가 부족할 수 있다. 적절한 절충안은 데이터베이스를 신뢰할 수 있는 제공자가 호스팅하면서, 업데이트를 데이터베이스와 블록체인 모두에 기록하여 사용자가 원할 때 블록체인으로 데이터베이스를 검증할 수 있게 하는 것이다.

## 예시

공개 블록체인 vs 허가형 블록체인 비교:

```
┌──────────────┬───────────────────┬────────────────────┐
│  특성         │ 공개 블록체인      │ 허가형 블록체인      │
├──────────────┼───────────────────┼────────────────────┤
│ 참여 방식     │ 누구나 참여 가능   │ 허가 기관 승인 필요  │
│ 합의 메커니즘 │ PoW / PoS         │ Byzantine Consensus │
│ 처리 속도     │ 느림 (<10 TPS)    │ 빠름 (수천 TPS 가능) │
│ 에너지 소비   │ 매우 높음          │ 낮음                │
│ 익명성        │ 높음 (의사 익명)   │ 낮음 또는 없음      │
│ 시빌 공격     │ 가능 (PoW로 방어)  │ 불가능 (허가 통제)   │
│ 사용 사례     │ Bitcoin, Ethereum │ Hyperledger, Corda  │
└──────────────┴───────────────────┴────────────────────┘
```

허가형 블록체인 아키텍처 예시 (공급망):

```
허가 기관: 제조업체 컨소시엄

참여 노드:
  - 원자재 공급업체 (노드 1-5)
  - 부품 제조업체 (노드 6-10)
  - 최종 조립 공장 (노드 11-15)
  - 물류 회사 (노드 16-20)
  - 감사 기관 (노드 21)

합의: PBFT (Practical Byzantine Fault Tolerance)
  - 전체 21 노드 중 최대 ⌊(21-1)/3⌋ = 6 노드의
    악의적 행동을 허용하면서 합의 유지

각 트랜잭션:
  "부품 P-123이 노드 7에서 노드 12로 이동"
  → 디지털 서명으로 부인 불가
  → 블록체인에 영구 기록
```
