## 핵심 개념

**장애 모델의 차이**: 전통적인 분산 합의 프로토콜(Paxos, Raft)은 **장애 중단(fail-stop)** 모델을 가정한다. 이 모델에서 노드는 작동을 멈추거나 연결이 끊어지는 것만 가능하며, 악의적 행동은 불가능하다. 반면, **비잔틴 장애(Byzantine failure)** 모델에서는 "장애" 노드가 임의의 방식으로 행동할 수 있다고 가정한다. 이에는 합의를 방해하거나, 거짓 메시지를 보내거나, 네트워크를 파괴하는 데 필요한 정확한 단계를 수행하는 것까지 포함된다.

예를 들어, 악의적 노드 n1이 노드 n2에게는 트랜잭션을 커밋하겠다고 말하면서, 노드 n3에게는 중단하겠다고 말할 수 있다. 이러한 상황에서도 올바른 합의에 도달해야 한다.

**비잔틴 합의의 역사와 발전**: 비잔틴 합의 알고리즘의 개발은 1980년대 초에 시작되었다(Pease et al., 1980; Lamport et al., 1982). 초기 연구는 네트워크의 메시지 전달 시간이나 네트워크의 동기성에 대한 가정에 기반했다. 현대의 비잔틴 합의 알고리즘은 실제 환경에 기반하며, 위조 메시지를 방지하기 위한 암호학적 서명을 통합한다.

**실용적 비잔틴 장애 허용(PBFT)**: 가장 널리 사용되는 접근법인 PBFT(Practical Byzantine Fault Tolerance)는 최대 ⌊(n-1)/3⌋ 노드의 악의적 장애를 허용한다. 즉, 전체 n개 노드 중 최대 약 1/3의 노드가 악의적으로 행동해도 나머지 정직한 노드들이 올바른 합의에 도달할 수 있다. 이는 기본 합의 프로토콜보다 더 많은 메시지 교환을 필요로 하지만, 작업 증명이나 지분 증명보다 비용이 훨씬 낮다.

**블록체인에서의 적용**: 비잔틴 합의는 시빌 공격 문제를 해결하지 못하므로, 공개 블록체인에는 적합하지 않다. 그러나 노드 진입이 통제되는 허가형 블록체인에서는 매우 효과적이다. 허가 기관이 악의적 사용자의 과도한 노드 추가를 쉽게 거부할 수 있기 때문이다.

**장애 중단 vs 비잔틴 장애 비교**:
- 장애 중단: 노드가 응답을 멈추거나 메시지를 잃는 것만 허용. 2PC, Paxos, Raft가 이 모델을 사용.
- 비잔틴 장애: 노드가 어떤 행동이든 할 수 있음. 거짓 정보 전송, 선택적 메시지 전달, 의도적 지연 등 포함.

**확장성 고려사항**: 2PC에서 고정된 수의 노드(예: 5개)에 접근하는 트랜잭션은 시스템이 확장되어도 5개 노드의 합의만 필요하다. 반면, 비잔틴 합의에서는 모든 트랜잭션이 비장애 노드의 과반수 합의를 필요로 하므로, 노드 수가 증가하면 합의에 필요한 노드 수도 증가한다. 이는 확장성에 제약을 준다.

완전 비동기적 장애 허용 합의는 이론적으로 불가능함이 증명되었다(FLP 불가능성). 따라서 실용적 프로토콜은 부분적 동기성 가정하에 운영된다.

## 예시

비잔틴 장애의 예시 (공급망 시나리오):

```
공급망 블록체인 참여자:
  - 공급업체 A (정직)
  - 공급업체 B (악의적)
  - 제조업체 C (정직)
  - 물류사 D (정직)

시나리오:
  공급업체 B가 불량 부품을 공급하고
  데이터를 위조하려 시도:

  B → A: "부품 P의 품질 등급은 A입니다"
  B → C: "부품 P의 품질 등급은 B입니다"
  B → D: "부품 P의 품질 등급은 A입니다"

  PBFT 합의:
  - 4개 노드 중 최대 ⌊(4-1)/3⌋ = 1개 악의적 허용
  - 정직한 3개 노드(A, C, D)가 불일치 감지
  - 정직한 과반수(3 > 4×2/3)로 올바른 합의 도달
```

PBFT 프로토콜 메시지 흐름 (단순화):

```
클라이언트 → 리더: 요청(Request)

1단계: Pre-prepare
  리더 → 모든 노드: pre-prepare 메시지

2단계: Prepare
  각 노드 → 다른 모든 노드: prepare 메시지
  (2f+1개의 prepare 수신 시 "prepared" 상태)

3단계: Commit
  각 노드 → 다른 모든 노드: commit 메시지
  (2f+1개의 commit 수신 시 실행 및 응답)

f = 허용 가능한 악의적 노드 수 = ⌊(n-1)/3⌋
총 메시지 수: O(n²) (기본 합의의 O(n)보다 높음)
```

확장성 비교:

```
┌─────────────┬──────────┬──────────────────┐
│ 노드 수      │ 2PC      │ Byzantine        │
│             │ (5노드)  │ Consensus        │
├─────────────┼──────────┼──────────────────┤
│ 10 노드     │ 5개 합의 │ 7개 이상 합의     │
│ 100 노드    │ 5개 합의 │ 67개 이상 합의    │
│ 1000 노드   │ 5개 합의 │ 667개 이상 합의   │
└─────────────┴──────────┴──────────────────┘

→ 비잔틴 합의는 노드 증가 시 확장성이 제한됨
```
