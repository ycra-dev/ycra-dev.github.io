멀티태스킹 (Multitasking)

핵심 개념

섹션 제목: “핵심 개념”

멀티태스킹은 멀티프로그래밍의 논리적 확장으로, CPU가 여러 프로세스 간에 빠르게 전환하여 사용자에게 빠른 응답 시간을 제공하는 시스템이다.

프로세스가 실행될 때, 보통 완료되거나 I/O를 수행해야 하기 전까지 짧은 시간만 실행된다.

멀티프로그래밍 vs 멀티태스킹

섹션 제목: “멀티프로그래밍 vs 멀티태스킹”
구분멀티프로그래밍멀티태스킹
목적CPU 활용률 향상빠른 응답 시간 제공
전환 시점프로세스가 대기할 때 전환자주 전환 (타임 슬라이스)
초점시스템 효율성사용자 경험

동작 원리

섹션 제목: “동작 원리”
  • CPU가 여러 프로세스 간에 전환하며 실행하되, 전환이 자주 발생
  • 사용자에게 빠른 응답 시간(response time) 제공
  • 대화형 컴퓨팅 지원

대화형 I/O의 특성

섹션 제목: “대화형 I/O의 특성”
특성설명
입력 소스키보드, 마우스, 터치스크린
출력 대상디스플레이
속도”사람 속도”로 실행되어 완료하는 데 오래 걸릴 수 있음

예: 입력은 사용자의 타이핑 속도에 제한될 수 있음 (초당 7자는 사람에게 빠르지만 컴퓨터에는 매우 느림)

CPU가 대화형 입력이 진행되는 동안 유휴 상태로 두지 않고, 운영체제가 빠르게 CPU를 다른 프로세스로 전환한다.

멀티태스킹의 요구사항

섹션 제목: “멀티태스킹의 요구사항”
요구사항설명
메모리 관리여러 프로세스를 메모리에 동시에 유지
CPU 스케줄링여러 프로세스가 실행 준비 시 다음 프로세스 선택
가상 메모리메모리에 완전히 없는 프로세스의 실행 허용

가상 메모리 (Virtual Memory)

섹션 제목: “가상 메모리 (Virtual Memory)”

합리적인 응답 시간을 보장하는 일반적인 방법:

  • 메모리에 완전히 없는 프로세스의 실행을 허용
  • 사용자가 실제 물리 메모리보다 큰 프로그램을 실행할 수 있게 함
  • 주 메모리를 크고 균일한 저장 배열로 추상화
  • 프로그래머를 메모리 저장 제한에 대한 걱정에서 해방
